Anaconda

패키지 관리와 배포를 단순화하려는 목적으로 과학 컴퓨팅을 위해 Python과 R 프로그래밍 언어의 배포판

Conda 가상 환경 생성

# 가상환경 이름과 사용할 파이썬 버전을 지정하여 가상환경 생성
$ conda create --name <virtual env name> python=<python version>

# 사용자가 지정한 경로에 가상환경 생성
$ conda create --prefix=<virtual env path>

Conda의 가상 환경 복제

# 지정한 경로에 기존 가상환경을 복제하여 새로 생성
$ conda create --prefix=[가상환경 경로] --clone=[복제할 가상환경명]

conda 가상 환경 접속

# 가상환경 접속(activate)
$ conda activate <virtual env>

Pip

Pip Installs Packages의 약자로 Python 패키지를 설치하고 관리하는 패키지 관리자

Pip으로 특정경로에 패키지 설치

$ pip install -target=[경로] [라이브러리명]

# example
$ pip install -target=/usr/.local/lib/python3.8/site-packages sqlalchemy

app 내부에서 import 가능한 패키지 보기

# 리스트 형식의 패키지와 버전을 이쁘게 보기 위해 사용
from pprint import pprint

# 파이썬의 내장 모듈. working_set의 reuturn type은 &lt;class 'pkg_resources.WorkingSet' &gt;. 이를 리스트로 변환 필요하다.
from pkg_resources import working_set
pprint(list(working_set))

# 출력 
# numpy 1.21.6 (/home/user/b2en/.anaconda3/lib/python3.7/site-packages)

# 해석 : [패키지] [버전] ([패키지 경로])

# pip install [패키지를 하면] 기본적인 설치경로는 anaconda 디렉토리의 lib/python<version>/site-packages 이다.

My Tip

가끔 pip uninstall이라는 패키지 삭제 명령어가 오류가 날 때가 있다. 이런 경우 파이썬 패키지가 어떤 경로에서 임포트하는 지 해당 과정을 알면 직접 패키지를 수동으로 삭제할 수도 있다.

site-package 위치 찾기

다음 커맨드를 치면 현재 사용중인 파이썬(가상환경을 쓰고 있다면 가상환경으로 반드시 접속해야 한다.)이 패키지를 임포트하는 경로들이 나온다. 그 중 site-package라는 단어가 있는 경로로 접속한다.

$ python -m site

그럼 여러 폴더들이 나오는 데, 배열(리스트)의 첫 번째 부터 순차적으로 임포트할 패키지가 있는 지 확인한다. 그 중 site-package 폴더 안에는 또 여러개의 폴더(각각 설치한 패키지)가 있고, 해당 폴더를 지우시면 패키지가 삭제됩니다.

가끔 파이썬 패키지 중에는 바이너리(bin) 파일을 가진 패키지들이 있다. 예를 들면 airflow, dbt들은 CLI로 실행이 가능한 패키지이다. 이런 패키지들을 수동으로 제거할 때는 바이너리 파일이 별도의 폴더에 저장되기 때문에 별도로 제거하는 과정이 더 필요하다. 바이너리 파일의 위치는 아래 명령어로 찾을 수 있습니다.

$ which <실행 가능한 패키지>

$ which poetry
/opt/homebrew/bin/poetry

Jupyter Lab

notebooks, code, data 작업을 위한 최신 웹 기반 인터랙티브 개발 환경. 인터페이스는 사용자들이 data science, scientific computing, computational journalism, machine learning에서 워크플로우를 구성하고 배열할 수 있도록 해준다.

Jupyter Lab 설치

$ pip install jupyter-lab

jupyter_lab_config.py

JupyterLab의 설정 파일. 일반적으로 사용자의 홈의 서브 디렉터리(~/.jupyter)에 위치한다.

# jupyter_lab_config.py 파일이 존재하지 않는 경우, 커맨드로 생성
$ jupyter-lab --generate-config

Config 설정

c.ServerApp.ip = ['IP'] # '0.0.0.0' : 모든 IP 접속 허용
c.ServerApp.port = [PORT] # jupyter 프로세스를 접속할 수 있는 개방 포트
c.ServerApp.notebook_dir = ['경로']
c.ServerApp.password = ['패스워드']

jupyter-lab 실행

$ jupyter-lab

Tensorflow

수치 계산, 대규모 머신 러닝, 딥러닝, 기타 통계 및 예측 분석 워크로드를 위한 오픈 소스 라이브러리

Tensorflow GPU 세팅하기

  • tensorflow GPU 호환 버전 확인
  1. tensorflow 설치 (cpu, gpu 통합)
# tensorflow 설치
$ pip install tensorflow=[버전]

# GPU 사용을 위한 cuda 라이브러리 설치
$ conda install -c conda-forge cudatoolkit=[호환버전] cudnn=[호환버전]
  1. device(cpu, gpu) 상태 확인
from tensorflow.python.client import device_lib
device_lib.list_local_devices()
  1. gpu 사용 옵션 제거 (cpu 사용)
  • GPU 세팅이 되면 default로 GPU를 사용하도록 되어있다. GPU 사용을 끄고 싶다면 프로그램 실행 초반부에 아래와 같은 코드를 작성한다.
import os
os.environ["CUDA_VISIBLE_DEVICES"] = "0"
