[Python] pandas의 append()를 안 쓰기 시작한 이유

Publish Date:   2023-06-23
Python

pandas.DataFrame.append() 의 deprecated

pandas.DataFrame.append() 함수를 쓰면 FutureWarning 이 발생하면서  pandas.concat() 함수를 대신 사용하라고 주의를 준다. 그럼 왜 concat을 대신 사용하라고 할까? 그 이유는 pandas.DataFrame.append() 함수는 메모리 사용과 처리 서능에 있어 비효율적인 문제가 있기 때문이다.  

  • pandas.DataFrame.append()는 내부적으로 결합할 데이터프레임을 반드시 복사(copy)하여 결합한다.

  • 데이터가 크면 클수록 처리할 때 메모리 부족(Out Of Memory)가 발생할 가능성이 높고 처리 속도가 느리다.

pandas.concat() 사용하기

pandas.concat() 함수는 여러 데이터프레임을 Iterable 객체 안에 넣어 행 또는 열로 병합하는 함수이다. 그리고 옵션 설정을 통해 데이터프레임을 copy하지 않고 병합 가능하다.  

  • 데이터프레임을 복사하지 않고 연결가능하다. (copy 옵션을 변경가능하다. default value는 true)

  • 이에 따라 메모리 절약과 성능 향상의 이점이 있다.

pandas.DataFrame.from_dict() 활용하기

기존에 주로 사용하던 pandas.DataFrame.append() 함수를 쓰지 않으려고 하다보니 한 가지 불편한 문제가 생겼다. kafka에서 받은 하나의 row를 데이터프레임에 추가하려고 하다 보니 append()에 아규먼트로 딕셔너리를 스키마에 맞게 바로 넣어주면 데이터를 바로 추가할 수 있었던 그러한 이점을 활용 못하게 되었다. 왜냐하면 concat() 함수는 결합 대상을 dict가 아닌 Series or DataFrame objects의 sequence를 받기 때문에 중간에 dict를 데이터프레임으로 변환하는 과정이 필요하다.

물론 주의를 주는 append() 함수를 쓰더라도 딕셔너리 객체 하나를 삽입하기 때문에, copy 하는 데 큰 메모리가 들지 않겠지만 그래도 향후 deprecated 될 가능성이 있고, 코드 상의 통일성을 맞추기 위해 앞으로는 concat() 함수를 사용하기로 하였다.

그럼 하나의 데이터프레임에 하나의 딕셔너리 객체를 추가하려면 어떤 함수를 사용해야 할까? 그 때 사용하기 좋은 함수가 바로 pandas.DataFrame.from_dict() 이다. 이름에 걸맞게 딕셔너리를 dataframe으로 변환해주는 함수이다. DataFrame 클래스의 클래스메서드이며, 필수 파라미터로 data에 key : value인 딕셔너리 객체를 넣어준다.

@classmethod
def from_dict(
    cls,
    data: dict,
    orient: str = "columns",
    dtype: Dtype | None = None,
    columns: Axes | None = None,
) -> DataFrame:

 
from_dict() 함수를 이용해서 데이터프레임을 행으로 병합하는 코드를 작성해보자.  

import pandas as pd

# 딕셔너리 데이터
data = {
    'Name': ['John', 'Emma', 'Peter'],
    'Age': [25, 28, 30],
    'City': ['New York', 'London', 'Paris']
}

# dict to pandas.DataFrame
df = pd.DataFrame.from_dict(data)

print(df)

######### 출력 #########
#    Name  Age      City
# 0  John   25  New York
# 1  Emma   28    London
# 2 Peter   30     Paris

concat() 사용시 copy 옵션을 False로 할 때 주의할 점이 있다.  

  • 데이터프레임으로 결합할 딕셔너리의 값(value)은 array-like 형태로 넣어주어야 한다.

  • 결합 대상이 되는 데이터프레임은 더 이상 바뀌지 않은 불변(immutable) 객체로 사용해야 한다.
