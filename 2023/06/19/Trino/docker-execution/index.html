[Trino] Docker Test

Publish Date:   2023-06-19
Trino

사전 준비

Trino_The Definitive Guide에 나와 있는 대로, 도커를 이용해서 가볍게 테스트 할 수 있습니다. trino 이미지에는 테스트용 데이터가 준비되어 있기 때문에 따로 데이터를 준비할 필요는 없습니다.

Docker Image Download & Container Running

docker run 명령어를 사용하면 dockerhub에서 이미지를 다운받으면서 바로 컨테이너를 실행할 수 있습니다. 아래 커맨드는 trino offical 이미지 레포지터리인 trinodb/trino에서 먼저 이미지를 다운받습니다. 그리고 내 로컬 컴퓨터에서 trino-trial라는 컨테이너 네임을 지정하여 detach(백그라운드 실행)으로 8080 포트를 열어 프로세스를 뛰웁니다.

$ docker run -d -p 8080:8080 --name trino-trial trinodb/trino

# docker command를 모르겠다면 --help 옵션을 통해 확인 가능합니다.
Usage:  docker run [OPTIONS] IMAGE [COMMAND] [ARG...]
  -d, --detach : Run container in background and print container ID
  -p, --publish :  Publish a container's port(s) to the host
  --name : Assign a name to the container

Trino Interactive Shell

현재 실행된 컨테이너는 백그라운드로 동작만 하고 있기 때문에 아래 커맨드 사용을 통해 터미널에서 interactive하게 쿼리를 날릴 수 있도록 trino라는 명령어로 실행해 달라고 요청합니다.

$ docker exec -it trino-trial trino

Usage:  docker exec [OPTIONS] CONTAINER COMMAND [ARG...]
  -i, --interactive : Keep STDIN open even if not attached
  -t, --tty :  Allocate a pseudo-TTY

명령어를 실행하면 trino에서 쿼리를 날릴 수 있는 대화형 쉘이 나옵니다. 아래 이미지는 help를 통해 사용가능한 명령어 모음을 살펴보았습니다. 그럼 다시 이어서 가이드의 예제를 수행해보겠습니다. trino 이미지는 내부에 테스트 데이터를 이미 포함하고 있습니다. 다음 커맨드를 통해 SELECT문을 확인합니다.

SELECT COUNT(*) FROM tpch.sf1.nation

Trino Monitoring Web

전 단계에서 -p 로 포트포워딩을 했습니다. 내 local에서 8080 포트의 웹 서버(http://localhost:8080)로 연결하면 아래 이미지처럼 trino의 작업을 모니터링 웹이 볼 수 있습니다.

계정과 비밀번호는 정해지지 않고 치면 들어가집니다.

  • RUNNING QUERIES : Total number of queries currently running

  • QUEUED QUERIES : Total number of queries currently queued and awaiting execution

  • BLOCKED QUERIES : Total number of queries currently blocked and unable to make progress

  • ACTIVE WORKERS : Total number of active worker nodes

  • RUNNABLE DRIVERS : Moving average of total running drivers

  • RESERVED MEMORY (B) : Total amount of memory reserved by all running queries

  • ROWS/SEC : Moving average of input rows processed per second

  • BYTES/SEC : Moving average of input bytes processed per second

  • WORKER PARALLELISM : Moving average of CPU time utilized per second worker
