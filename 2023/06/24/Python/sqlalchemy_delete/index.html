[Python] sqlalchemy delete

No tag
Python
Publish Date:   2023-06-24

Problem

spark.sql API로 적재하던 해당 날짜의 데이터를 지울 일이 생겼다. 모듈은 Airlfow의 스케줄링에 의해 배치로 돌아가는 상황인데, 실패시 retry를 수행하기 때문에 사실상 다시 재실행하면 기존 날짜로 적재하던 테이블을 다시 비워주고 데이터를 넣어야 했다.

Think

그것을 하기위해 어떻게 할까 고민을 했고, python에서 사용할 수 있는 sqlalchemy 모듈을 사용해 보았다. sqlalchemy에서 쿼리를 수행하기 위해 접속하는 방법은 여러 가지가 있는 것 같다. 그 중에서 나는 session을 이용하기로 하였다. 그 이유는 내가 만든 프로그램은 한 번 실행하면 세 개의 테이블에 데이터를 적재하기 때문에 세 개의 테이블에 데이터를 비워줘야 했기 때문이다. 3번의 연결을 하고 끊고를 반복하는 것보다는 한 번 세션을 연결하고 처리를 하고 세션을 닫는 게 훨씬 더 효율적이라 판단하였다.

Solve

sqlalchemy를 사용하여 postgres에서 delete문을 수행하는데, 역시나 delete문의 속도가 TRIGGER ALL (트리거의 동작범위)에 의해 많은 차이가 났다. 해당 데이터를 삽입하는 테이블은 다른 테이블 간의 관계성이 없기 때문에 데이터 삭제 전 ALTER TABLE {table} DISABLE TRIGGER ALL 을 수행하여 트리거를 비활성화하고, 데이터 삽입 후 ALTER TABLE {table} ENABLE TRIGGER ALL 를 수행하여 다시 원상 복구했더니 delete문의 속도가 많이 개선되었다.

import logging
from typing import Tuple

from sqlalchemy import create_engine
from sqlalchemy.orm import sessionmaker

def delete_table_data(today: str):
	
    # postgresSQL 연결 정보
    user = None # set your user
    password = None # set your user pasword
    host = "localhost" # 127.0.0.1
    port = 5432 # postgreSQL default port
    database = None # set your db name
    
    # an Engine, which the Session will use for connection
    engine = create_engine(f"postgresql://{user}:{password}@{host}:{port}/{database}")

    # 세션 팩토리 생성
    Session = sessionmaker(engine)

    # 세션을 사용하여 DB 작업 수행 (with context 문을 사용하면 session.close()을 자동적으로 수행한다)
    with Session() as session:
        try:
            talbes: Tuple[str] = (
                "SOSS.DM_SAFE_IDEX_GRID", # my table 1
                "SOSS.DM_SAFE_IDEX_ADMD", # my table 2
                "SOSS.DM_SAFE_IDEX_SGG" # my table 3
            )

            for table in talbes:
                # 데이터 삭제
                session.execute(f"DELETE FROM {table} WHERE stdr_de='{today}'")
                
            # 삭제를 했다면 commit. commit이 실행되지 않으면 삭제가 되지 않는다.
            session.commit()
            logging.info(f"{today} 날짜 삭제!")
    except:
        # 예외가 발생한 경우 롤백
        session.rollback()

if __name__ == "__main__":
    # logging setting
    format = "%(asctime)s: %(message)s"
    logging.basicConfig(format=format, level=logging.INFO, datefmt="%H:%M:%S")

    # execute my function
    delete_table_data(today)
