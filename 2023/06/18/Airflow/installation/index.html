[Airflow] Installation

Airflow
Airflow
Publish Date:   2023-06-18

인지 사항

  • 프로그램을 설치하는 방법은 블로그마다 매우 상이합니다. 이는 버전에 따라 달라지기도 하고, 운영할 환경에 따라 달라지므로 사용할 환경을 먼저 이해하는 것이 중요합니다.예를 들어, celery나 kubernetes로 airflow를 운영한다면 확장성을 고려해 컨테이너 기반으로 사용하는 게 좋을 것입니다. 반면, 테스트용으로 단일 서버에서 간단히 시험해보려면 local executor로 운영해도 무리는 없습니다.

  • 항상 달라지는 환경에서 내가 필요한 환경을 이해하고 맞춰 설치하는 가장 옳은 방법은 무엇보다도 공식 문서를 읽는 확인하는 것입니다.

Installtion Using PyPI

1. Python pip install apach-airlfow

# Airflow 패키지 설치(특정 버전 기준으로 호환 패키지 같이 설치)
$ CONSTRAINT_URL="https://raw.githubusercontent.com/apache/airflow/constraints-${AIRFLOW_VERSION}/constraints-${PYTHON_VERSION}.txt"

$ pip install "apache-airflow==${AIRFLOW_VERSION}" --constraint "${CONSTRAINT_URL}"

# example
$ pip install "apache-airflow==2.6.0" --constraint "https://raw.githubusercontent.com/apache/airflow/constraints-2.6.0/constraints-3.8.txt"

2. AIRFLOW_HOME 환경변수 설정 및 기초파일 초기화

# AIRFLOW_HOME 폴더 생성
$ mkdir ~/airflow

# bash_profile(또는 bash_rc)에 환경변수 추가
$ echo "export AIRFLOW_HOME=~/airflow" >> .bash_profile

# airflow db 초기화
$ airflow db init

3. Airflow 관리자 계정 생성

$ airflow users create -h

# 관리자 생성 명령어 입력 후 초기 비밀번호 입력
$ airflow users create \
    --username <계정> \
    --firstname <성> \
    --lastname <이름> \
    --role Admin \
    --email <이메일>

4. Airflow 메타데이터 DB로 PostgreSQL 로 연동하기

먼저, 백엔드로 사용할 수 있는 DBMS는 Airflow Version이나 시간의 변화에 따라 추가되거나 변동 될 수 있음을 인지해야 합니다. 상황에 따라 달라질 수 있으니 항상 설치 시점에 이용가능한 백엔드 데이터베이스를 확인해야 합니다. Choosing database backend]

| 백엔드 DBMS 리스트 | 버전 |
|——————————————–
| PostgreSQL | 11, 12, 13, 14, 15 |
| MySQL | 5.7, 8 |
| MSSQL (Experimental) | 2017, 2019 |
| SQLite | 3.15.0+ |

저는 오픈소스인 PostgreSQL을 메타스토어로 사용하였다.

PostgreSQL 데이터베이스 설정

  • 데이터베이스는 UTF-8 문자 집합을 사용해야 한다.
-- PostgreSQL 데이터베이스 생성
CREATE DATABASE <airflow_db>;

-- PostgreSQL 유저 생성
CREATE USER <airflow_user> WITH PASSWORD '<airflow_pass>';

-- PostgreSQL 유저에게 Airflow 데이터베이스 권한부여
GRANT ALL PRIVILEGES ON DATABASE <airflow_db> TO <airflow_user>;

-- PostgreSQL 15 버전 이상시 추가 권한 부여
USE <airflow_db>;
GRANT ALL ON SCHEMA public TO <airflow_user>;

  • 데이터베이스 URI에서 특정 스키마를 지정하는 방법을 노출하지 않기 때문에, Postgres 사용자의 search_path에 스키마 public이 포함되어 있는지 확인

    • 새로운 Postgres 사용자의 기본 search_path는 “$user”, public

    • 기존의 Postgres 사용자를 사용하고 사용자 지정 search_path가 있는 경우, 다음 명령을 사용하여 search_path를 변경

ALTER USER airflow_user SET search_path = public;

연결이 안된다면 PostgreSQL pg_hba.conf 수정하기

사용자를 데이터베이스 액세스 제어 목록에 pg_hba.conf 추가하기

/etc/postgresql/~/main 경로에 있는 pg_hba.conf 파일 열고 아래와 같이 외부에서 접근 가능한 IP를 개방해준다. 0.0.0.0/0는 모든 IP에 대한 개방을 허용하겠다는 의미이다.

pg_hba.conf
# IPv4 local connections:
host		all             all             0.0.0.0/0               md5

Q & A

  • Q. airflow에서 기본적으로 sqlite db를 제공하는 데, 왜 따로 backend database를 구축하여 사용하는 가?

  • A. sqlite를 기본적으로 사용하는 airflow executor는 sequential executor이다. 말 그대로, sequential(순차적)으로 작업을 처리할 때 사용가능하다. 다만, 실제 production 환경에서는 작업을 병렬로 처리해야 한다. 하나의 일이 끝나고 순차적으로 뒤의 task를 처리하는 sequential executor는 실제로 많은 일을 빠른 시간에 처리해야 환경에서는 실용적이지 못하다.

즉, task의 병렬처리를 위해 확장성 있으며 여러 세션으로 동시접근하여 read, write가 가능한 db가 필요하다. 일반적으로 상용화된 db는 동시제어와 접근을 가능하게 하며, 이에 대한 리소스 관리(connection pool)를 설정할 수 있다.
** Airflow는 수행할 작업 계획, 작업 결과 (successs or fail), 작업 이력 등을 모두 db에 쓰고 읽음으로써 관리한다. 

5. airflow.cfg에 메타스토어에 대한 SqlAlchemy 연결 문자 지정

Configuration Reference

  • SQLAlchemy 1.4.0 이상에서는 postgres://를 사용하면 오류가 발생하므로 sql_alchemy_conn에서 문자열로 postgresql://을 사용

  • SQLAlchemy를 사용할 때 psycopg2 드라이버를 사용하고, SQLAlchemy 연결 문자열에서 명시하는 것을 권장

  • database.sql_alchemy_conn는 메타데이터 데이터베이스에 대한 SqlAlchemy 연결 문자열

airflow.cfg
[database]
sql_alchemy_conn = postgresql+psycopg2://<user>:<password>@<host>/<db>
