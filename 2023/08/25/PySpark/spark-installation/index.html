[PySpark] Installation

Publish Date:   2023-08-25
PySpark

Ubuntu에서 단일 노드(Single Node) 스파크 배포

  1. 패키지 색인 파일(패키지 저장소가 제공하는 패키지에 대한 정보 파일) 업데이트
# Debian 계열 리눅스(Ubuntu)
$ sudo apt-get update

# RedHat 계열 리눅스(CentOS, LHEL)
$ sudo update yum
  1. 자바(openjdk 설치) 및 환경변수 등록
# Debian 계열 리눅스(Ubuntu)
$ sudo apt-get install openjdk-8-jdk

# RedHat 계열 리눅스(CentOS, LHEL)
$ sudo yum install java-1.8.0-openjdk
# .bashrc에 환경변수 등록
export JAVA_HOME=/usr/lib/jvm/java-8-openjdk-amd64
export PATH=$PATH:/usr/lib/jvm/java-8-openjdk-amd64/bin
  1. 스파크 설치 및 환경변수 등록

Apache Spark Installation

‘without Hadoop’ 빌드

  • 하둡을 사용하지 않고 독립실행형 모드로 설치할 때 사용

  • ‘without Hadoop’ 또는 ‘user-provided Hadoop’ 또는 ‘spark<x.y.z>-bin-without-hadoop.tgz’ 옵션 다운로드

  • 하둡에서 구현되는 필수 클래스 중 상당수가 시스템에 존재

# 다운받은 압축파일 풀기
$ tar -zxvf spark-<x.y.z>-bin-hadoop3.tgz -C <경로>

# 시스템 환경변수 파일 열기
$ vi ~/.bashrc

$ cat << EOF >> ~/.bashrc
# SPARK_HOME 등록
SPARK_HOME=<경로>/spark-<x.y.z>-bin-hadoop3

# PATH 추가
export PATH = $PATH:$SPARK_HOME/bin
export PATH = $PATH:$SPARK_HOME/sbin
EOF

# 시스템 환경변수 활성화
$ source ~/.bashrc

# 설치 테스트
$ spark-submit --class org.apache.spark.examples.SparkPi \
    -- master local \
    $SPARK_HOME/examples/jars/spark-examples*.jar 1000

다중 노드(Multi Node) 스파크 클러스터 배포

  1. 클러스터의 통신을 위한 hosts 등록
# hosts 파일 열기
$ cat << EOF >> /ect/hosts
<Master IP>    <Master Hostname> # spark-mater
<Master1 IP>   <Worker1 Hostname> # spark-worker1
<Master2 IP>   <Worker2 Hostname> # spark-worker2
<Worker3 IP>   <Worker3 Hostname> # spark-worker3
EOF

# 네트워크 연결 확인
ping spark-worker1
  1. 각 호스트에 spark-defaults.conf 등록
# spark-defaults.conf 파일 구성 
$ cd $SPARK_HOME/conf/
$ sudo cp spark-defaults.conf.template spark-defaults.conf
spark-defaults.conf
# 1. Spark Cluster Manager : Standalone Mode
spark.master    spark://<Master Hostname>:<Port>

# 2. Spark Cluster Manager : YARN
spark.master    yarn 

# Spark Event Log Use? 
spark.eventLog.enabled    true

# Spark Event Log Directory
spark.eventLog.dir    <Spark Log Directory>

# Spark Driver Memory 
spark.driver.memory    <N>g

# Spark Executor Memory
spark.executor.memory    <N>g

# Spark Executor Cores
spark.executor.cores    <N>
  1. 각 호스트에 spark-env.sh 환경변수 등록
# spark-defaults.conf 파일 구성 
$ cd $SPARK_HOME/conf/
$ sudo cp spark-env.sh.template spark-env.sh
spark-env.sh
# SPARK_MASTER_HOST, SPARK_MASTER_PORT
export SPARK_MASTER_HOST=<Master Hostname>
export SPARK_MASTER_PORT=7077

# HADOOP_CONF_DIR 
export HADOOP_CONF_DIR=<Hadoop Config Directory>

# SPARK_MASTER_WEBUI_PORT
export SPARK_MASTER_WEBUI_PORT=8080

# PYSPARK_PYTHON
export PYSPARK_PYTHON=<Python Binaray File>

Stand-alone 스파크 클러스터 배포

SPARK_VERSION=3.4.1

# 스파크 다운로드
$ wget https://dlcdn.apache.org/spark/spark-${SPARK_VERSION}/spark-spark-${SPARK_VERSION}-bin-hadoop3.tgz

# 압축해제
$ tar -xvf spark-${S\PARK_VERSION}-bin-hadoop3.tgz

# 압축 파일 이름 변경
$ mv spark-${SPARK_VERSION}-bin-hadoop3 spark

# 환경변수 설정 (.bashrc)

$ cat <<EOF >> ~/.bashrc

SPARK_HOME=~/spark

echo PATH=${PATH}:${SPARK_HOME}:sbin/:${SPARK_HOME}/bin >> ~/.bashrc
EOF

# 확인
$ source ~/.bashrc
$ printenv | grep spark

스파크 Standalone Environment Variable 설정

# ${SPARK_HOME}/conf/spark-env.sh.template 템플릿 이름 복사
$ cp spark-env.sh.template spark-env.sh

# ${SPARK_HOME}/conf/spark-env.sh 수정

# 설정 가능 변수에 대해서는 아래에서 확인
cat <<EOF >> spark-env.sh
export SPARK_WORKER_CORES=3
export SPARK_WORKER_MEMORY=3g
EOF

Environment Variable 설명

Environment Variable Description default value
SPARK_MASTER_HOST Bind the master to a specific hostname or IP address, for example a public one.
SPARK_MASTER_PORT Start the master on a different port 7077
SPARK_MASTER_WEBUI_PORT Port for the master web UI 8080
SPARK_WORKER_CORES Total number of cores to allow Spark applications to use on the machine
SPARK_WORKER_MEMORY Total amount of memory to allow Spark applications to use on the machine, e.g. 1000m, 2g 1 GB
  • Standalone Mode 추가 환경 변수 설명 참조

스파크 Standalone Master, Worker process 시작

Master 프로세스 시작/중지

$ ./${SPARK_HOME}/sbin/start-master.sh
$ ./${SPARK_HOME}/sbin/stop-master.sh

Worker 프로세스 시작/중지

master-spark-URL은 master를 실행한 서버의 web UI(http://localhost:8080)에서 확인할 수 있습니다. 보통 8080 포트를 웹 프로세스들이 많이 쓰기 때문에 다른 프로세스가 이미 점유하고 있다면, 위의 spark.env.sh에서 SPARK_MASTER_WEBUI_PORT를 다른 포트로 정의해주세요.

$ ./${SPARK_HOME}/sbin/start-workers.sh <master-spark-URL>
$ ./${SPARK_HOME}/sbin/stop-workers.sh

Spark App에서 SparkSession 생성시 master 설정

# master: SPARK_MASTER_HOST:SPARK_MASTER_PORT
spark = SparkSession.builder \
  .master(f"{master}") \ 
  .appName("Spark Application") \
  .config("spark.driver.memory", "10g") \
  .config("spark.executor.memory", "4g") \
  .config("spark.driver.maxResultSize", "1g") \
  .config("spark.executor.cores",  1) \
  .getOrCreate()

Process별 Spark Web UI

| Master Web UI (SPARK_MASTER_IP:SPARK_MASTER_WEBUI_PORT) | SPARK_MASTER_HOST:7077 
| Worker Web UI (SPARK_MASTER_IP:SPARK_WORKER_WEBUI_PORT) | SPARK_WORKER_HOST1:8081, SPARK_WORKER_HOST2:8081, SPARK_WORKER_HOST3:8081 .. 
| Application Web UI(Driver) | ClinetNode:4040 (i.e. localhost:4040)
