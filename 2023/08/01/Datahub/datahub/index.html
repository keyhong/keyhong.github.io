Datahub 정의

DataHub는 메타데이터 관리, 데이터 검색 및 데이터 거버넌스를 간소화하도록 설계된 최신 데이터 카탈로그입니다. 이를 통해 사용자는 데이터를 효율적으로 탐색 및 이해하고, 데이터 계보를 추적하고, 데이터 세트를 프로파일링하고, 데이터 계약을 설정할 수 있습니다. 이 확장 가능한 메타데이터 관리 플랫폼은 개발자가 빠르게 진화하는 데이터 생태계의 복잡성을 관리하고 데이터 실무자가 조직 내에서 데이터의 총 가치를 활용할 수 있도록 구축되었습니다.

Quickstart

Prerequisites

Python 3.8+ 설치 및 구성되어 있는지 확인하세요. (Check using python3 –version).

DOCKER RESOURCE ALLOCATION

Docker 엔진에 충분한 하드웨어 리소스를 할당하십시오.
테스트 및 확인된 구성: CPU 2개, RAM 8GB, 스왑 영역 2GB, 디스크 공간 10GB.

Install the DataHub CLI

poetry add acryl-datahub
poetry shell
datahub version

Start & Stop DataHub

# Start DataHub
datahub docker quickstart

# Stop DataHub
datahub docker quickstart --stop

Open Web UI

http://localhost:9002

username: datahub
password: datahub

Sample data Ingestion

datahub docker ingest-sample-data

Reset Datahub

To cleanse DataHub of all of its state (e.g. before ingesting your own), you can use the CLI nuke command.

datahub docker nuke

Customize installation

datahub docker quickstart --quickstart-compose-file <path to compose file>

Back up DataHub

빠른 시작 이미지는 프로덕션 인스턴스로 사용하지 않는 것이 좋습니다.
그러나 현재 빠른 시작 상태를 백업하려는 경우(예: 곧 회사에 대한 데모가 있고 나중에 복원할 수 있도록 빠른 시작 데이터의 복사본을 만들고 싶은 경우) 다음을 수행할 수 있습니다. 빠른 시작에 --backup 플래그를 제공합니다 

# Back up (default)
datahub docker quickstart --backup

# Back up to custom directory
datahub docker quickstart --backup --backup-file <path to backup file>

Restore DataHub

Note: 맥 M1/M2

Apple Silicon(M1, M2 등)이 탑재된 Mac 컴퓨터에서는 다음과 같은 오류가 나타날 수 있습니다 no matching manifest for linux/arm64/v8 in the manifest list entries. 이는 일반적으로 datahub cli가 Apple Silicon에서 실행 중인지 감지할 수 없음을 의미합니다. 이 문제를 해결하려면 다음을 실행하여 기본 아키텍처 감지를 재정의하세요.

datahub docker quickstart --arch m1

프로세스(컴포넌트) 생성 순서

elasticsearch >> elasticsearch-setup >> datahub-upgrade >> datahub-gms >> [ datahub-actions, datahub-frontend-react ]
zookeeper >> broker >> kafka-setup >> datahub-upgrade >> datahub-gms >> [ datahub-actions, datahub-frontend-react ]
mysql >> mysql-setup >> datahub-upgrade >> datahub-gms >> [ datahub-actions, datahub-frontend-react ]

Docker Service Name Description
datahub-upgrade DataHub 플랫폼의 업그레이드와 마이그레이션을 관리
datahub-gms DataHub의 메타데이터를 저장하고 관리하는 백엔드 서비스
datahub-actions 특정 이벤트 발생 시 자동화된 작업을 수행
datahub-frontend-react DataHub의 웹 프론트엔드로, React 기반으로 구현

프로세스 설명

  1. DataHub Actions
    DataHub Actions는 DataHub에서 특정 이벤트가 발생할 때 자동화된 작업을 수행하는 기능입니다.
Main Role Description
이벤트 트리거 DataHub 내에서 데이터셋 생성, 수정, 삭제 등의 이벤트 발생 시 트리거됩니다.
작업 정의 및 실행 이벤트에 반응하여 자동으로 수행될 작업을 정의하고 실행합니다. 예를 들어, 데이터셋이 수정될 때 Slack 알림을 보내거나 데이터 품질 검사를 실행할 수 있습니다.
통합성 외부 시스템과 쉽게 통합하여 다양한 자동화 작업을 수행할 수 있습니다. 예를 들어, Jira 티켓 생성, 이메일 알림, Slack 메시지 전송 등이 가능합니다.
모니터링 및 로깅 실행된 작업을 모니터링하고 로깅하여 문제 발생 시 해결할 수 있도록 지원합니다.
  1. DataHub Upgrade
    DataHub Upgrade는 DataHub 플랫폼의 업그레이드 및 마이그레이션을 관리하는 도구입니다.
Main Role Description
데이터베이스 스키마 마이그레이션 DataHub의 메타데이터 저장소(DB)의 스키마를 최신 버전으로 마이그레이션합니다.
버전 관리 DataHub의 다양한 컴포넌트의 버전을 관리하고, 업그레이드 시 호환성을 보장합니다.
자동화된 업그레이드 프로세스 업그레이드 과정이 자동화되어, 사용자가 최소한의 개입으로 DataHub를 최신 버전으로 유지할 수 있습니다.
  1. DataHub Frontend React
    DataHub Frontend React는 DataHub의 웹 프론트엔드를 구현한 React 기반 애플리케이션입니다.
Main Role Description
유저 인터페이스 사용자에게 직관적인 UI를 제공하여 데이터셋 검색, 탐색, 필터링 등을 용이하게 합니다.
데이터 시각화 데이터 계보, 메타데이터 정보 등을 시각화하여 사용자에게 제공합니다.
사용자 정의 가능성 React 기반으로 쉽게 커스터마이징 및 확장이 가능합니다.
API 통신 DataHub의 백엔드(GMS)와 통신하여 실시간으로 데이터를 가져오고 표시합니다.
  1. DataHub GMS (Graph Metadata Service)
    DataHub GMS는 DataHub의 백엔드 서비스로, 메타데이터를 저장하고 관리하는 핵심 컴포넌트입니다.
Main Role Description
메타데이터 저장소 다양한 데이터 소스의 메타데이터를 저장하고 관리합니다.
API 제공 메타데이터 검색, 업데이트, 삭제 등을 위한 API를 제공합니다.
데이터 계보 관리 데이터셋 간의 관계와 의존성을 추적하여 데이터 계보를 관리합니다.
확장성 확장 가능한 아키텍처를 통해 대규모 메타데이터를 효과적으로 관리할 수 있습니다.
