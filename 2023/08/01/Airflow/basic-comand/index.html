자주 사용하는 CLI 정리

Publish Date:   2023-08-01
Airflow

글을 쓰는 이유

Airflow는 파이썬의 Web Framework인 Flask로 만든 편리한 Web Server를 컴포넌트로 제공하고 있다. Web으로도 충분한 활용이 가능하지만, GUI가 Airflow API를 CLI로 호출하는 명령어와 매핑한 UI라는 걸 알면 플랫폼을 바라보는 관점이 넓어진다.

또 Web Server를 못 사용할 때 CLI를 사용할 수 있다는 건 매우 강력한 장점이다. 그래서 자주 썼었던 일부 API를 정리해보았다. API에는 도큐먼트의 링크를 같이 걸어 났는 데, 아마 Airflow를 써봤다면 익숙한 커맨드들도 많이 볼 수 있을 것 같다.

중요한 건 모든 API를 외우는 게 아니다. API를 외우지 못하더라도 필요할 때마다 --help 커맨드를 쓰거나 도큐먼트를 보는 법을 알면 언제든 찾아서 쓸 수 있다. 그렇기 때문에 API 자체보다도 도큐먼트를 중심으로 이해하는 연습을 하면 좋다.

dags

airflow dags list

모든 DAG 목록 조회

  • Web UI의 메인화면에 나타나는 모든 dag의 리스트와 같다.
airflow dags list
# List all the DAGs 
$ airflow dags list [-h] [-o table, json, yaml, plain] [-S SUBDIR] [-v]

airflow dags delete

DAG의 모든 메타데이터를 제거

  • Web UI의 메인화면에서 dag를 선택해서 지우는 기능과 같다. 메타스토어에서 해당 dag와 관련된 모든 데이터가 제거된다. DAG파일(.py)을 삭제되지 않다.
airflow dags delete
# Delete all DB records related to the specified DAG
$ airflow dags delete [-h] [-v] [-y] dag_id
    - y, --yes : Do not prompt to confirm. Use with care! (Default: False)

tasks

airflow tasks list

Task 목록 조회

  • 특정 dag에 정의된 task 리스트를 확인할 때 사용한다..
airflow tasks list
# List the tasks within a DAG
$ airflow tasks list [-h] [-S SUBDIR] [-t] [-v] dag_id

airflow tasks test

Task 테스트

  • 현재 스케줄링에 영향을 주지않고 실행 날짜(execution)를 같이 넣어서 단일 task를 테스트 할 수 있다. 아주 중요한 커맨드이다.
airflow tasks test
# Test a task instance. This will run a task without checking for dependencies or recording its state in the database
$ airflow tasks test [-h] [-n] [--env-vars ENV_VARS] [--map-index MAP_INDEX]
                     [-m] [-S SUBDIR] [-t TASK_PARAMS] [-v]
                     dag_id task_id [execution_date_or_run_id]

airflow tasks clear

특정 DAG의 Task 실행 기록 초기화

  • Web UI에서 실행중인 dag의 task를 버튼으로 팝업창에서 멈추능 기능과 같다. 해당 커맨드가 그 역할을 한다.
airflow tasks clear
# lear a set of task instance, as if they never ran
$ airflow tasks clear [-h] [-R] [-d] [-e END_DATE] [-X] [-x] [-f] [-r]
                      [-s START_DATE] [-S SUBDIR] [-t TASK_REGEX] [-u] [-v] [-y]
                      dag_id
    -d, --downstream : Include downstream tasks (Default: False)
    -u, --upstream : Include upstream tasks (Default: False)
    -f, --only-failed : Only failed jobs (Default: False)
    -r, --only-running : Only running jobs (Default: False)

airflow tasks pause/unpause

실행중인 DAG 중지 & 중지 해제

  • 실행중인 dag를 일시 중지 시킬 수 있다.
airflow dags pause
# Pause a DAG
$ airflow dags pause [-h] [-S SUBDIR] [-v] dag_id
airflow dags unpause
# Resume a paused DAG
$ airflow dags unpause [-h] [-S SUBDIR] [-v] dag_id

db

airflow db init

메타데이터 DB 초기화

  • airflow를 설치하고 airflow.cfg을 읽어 처음에 메타데이터 데이터베이스를 설정하고 초기화 시켜주는 커맨드이다.
airflow db init
$ airflow db init [-h] [-v]

airflow db upgrade

메타데이터 DB 업데이트

  • dag를 올리고 web에 바로 반영이 안될 때 제가 사용하는 커맨드이다. DAG폴더와 현재 메타데이터를 바로 동기화 시켜주는 역할을 한다.
airflow db upgrade
$ airflow db upgrade [-h] [--from-revision FROM_REVISION]
                     [--from-version FROM_VERSION] [-s] [-r TO_REVISION]
                     [-n TO_VERSION] [-v]

airflow db reset

메타데이터 DB 소각

  • 전체 메타데이터를 소각시키는 커맨드이다. 프로덕션에서는 절대 쓰면 안된다. 해당 명령어에 대한 주의를 인지해야 해서 넣었다.
airflow db reset
# Burn down and rebuild the metadata database
$ airflow db reset [-h] [-s] [-v] [-y]

users

airflow users create

Airflow 계정 생성

  • airflow 계정을 만드는 커맨드이다. airflow도 AWS의 IAM이나 Linux의 Permission을 부여하는 것처럼 다양한 계정의 권한을 설정할 수 있다.
airflow users create
# Create a user
$ airflow users create [-h] -e EMAIL -f FIRSTNAME -l LASTNAME [-p PASSWORD] -r
                       ROLE [--use-random-password] -u USERNAME [-v]
    -e, --email : Email of the user
    -f, --firstname : First name of the user
    -l, --lastname : Last name of the user
    -p, --password : Password of the user, required to create a user without –use-random-password
    -r, --role : Role of the user. Existing roles include Admin, User, Op, Viewer, and Public

airflow users list

  • Airflow 계정 조회

  • 만들어진 계정들을 탐색하는 커맨드이다.

# List users
$ airflow users list [-h] [-o table, json, yaml, plain] [-v]

webserver

airflow webserver

  • 웹서버 시작하기

웹서버 프로세스를 시작하는 커맨드이다. -D 옵션을 주면 nohup ~ $ 을 쓰지 않아도 백그라운드로 실행시킬 수 있다.

airflow webserver
# Start a Airflow webserver instance
$ airflow webserver [-h] [-A ACCESS_LOGFILE] [-L ACCESS_LOGFORMAT] [-D] [-d]
                    [-E ERROR_LOGFILE] [-H HOSTNAME] [-l LOG_FILE] [--pid [PID]]
                    [-p PORT] [--ssl-cert SSL_CERT] [--ssl-key SSL_KEY]
                    [--stderr STDERR] [--stdout STDOUT] [-t WORKER_TIMEOUT]
                    [-k {sync,eventlet,gevent,tornado}] [-w WORKERS]
    -D, --daemon :  Daemonize instead of running in the foreground (Default: False)

scheduler

airflow scheduler

스케줄러 프로세스를 시작하는 커맨드이다. -D 옵션을 주면 nohup ~ $ 을 쓰지 않아도 백그라운드로 실행시킬 수 있다.

  • 스케줄러 시작하기
airflow scheduler
# Start a scheduler instance
$ airflow scheduler [-h] [-D] [-p] [-l LOG_FILE] [-n NUM_RUNS] [--pid [PID]]
                    [-s] [--stderr STDERR] [--stdout STDOUT] [-S SUBDIR] [-v]
    -D, --daemon :  Daemonize instead of running in the foreground
