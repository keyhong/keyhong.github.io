Kubernetes Executor Installation

Publish Date:   2023-08-01
Airflow

Kubernetes Installtion

먼저 VirtaulBox와 Vagrant를 설치합니다.

그 이후 저는 아래 글을 참고하였습니다. 저의 경우 window10을 사용하고 있습니다.

https://kubetm.github.io/k8s/02-beginner/cluster-install-case6/

mkdir k8s && cd k8s

# Vagrant 스크립트 다운로드
curl -O https://kubetm.github.io/yamls/k8s-install/Vagrantfile

# Rocky Linux Repo 세팅
curl -O https://raw.githubusercontent.com/k8s-1pro/install/main/ground/k8s-1.27/vagrant-2.3.4/rockylinux-repo.json
vagrant box add rockylinux-repo.json

# Vagrant Disk 설정 Plugin 설치 
vagrant plugin install vagrant-vbguest vagrant-disksize

설치 시간이 길어져 잠깐 나갔다 왔더니, 저의 경우 설치 도중에 SSH Connection이 끊기는 문제가 생겼습니다. 그래서 Vagrantfile 파일에 config.ssh.keep_alive = true을 추가하여 다시 실행하였습니다.

# vagrant up으로 머신 생성
vagrant up --no-provision --destroy-on-error

만약 저와 같이 중간에 설치하다 문제가 생긴 경우, destroy로 machine을 지운 후 다시 vagrant up으로 올려주시면 됩니다. 꼭 정상 설치가 되야 하는 이유는 중간 설치가 끊기더라도 machine을 살아있지만 k8s 패키지들이 설치되지 않아 사용할 수 없기 때문입니다.

# vagrant machine 상태 조회
vagrant status

# vagrant destory NAME
vagrant destroy k8s-master

# 다시 vagrant up으로 머신 생성
vagrant up

정상적으로 3개의 머신을 설치 후 kubeadmin으로 master 노드의 쿠버네티스 클러스터에 join할 token을 워커 노드에 등록해서 클러스터에 조인하게 해야 합니다.

Airflow Helm Repo Installation
# helm repository 등록
$ helm repo add apache-airflow https://airflow.apache.org

# helm repository 목록 조회
# $ helm repo list 

# helm repository 삭제
# $ helm repo remove 

# helm chart에서 repository 검색
# $ helm search repo airflow 

# airflow 네임스페이스를 만들고, repo upgrade & install(실행 커맨드) 
$ helm upgrade --install airflow apache-airflow/airflow --namespace airflow --create-namespace --debug --timeout 10m0s

# helm release uninstall (차트의 모든 리소스 제거)
# helm uninstall airflow --namespace airflow

좋은 컴퓨터라면 여기까지 했을 때 쿠버네티스 위에서 celery executor를 사용할 수 있습니다. 여기서 kubernetes executor를 사용하려면 차트의 value.yaml를 설정하여야 합니다. 저는 노트북 리소스가 부족해서.. deployment가 생성되다 중지됩니다. 아무래도, helm chart의 default executor가 celery인데 flower, redis, postgres, webserver, scheduler, worker 등 기본 프로세스 + 쿠버네티스 service, cronjob, secret 등 함께 올라가서 그런 것 같습니다. 저는 제 컴퓨터 리소스에 맞게 다른 방법을 찾아야 할 것 같습니다.

뛰어지는 pod는 airflow 헬름 차트의 템플릿에서 확인 할 수 있습니다. 차후 value.yaml을 kubernetes executor로 변경하면 다시 글을 보완하러 와야 할 것 같습니다.
