YARN 구조 파악하기

Publish Date:   2023-10-01
Hadoop

YARN (Yet Another Resource Negotiator)

하둡의 클러스터 자원 관리 시스템

  • Hadoop 2.X 부터 도입되었다.

  • MapReduce 성능을 높이기 위한 시스템이다.

  • 다른 분산 컴퓨팅 도구(예를 들면 spark)도 지원을 한다.

  • 사용자는 YARN이 내장된 분산 컴퓨팅 프레임워크에서 고수준 API를 작성하면 된다. 사용자가 자원 관리의 자세한 내용은 알 수 없다.

전체적인 구조

클러스터 저장 계층

HDFS와 HBase

클러스터 계산 계층

YARN

애플리케이션 계층

MapReduce, Spark, Pig, Hive 등과 같은 분산 컴퓨팅 프레임워크. 사용자는 AM을 통해서 이용할 뿐 YARN에 직접적으로 접근하지 않는다.

YARN 애플리케이션 수행 해부

YARN은 리소스 매니저와 노드 매니저 등 두 가지 유형의 장기 실행 데몬 프로세스를 통해 핵심 서비스를 제공한다. YARN 자체는 client-master 프로세스 간에 서로 통신 기능을 지원하지 않는다. 상태 변경 전달과 결과 전송은 하둡 RPC와 같은 원격 호출 방식을 이용한다.

리소스 매니저 (RM, Resource Manager)

클러스터 전체 자원의 사용량을 관리한다.

노드 매니저 (NM, Node Manager)

  • 클러스터의 모든 머신 머신에서 실행한다.

  • 컨테이너를 구동하고 모니터링하는 역학을 수행한다.

  • 특정 애플리케이션의 프로세스(task)를 컨테이너에서 실행시킨다.

컨테이너

  • CPU, 메모리 등의 사용 한도를 가진 자원

  • YARN의 설정 방법에 따라 다르나, 일반적으로 Unix 프로세스나 리눅스의 cgroup을 이용한다.

YARN 애플리케이션 구동 방식

1. 클라이언트가 YARN에서 애플리케이션 구동을 위해 RM에게 접속

Application Maste 프로세스 (이하 AM) 구동 요청.

2. RM은 컨테이너에서 AM을 시작할 수 있는 노드 매니저 하나 탐색

  • 클러스터 외부나 클라이언트의 JVM에서 해당 클라이언트의 애플리케이션 마스터를 구동하는 것도 가능하다. (비관리 애플리케이션 마스터로, 예를 들면 스파크 클러스터의 deploy mode 중 client mode 실행)

  • AM 프로세스가 딱 한번 실행될지는 애플리케이션에 따라 상이하다.

3. AM이 단순한 계산을 단일 컨테이너에서 수행 후 아래 두 가지 행동 중 하나를 수행

  • 클라이언트에게 결과를 반환 후 종료

  • 리소스 매니저에 더 많은 컨테이너를 요청 후 분산 처리 수를 증대
