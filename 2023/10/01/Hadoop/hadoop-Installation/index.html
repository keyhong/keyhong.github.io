Hadoop 설치시 유용 유틸리티

운영시스템의 조건

Red Hat Linux, CentOS, SUSE Linux Enterprise Server 11, SP1 등 다양한 리눅스에서 설치 가능. 다만, 운영체제는 반드시 64비트 버전을 사용하여야 한다.

유용한 유틸리티 추천

yum (Yellowdog Updater, Modified)

  • Red Hat Linux RPM(Red Hat Package Manager) 소프트웨어 패키지를 가져와 설정, 삭제, 검색, 관리 할 수 있게 해주는 tool

  • Ubuntu Linux에서는 apt-get 유맅리티를 사용한다.

rpm (Red hat Package Manager)

  • Red Hat, SUSE, Fedora 리눅스 시스템의 패키지 관리 tool

  • yum은 최종적으로 필요한 의존 패키지까지 모두 자동으로 설치해주는 반면, rpm 단일 패키지만 설치해준다. 따라서 dependency 문제가 발생할 수 있으므로 yum 또는 apt-get을 사용 가능하다면 우선 사용하는 것이 좋다.

scp (Secure CoPy)

  • FTP 절이나 원거리 시스템에 접속하지 않고도 호스트 사이에서 안전하게 파일과 디렉터리를 복사할 수 있는 유틸리티
  • scp는 데이터 전송에 SSH(Secure Shell)을 사용하기 때문에 인증을 위한 authorized_keys나 패스워드가 필요하다. ssh는 보통 22번 포트를 사용한다.

curl (Client URL)

  • URL(Uniform Resource Locator)을 이용해 파일을 보내거나 받을 수 있는 유틸리티
    ** URL : 인터넷에서 웹 페이지, 이미지, 비디오 등 리소스의 위치를 가리키는 문자열

wget (Web GET)

  • 네트워크 유틸리티로 HTTP나 FTP 프로토콜을 사용해 파일을 다운로드 받을 수 있는 유틸리티

pdsh (Parallel Distributed SHell)

  • 병렬화된 원격 셸. 명령을 원격으로 실행할 수 있다.

Hadoop 설치 모드

독립형(standalone)

  • 모든 하둡 서비스는 하나의 JVM에서 동작
  • YARN이나 HDFS 같은 백그라운드에서 동작하는 데몬(daemon)은 없음
  • 데이터를 저장하는 데 HDFS가 아닌 로컬 파일 시스템을 사용
  • MapReduce 잡은 단일 mapper와 단일 reducer를 이용
  • 자신의 코드를 실행하는 개발자에 적합

가상 분산형(Pseudo-distributed)

  • 실제 멀티 노드 클러스터의 시뮬레이션. 실제 분산 클러스터처럼 모든 서비스를 설정 가능
  • 모든 데몬(네임노드, 데이터 노드, 리소스 매니저)은 단일 노드에서 동작
  • 실제 프로덕션에서 데이터 복제(replication)와 고가용성(fault tolerance) 설정을 할 수 없기 때문에 사용 불가
  • 단일 노드이기 때문에 병렬 프로세싱의 이점을 누릴 수 없음

완전 분산형(Fully distributed)

  • 실제 프로덕션에서 사용하는 최종 목적의 형태로, 여러 서버에 걸쳐 하둡이 설치된 형태
  • 데이터 복제 설정을 할 수 있고, 원한다면 고가용성 설정도 가능
  • 각각의 프로세스들을 위한 데몬이 있고, 데몬은 다중 서버를 통해 운영

Hadoop Installation with HDP

  1. Virtual Box (VM) 다운로드
    Virtual Box

  2. Hortonworks HDP 2.6.5 버전 다운로드
    Hortonworks HDP 2.6.5

  3. 하둡3 설치 및 환경변수 등록
    Apache Hadoop Installation

# 다운받은 압축파일 풀기
$ tar -zxvf hadoop-<x.y.z>.tar.gz -C <경로>

$ vi ~/.bashrc
# HADOOP_HOME 등록
export HADOOP_HOME=<경로>/hadoop-<x.y.z>
export PATH=$PATH:$HADOOP_HOME/bin 
export PATH=$PATH:$HADOOP_HOME/sbin

export HDFS_NAMENODE_USER="hdfs"
export HDFS_DATANODE_USER="hdfs"
export YARN_RESOURCEMANAGER_USER="yarn"
export YARN_NODEMANAGER_USER="yarn"


export HADOOP_MAPRED_HOME=$HADOOP_HOME 
export YARN_HOME=$HADOOP_HOME 
export HADOOP_CONF_DIR=$HADOOP_HOME/etc/hadoop 
export HADOOP_COMMON_LIB_NATIVE_DIR=$HADOOP_HOME/lib/native 
export HADOOP_OPTS="-Djava.library.path=$HADOOP_HOME/lib/native" 
export HADOOP_STREAMING=$HADOOP_HOME/share/hadoop/tools/lib/hadoop-streaming-<x.y.z>.jar
export HADOOP_LOG_DIR=$HADOOP_HOME/logs 
export PDSH_RCMD_TYPE=ssh

Installation On Premise(Pseudo Distributed)

가상 분산 모드(Pseudo-distributed)

하나의 서버에 하둡을 완전분산모드 형태로 운영하는 모드

설치 환경

  • H/W : Virtual Machine 1
  • OS : Linux

설치순서

  1. 패키지 색인 파일(패키지 저장소가 제공하는 패키지에 대한 정보 파일) 업데이트
# Debian 계열 리눅스(Ubuntu)
$ sudo apt-get update

# RedHat 계열 리눅스(CentOS, RHEL)
$ sudo yum update
  1. 자바(openjdk 설치) 및 환경변수 등록
# Debian 계열 리눅스(Ubuntu)
$ sudo apt-get install openjdk-11-jdk

# RedHat 계열 리눅스(CentOS, RHEL)
$ sudo yum install -y java-11-openjdk-devel.x86_64
# .bashrc에 환경변수 등록
export JAVA_HOME=/usr/lib/jvm/java-11-openjdk-amd64
export PATH=$PATH:$JAVA_HOME/bin
  1. 하둡 설치 및 환경변수 등록

Apache Hadoop Installation

# 하둡 다운로드
$ sudo wget https://dlcdn.apache.org/hadoop/common/hadoop-<x.y.z>/hadoop-<x.y.z>.tar.gz

# 다운받은 압축파일 풀기
$ tar -zxvf hadoop-<x.y.z>.tar.gz -C <경로>

$ vi ~/.bashrc
# HADOOP_HOME 등록
export HADOOP_HOME=<경로>/hadoop-<x.y.z>
export PATH=$PATH:$HADOOP_HOME/bin:$PATH:$HADOOP_HOME/sbin
  1. 암호 없는 SSH 설정
$ ssh-keygen -t rsa -P '' -f ~/.ssh/id_rsa 
$ cat ~/.ssh/id_rsa.pub >> ~/.ssh/authorized_keys 
$ chmod 0600 ~/.ssh/authorized_keys
  1. hadoop-env.sh 설정
export JAVA_HOME=$JAVA_HOME
  1. core-site.xml 설정

core-site.xml Configuration

  • Namenode | 9000 IPC 프로토콜 | 파일시스템 메타데이터 작업관련용 |
<configuration>
    <property>
        <name>dfs.replication</name>
        <value>1</value>
        <description>기본 블록 복제. 파일을 만들 때 실제 지정하는 복제 수</description>
    </property>
</configuration>
  1. hdfs-site.xml 설정

hdfs-site.xml Configuration

  • Namenode | 9870 HTTP 프로토콜 | HDFS 파일시스템 조회, 탐색 등 Web UI
<configuration>
    <property>
        <name>dfs.replication</name>
        <value>1</value>    
        <description>기본 블록 복제. 파일을 만들 때 실제 지정하는 복제 수</description>
    </property>
    <property>
        <name>dfs.namenode.http-address</name>
        <value>0.0.0.0:5000</value>
        <description>dfs 네임노드 Web UI가 수신 대기할 주소 및 기본 포트</description>
    </property>    
</configuration>
  1. MapReduce 작업을 로컬에서 실행
$ hdfs namenode -format

$ start-dfs.sh
