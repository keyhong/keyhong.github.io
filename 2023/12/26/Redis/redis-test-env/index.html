Redis 실습환경 구성하기

Redis 실습환경을 구성하기 위한 방법으로 강의에서 2가지를 제시해주었다.  

  1. Redis Cloud에서 인스턴스 만들기

  2. 로컬 머신에서 Redis를 Install하고 Run하는 방법

 
강사분께서는 “1번 Redis Cloud에서 인스턴스 대여하기” 방법을 적극적으로 권유하셨고 나는 2번 방법으로 환경을 구축하였다. 이에 대한 구축 방법을 소개하고자 한다. 아참, 참고로 소개할 이 방법은 워낙 유명해서 연습용으로 많이 쓰이는 방법이라 한다.

Redis Cloud에서 인스턴스 대여하기

Redis가 운영하는 공식 홈페이지(https://redis.com/)에 들어가면 무료 Redis 인스턴스를 생성할 수 있다. Google이나 GitHub 계정이 있다면 빠른 로그인이 가능하다. 접속을 하면 보다시피 가장 왼쪽의 무료버전 인스턴스를 대여할 수 있다.

Redis 공식 홈페이지에 들어가면 무료 Redis 인스턴스를 생성할 수 있다. 클라우드 프로바이더 3사 AWS, GCP, Azure에서 무료 인스턴스를 대여해 자동으로 레디스를 쓸 수 있게 구축해주는 데, 혼자 연습하기에는 충분히 짭잘하다. Google이나 GitHub 계정이 있다면 빠른 로그인이 가능하다. 접속을 하면 보다시피 가장 왼쪽의 무료버전 인스턴스를 대여할 수 있다. 필요하신 분이라면 이미지의 빨간 네모칸을 따라 같이 만들어보길 바랍니다!

스크롤을 내리면 Cloud Provider를 정할 수 있다. 나는 aws와 region(서버를 빌릴 머신이 있는 가용영역)으로 tokyo를 선택했다. 콤보박스 안에서 리전을 선택할 수 있는 데 한국은 없어서 가장 가까운 일본을 선택했다.

다음은 메모리 설정 같은 데 Free를 이용하는 우리에게 선택은 없다.. 이어 쭉쭉 나가자.

초록색 체크가 들어오면 사용할 수 있는 준비가 된 것이다.

추가 설정 하기

데이터베이스에 데이터를 추가하고 쿼리도 날리려면 추가 설정을 해야한다. 일단 Redis에서 CLI를 사용하고 테스트하려면 테스트를 위한 웹이 필요하다. rbook cloud라는 웹에서 테스트를 할 수 있다. 여기서 부터는 redis에 접근할 수 있는 플랫폼이 있다면 본인의 플랫폼을 사용해도 된다. 필요하다면 connection을 연결하는 방법만 봐도 무방하다.
