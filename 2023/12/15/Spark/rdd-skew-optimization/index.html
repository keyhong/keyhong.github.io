[Spark] RDD Skewness Optimization

Publish Date:   2023-12-15
Spark

Optimizing Skewness

스파크에서 skewness(왜도, 비대칭도)를 해결하는 것은 스파크 튜닝의 중요한 요소이다. 왜 skewness를 해결해야 하는 지는 스파크가 분산 처리 프레임워크라는 것을 생각한다면 당연하게 받아 들일 수 있다. 우리가 소싱한 데이터는 파티션에 의해 여러 개로 수평 분할되어 클러스터의 여러 노드에 전달 된다.

각 노드에서 데이터를 처리(transformation, action)하는 데, 처리하는 과정 속에서 데이터를 특정 조건으로 filter 하거나 shuffle 연산(groupBy, reduceByKey ..)을 수행하면 처음에 동일 혹은 비슷하게 분배했었던 파티션별 소유 데이터의 양이 한 쪽으로 치우치는 현상을 만들어낸다. 이미지를 그려내보면 다음과 같다.

어떤 연산을 수행한 다음 특정 RDD의 파티션으로 데이터가 몰린 상황이 발생했다고 가정하자. Partition3는 적어지고 Partition1은 많아졌다. 만약 이 다음에 수행될 연산이 map 연산(narrow transformation)과 shuffle 연산(wide transformation)이 연이어 일어난다면 어떻게 될까? 각 RDD별로 map연산을 수행하고, 다음 shuffle 연산을 하기 직전까지 모든 파티션은 다음 shuffle 연산을 실행해도 되는 상황인지 확인하고 대기할 것이다.

문제는 여기서 발생한다. 각 RDD별로 처리해야 할 데이터 크기가 다르기 때문에 양이 적은 Partition3, Partition4, Partition2 순서대로 일찍 작업을 끝내고 Partition1이 작업을 끝내기 전까지 대기할 것이다. 이 때 대기하는 파티션은 프로세싱을 할 수 있는 리소스가 있음에도 불구하고 처리할 수 있는 데이터가 더 이상 없어 처리를 못하는 CPU 휴유시간(idle time)이 발생한다. 반면, 양이 많은 Partition1은 단일 코어(일반적으로 1 partition = 1 core = 1 task)에서 거대한 데이터를 처리해야 하므로 데 오랜 시간이 걸릴 것이다.

스파크의 목적은 빅데이터를 여러 노드를 이용하여 빠르게 분산 처리하기 위함에 있다. 병목현상은 스파크가 본래의 목표를 제대로 달성하지 못하게 한다. 결국 분산 처리를 하기 위해서는 가급적이면 모든 파티션이 최대한 동등한 데이터의 크기를 나눠갖게 하고, 비슷한 시점에 일이 끝날 수 있게 사용자가 최적화하는 작업을 해주어야 한다. 이를 Skewness를 Optimizing 한다고 표현한다.

Salting

조인키(join key)에 랜덤한 값을 추가하여 데이터를 균등하게 파티셔닝 할 수 있는 기술

Pyspark에서 Salting Code 구현

(1) Skew가 발생하도록 데이터를 불규칙적으로 생성

# Skew 데이터 불러오기
import numpy as np
import random

key_1 = ['a'] * 10
key_2 = ['b'] * 6_000_000 # skew가 발생하는 key
key_3 = ['c'] * 800
key_4 = ['d'] * 10_000

keys = key_1 + key_2 + key_3 + key_4
random.shuffle(keys) # shuffle

values_1 = list(np.random.randint(low=1, high=100, size=len(key_1)))
values_2 = list(np.random.randint(low=1, high=100, size=len(key_2)))
values_3 = list(np.random.randint(low=1, high=100, size=len(key_3)))
values_4 = list(np.random.randint(low=1, high=100, size=len(key_4)))

values = values_1 + values_2 + values_3 + values_4

pair_skew = list(zip(keys, values))

# RDD로 데이터 로드하기 (8개의 파티션)
rdd = sc.parallelize(pair_skew, 8)

(2) Salting을 위해 Key에 랜던함 숫자 문자열을 붙여줄 수 있는 함수 생성

기존에는 [(‘b’: 24), (‘b’: 13), (‘b’: 32), (‘b’: 41) …] 이런 형태이다. value는 많지만 key는 ‘a’, ‘b’, ‘c’, ‘d’ 4개 밖에 없기 때문에 groupby를 하면 데이터가 많은 ‘b’가 하나의 파티션으로 데이터가 쏠린다(skewness).** 이러한 쏠림을 방지하지 위해 salting은 기존에 있던 key에 랜덤한 숫자를 덧붙여주어 key를 변형시키고 분산시키는 과정이다.

위 예시를 들면, 'b' + '_' + 랜덤한 숫자 형태로 만들어준다. 그 결과 [('b_2': 24), ('b_4': 13), ('b_1': 32), ('b_5': 41) ...] 처럼 기존의 a, b, c, d 4개 보다 훨씬 많은 key가 만들어지게 된다. 이 후 groupby를 하면 key를 여러 파티션에 고루 분산되고 따라서 모든 파티션이 이전보다 균등한 작업량으로 병렬처리를 할 수 있게 된다.

import random

def salting(val):
    tmp = val + '_' + str(random.randint(0, 5)) # 랜덤 값의 범위는 시도해야 함
    return tmp
    
# salting method
rdd_salting = rdd.map(lambda x: (salting(x[0]), x[1]))

# groupbykey
grouped_rdd = rdd_salting.groupByKey().cache()

# skew 데이터에서 map()을 통해 간단한 data transformation 실행해보기
grouped_rdd.map(lambda pair: (pair[0], [(i+10) for i in pair[1]])).count()
