[Hive] Partitioning

Publish Date:   2023-12-14
Hive

Partitioning

분할 키를 기반으로 테이블의 데이터를 여러 부분으로 구성하는 방법

Partitioning의 두 종류는 정적 파티셔닝과 동적 파티셔닝이 있다.

  • Static Partitioning (정적 파티셔닝)
  • Dynamic Partitioning (동적 파티셔닝)

Static Partitioning

  • 유저가 파티션의 값을 확인해 직접 파티션명을 지정하여 파티션에 데이터를 수동으로 적재한다. 따라서 데이터에 대한 사전 탐색이 필요하다.

  • 정적 파티셔닝에서 파티션으로 사용하는 컬럼은 CREATE 문에 포함되서는 안된다.

  • INSERT가 아닌 LOAD로 정적 파티션 테이블에 적재하는 경우에 파티션별 분리된 파일을 각각 가지고 있어야 한다.

  • 수동으로 데이터를 넣어야 하는 불편함이 있지만, 동적 파티셔닝보다 빠르다.

정적 파티션 테이블 생성하기

-- deptname 컬럼으로 파티셔닝
 
CREATE TABLE IF NOT EXISTS part_dept ( -- 생성할 테이블: part_dept
    dept_no int,
    empname string,
    sal int
)
PARTITIONED BY (deptname string) -- 생성할 파티션 컬럼: deptname
ROW FORMAT DELIMITED
FIELDS TERMINATED BY ','
LINES TERMINATED BY '\n'
STORED AS TEXTFILE
;

정적 파티션에 INSERT로 데이터 적재하기

INSERT INTO TABLE part_dept
PARTITION (deptname='HR') -- 생성할 파티션 컬럼값이 'HR'이라고 명시적으로 지정
SELECT col1, col2, col3 -- 파티션에 넣을 데이터 (SELECT 문)
FROM dept
WHERE col2='HR' -- 파티션 생성 이유: 파티션별 데이터를 조건에 따라 나누어 적재하기 위함
;
/**
INSERT를 할 때, 파티션 컬럼 값만 명시적으로 지정해준다면
실제 넣는 데이터와 상관없이 어떤 파티션 컬럼값으로든 만들 수 있다.
*/
INSERT INTO TABLE part_dept
PARTITION (deptname='XZ')
SELECT col1, col2, col3
FROM dept
WHERE col2='Accounts'
;

정적 파티션에 LOAD로 데이터 적재하기

/** act data

10,Accounts,Stewe,2384
10,Accounts,Pico,3833
10,Accounts,Lui,3833
*/

LOAD DATA LOCAL INPATH '/home/hong/act' INTO TABLE part_dept PARTITION (deptname='XZ');

Dynamic Partitioning

  • Hive가 파티션 값을 기반으로 파티션을 자동으로 생성하고 데이터를 적재한다.

  • 기본적으로 다이나믹 파티셔닝은 비활성화 되어 있다. 사용하려면 이 속성을 활성화 해주어야 한다.

동적 파티셔닝 활성화

-- 동적 파티셔닝 활성화
SET hive.exec.dynamic.partition = true;

/**
default는 strict이다.
strict에서는 동적 파티션과 함께 적어도 하나의 정적 파티션을 지정해야 한다.

예로, PARTITIONDE BY (deptname, sal) 일 때, 두 컬럼 중 하나는 정적 파티션이어야 한다.
이는 실수에 의해 동적으로 생성되는 거대한 파티션 수를 방지하기 위함이다.
*/

SET hive.exec.dynamic.partition.mode = nonstrict;

동적 파티션 테이블 생성하기

CREATE문 자체는 정적 파티션 테이블 생성하는 것과 차이가 없다.

-- deptname 컬럼으로 파티셔닝
CREATE TABLE IF NOT EXISTS part_dept (
    dept_no int,
    empname string,
    sal int
)
PARTITIONED BY (deptname string)
ROW FORMAT DELIMITED
FIELDS TERMINATED BY ','
LINES TERMINATED BY '\n'
STORED AS TEXTFILE
;

동적 파티션 테이블에 INSERT로 데이터 적재하기

/**
INSERT를 할 때, 파티션 컬럼 값을 데이터에 의해 결정하므로 
파티션을 분별하는 데이터 값과 파티션 컬럼값이 일치한다.

예를 들면, col2 컬럼의 값의 도메인이 (value1, value2, value3)라면
파티션 컬럼값도 col2 컬럼 값과 같이 value1, value2, value3로 만들어진다.
*/
INSERT INTO TABLE part_dept1
PARTITION (deptname) -- 정적 파티셔닝과 달리 파티션 이름을 명시적으로 하드코딩 하지 않는다.
SELECT col1, col2, col3, col2 -- 파티셔닝 대상이 되는 컬럼(col2)이 제일 마지막에 있어야 한다. 
FROM dept
;

테이블 파티션 변경하기

파티션 리스트 보기 (SHOW PARTITIONS )
SHOW PARTITIONS part_dept1;

특정 파티션 삭제하기 (ALTER TABLE ~ DROP PARTITION ~)

  • ☆ Internal 테이블일 경우, 데이터와 메타데이터 모두 파티션과 함께 삭제된다.

  • ☆ External 테이블일 경우, 메타데이터는 사라져 Hive에서 검색은 되지 않아도 HDFS에서는 존재 하는 것을 확인 가능하다.

ALTER TABLE part_dept1 DROP PARTITION (deptname='HR');

파티션 추가하기 (ALTER TABLE ~ ADD PARTITION ~)

-- 파티션 생성
ALTER TABLE part_dept1 ADD PARTITION (deptname='Dev');

-- 데이터 적재하기
LOAD DATA LOCAL INPATH '/home/hong/dev' INTO TABLE part_dept1 PARTITION (deptname='Dev');

☆ Q. 만약 Hive로 파티션을 생성하지 않고, HDFS로 파티션 폴더를 먼저 생성하면 Hive가 알까?

모른다. Hive는 HDFS로 만든 파티션 폴더에 대한 메타 데이터를 가지고 있지 않기 때문이다.

-- hdfs 명령어로 폴더 신규 추가
hdfs dfs -mkdir /user/hive/warehouse/part_dept1/dept_name=Finance

-- Hive에서 SHOW PARTITIONS로 생성된 파티션 확인
SHOW PARTITIONS;

/** Return
(hdfs 명령어로 파티션 폴더를 만들었지만, Hive에서는 파티션이 생긴지 인식하지 못한다.)

partition
dept_name=Accounts
dept_name=Dev
dept_name=Marketing
dept_name=Sales
*/

메타 데이터를 추가하려면 2가지 방법이 있다.

  • ① 파티션을 수동으로 추가하는 방법
-- 파티션을 수동으로 추가하는 방법
ALTER TABLE part_dept1 ADD PARTITION (deptname='Finance');
  • ② MSCK REPAIR TABLE 명령어를 사용하는 방법
-- MSCK REPAIR TABLE 명령어를 사용하는 방법
MSCK REPAIR TABLE part_dept1;

/** Return

Repair: Added partition to metastore part dept1:deptname=Finance
*/
