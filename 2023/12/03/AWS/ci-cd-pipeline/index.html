[AWS] EC2 CI/CI Pipeline

Publish Date:   2023-12-03
AWS

CI / CD

애플리케이션 개발 단계를 자동화하고 짧은 주기로 고객에게 제공하는 개발 방법

  • 자주 빌드, 자주 배포하여 유저에게 빠르게 배포

CI(Continuous Integration)

  • 개발자들이 코드 레포지터리로 푸시하는 방식

  • 코드가 푸쉬되면 테스트와 빌드서버가 코드를 체크

  • 개발자들은 테스트와 체크에 대해 성공/실패 피드백

  • 버그를 빠르게 발견하고 고칠 수 있음

  • 지속적 테스트, 배포를 통해 코드를 빠르게 전달

  • Tools : GitHub, Bitbucket, CodeCommit

CD(Continuous Delivery or Deployment)

  • 필요할 때 소프트웨어가 안전하고 빠르게 배포되도록 함

  • Tools : CodeDeploy, Jenkins CD, Spinnaker

  • Continuous Delivery vs Continuous Deployment

    • Continuous Delivery : 승인단계를 거친 후 자동으로 배포
    • Continuous Deployment : 배포 전체 자동화

Technology Stack for CICD

Code > Build > Test > Deploy > Provision

  • Code

    • AWS CodeCommit
    • GitHub or 3rd code repository

  • Build, Test

    • AWS CodeBuild
    • Jenkins CI or 3rd party CI servers

  • Deploy, Provision

    • AWS Elastic Beanstalk

  • Deploy

    • AWS CodeDeploy

  • Provision

    • User Managed EC2 Instance Fleet (CloudFormation)

  • Orchestrate

    • AWS CodePipeline

AWS CI/CI Pipeline

AWS Code Service

AWS의 소스관리, 빌드, 테스트, 배포, 관리 등을 위한 서비스

  • 자동화된 CI/CD 파이프라인 구축

1. AWS CodeCommit

프라이빗 Git Repository를 호스팅하는 안전하고 확장성이 뛰어난 소스제어 서비스

  • AWS 버전의 Github
  • IAM Credential을 통해 접근(기존의 HTTP, SSH 프로토콜도 지원)
  • AWS의 다양한 서비스와 연동
  • 이벤트 버스를 통한 이벤트 기반 로직 처리 가능

2. AWS CodeBuild

소스코드 컴파일 단계부터 테스트 실행 후 소프트웨어 패키지를 개발하여 배포하는 단계까지 마칠 수 있는 완전 관리형 지속적 통합 서비스

  • CI/CD 중 CI(Continuous Integration)를 담당
  • 별도의 프로비전 불필요
  • 간단하게 빌드 세팅이 된 서버 하나를 빌려서 사용하는 개념

3. AWS CodeDeploy

EC2, Fargate, Lambda 및 온프레미스 서버와 같은 다양한 컴퓨팅 서비스에 대한 소프트웨어 배포를 자동화하는 완전 관리형 배포 서비스

  • CI/CD 중 CD(Continuous Delivery or Deploy)를 담당
  • 빌드/테스트된 제품을 배포해주는 서비스
  • 다양한 배포 모드(In-Place, Blue/Green) 등 지원
  • 배포 실패 시 자동으로 롤백 지원
  • 배포 상태를 모니터링 가능

4. AWS CodePipeline

빠르고 안정적인 애플리케이션 및 인프라 업데이트를 위해 release pipeline을 자동화하는 데 도움이 되는 완전 관리형 지속적 전달 서비스

  • 전체 프로세스 오케스트레이션
  • Serverless
  • 이벤트 기반 운영 가능
  • 다양한 AWS 서비스 연동 가능
  • 다양한 외부 서비스 연동 가능 (Jenkins 등)

5. AWS CodeStar

AWS에서 애플리케이션을 신속하게 개발, 빌드 및 배포 가능. AWS CodeStar는 통합된 사용자 인터페이스를 제공하여 한 곳에서 소프트웨어 개발 활동을 손쉽게 관리 가능

  • Code Services의 모든 서비스를 통합하여 프로젝트 생성
  • 여러 템플릿에서 프로젝트에 맞는 템플릿 생성 가능
  • IAM으로 권한 관리 가능

아티팩트

다음 스테이지로 전달할 CodeBuild에서 작업을 수행한 결과물

  • 빌드된 애플리케이션
  • 테스트 결과 내용

다음 스테이지로 아티팩트를 전달

BuildSpec.yml

CodeBuild에서 수행할 내용을 정의한 문서 (yml 형식)

  • 정의 가능한 내용
  1. 빌드환경(node.js, Python 등) 환경변수
  2. 스테이지별 명령어
  • install, pre_build, build, post_build
  1. 아티팩트 설정
  • path 및 구성 방법
  1. 기타(cache, proxy 등)

appSpec.yml

CodeDeploy에서 수행할 내용을 정의한 문서 (yml 형식)

  • 정의 가능한 내용
  1. 배포 방법
  • 각 스테이지 별 수행 내용 정의

배경

버전 컨트롤(Version Contro)은 시간에 따라 코드에 일어나는 다양한 변화를 이해하는 기능

  • 커밋/롤백 가능

  • git repository는 자신의 레포지터리 뿐 아니라 다른 사람의 레포지터리에도 살 수 있음

  • 다른 개발자들과 협업, 코드를 백업, 코드 보기, 언제든 롤백 가능

CodeCommit

AWS 계정에 대한 개인 Git 레포지터리

  • private Git repositories

  • No size limit on repositories (scale seamlessly)

  • Fully managed, high available

  • Code only in AWS Cloud account => increased security and compliance

  • Secure(encrypted, access control, etc..)

  • Integrated with Jenkins / CodeBuild / other CI tools

Git Command

  • HTTP 주소로 원격 레포지토리에 연결하기
$ git clone <원격 레포지터리 주소>
  • 브랜치를 만들고 이동하기
$ git checkoub -b <branch>
  • 만든 브랜치를 master 브랜치로 푸쉬하기
$ git push --set-upstream origin <branch>
