[Hive] Sorting

Hive
Hive
Publish Date:   2023-12-13

Hive에서 정렬을 수행하는 4개의 절

  • ORDER BY
  • SORT BY
  • DISTRIBUTED BY
  • CLUSTER BY

(1) ORDER BY

데이터 전체 정렬에 사용된다. 데이터의 전체 순서를 보장하며, 이를 위해 Single Reducer를 통과한다.

주의사항

  1. ORDER BY는 모든 데이터를 단일 리듀서를 통해 데이터를 반환하기 때문에 실제 프로덕션 환경에서는 상당히 고비용이 드는 쿼리이다. 따라서 LIMIT 절을 필수적으로 사용하여 전체 데이터를 모두 반환하는 게 아니라 필요 수 만큼만 반환하도록 해야 한다.

  2. sNULL 값 부터 시작하여 오름차순으로 정렬된다.

Query

SELECT col FROM table_name ORDER BY col;

(2) SORT BY

ORDER BY와 비슷하지만, 여러 개의 리듀서에서 각각 정렬이 수행된다는 점에서 차이가 있다. 따라서 전체 데이터의 정렬이 보장 되지 않고, 각 리듀서를 통과하는 데이터 내에서 각각 정렬이 된다.

Query

SELECT col FROM table_name SORT BY col;

(3) DISTRIBUTED BY

DISTRIBUTED BY 절은 키-값 쌍을 이용해 어떤 키가 어떤 리듀서로 갈 지를 결정한다. 컬럼 값에 따라 동일한 범주의 값을 가진 모든 행들이 특정한 리듀서로 모이도록 보장한다. 하지만, 이 구문은 실제 정렬을 수행하지는 않는다. 단순히 값들을 리듀서에 분배하는 Map 역할을 할 뿐이다. 정렬을 수행하려면 SORT BY 절을 결합해야 한다. 이 역시 SORT BY 이기 때문에 각 리듀서별로 정렬이 수행된다.

Query

SELECT col FROM table_name DISTRIBUTED BY col SORT BY col

(4) CLUSTER BY

( DISTRIBUTE BY + SORT BY )의 짧은 형태일 뿐이다. 로직 상으로 아무것도 다르지 않다.

Query

SELECT col FROM table_name CLUSTER BY col;
