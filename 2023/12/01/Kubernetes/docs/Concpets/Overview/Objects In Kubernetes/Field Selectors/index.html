Field Selectors

Field selectors를 사용하면 하나 이상의 리소스 필드 값에 따라 Kubernetes objects를 선택할 수 있습니다. 다음은 필드 셀렉터 쿼리의 예시입니다:

  • metadata.name=my-service
  • metadata.namespace!=default
  • status.phase=Pending

이 kubectl 명령은 status.phase 필드의 값이 Running인 모든 Pod를 선택합니다:

kubectl get pods --field-selector status.phase=Running

Note: 필드 셀렉터는 본질적으로 리소스 filters입니다. 기본적으로 선택기/필터가 적용되지 않으며, 즉 해당 유형의 모든 리소스가 선택됩니다. 이는 kubectl 쿼리 kubectl get pods와 kubectl get pods --field-selector ""이 동일하다는 것을 의미합니다.

Supported fields

지원되는 필드 셀렉터는 Kubernetes 리소스 유형에 따라 다릅니다. 모든 리소스 유형은 metadata.name 및 metadata.namespace 필드를 지원합니다. 지원되지 않는 필드 셀렉터를 사용하면 오류가 발생합니다. 예를 들어:

kubectl get ingress --field-selector foo.bar=baz
Error from server (BadRequest): Unable to find "ingresses" that match label selector "", field selector "foo.bar=baz": "foo.bar" is not a known field selector: only "metadata.name", "metadata.namespace"

Supported operators

=, == 및 != 연산자를 필드 셀렉터와 함께 사용할 수 있습니다 (=와 ==는 동일한 의미를 가집니다). 예를 들어, 이 kubectl 명령은 default 네임스페이스에 속하지 않는 모든 Kubernetes 서비스를 선택합니다:

kubectl get services  --all-namespaces --field-selector metadata.namespace!=default

Note: 집합 기반 연산자 (in, notin, exists)는 필드 셀렉터에 지원되지 않습니다.

Chained selectors

label 및 다른 셀렉터와 마찬가지로 필드 셀렉터는 쉼표로 구분된 목록으로 연결할 수 있습니다. 이 kubectl 명령은 status.phase가 Running이 아니고 spec.restartPolicy 필드가 Always인 모든 Pod를 선택합니다:

kubectl get pods --field-selector=status.phase!=Running,spec.restartPolicy=Always

Multiple resource types

여러 리소스 유형에 걸쳐 필드 셀렉터를 사용할 수 있습니다. 이 kubectl 명령은 default 네임스페이스에 속하지 않은 모든 Statefulset과 Service를 선택합니다:

kubectl get statefulsets,services --all-namespaces --field-selector metadata.namespace!=default
