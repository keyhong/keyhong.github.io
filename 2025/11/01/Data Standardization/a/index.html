데이터 표준화

Publish Date:   2025-11-01
데이터 표준화

표준화

코드, 용어, 데이터 도메인, 메타데이터 등의 표준을 수립하여 공공데이터베이스에 일관되게 적용하는 일련의 활동

표준 용어

데이터 사용자간 명확한 의사소통을 지원하기 위해 각 공공기관에서 업무적으로 사용하고 있는 단어 및 단어의 조합으로 구성되는 용어의 표준을 정의한 것

공통 표준 용어

행정안전부 장관이 정하여 공공기관들의 업무 및 데이터베이스에 공동 활용하는 표준 용어

데이터 표준화의 목적

  • 데이터 표준화는 누구나 같은 의미로 이해하고 같은 방법으로 사용할 수 있는 기준을 정의함으로서 정확한 정보의 유통 및 활용을 목적으로 한다.
누구나 같은 의미로 이해 같은 방법으로 사용
- 전산
- 현업
- 고객		 - 의미를 명확하게 알 수 있는 명칭
- 유일한 의미를 가지는 용어		 - 데이터 표현방법
- 제약 조건

데이터 명칭 표준화 데이터 형식, 규칙 표준화
- 포함되는 문자 ; 한글, 영문, 숫자, 특수문자 …
- 기본단어
- 기본단어의 배열 순서
- 주제별 명명 기준 : 조직, 고객, 매체, 연락처, 수, 금액 …
- 동음이의어, 이음동의어
- 풀어쓴 말, 줄인 말		 - 데이터 타입
- 데이터 길이
- 기본 값
- 제약조건
- 값의 범위
- 단위

데이터 표준화 구성요소

구성요소 주요내용
정책 데이터 표준화 지침(가이드) - 데이터 표준 관리 원칙, 절차, 조직, R&R
- 데이터 사전 생성 명명 규칙, 제약조건 등
- 용어 오너쉽(ownership): 생성 및 사용의 책임 소관 부서
데이터 사전 - 단어사전
- 용어사전
- 도메인사전
- 코드 사전
시스템 메타데이터 관리 시스템 - 데이터 표준 사전 정보 관리, 데이터 모델 형상 정보 관리, 데이터베이스 형상 정보 관리
- 데이터 영향도 분석

문제점 및 개선 효과

현재 데이터 운영 문제 데이터 표준화 개선 효과
각 업무영역 간 틀린 표준용어를 사용하며 커뮤니케이션에 어려움이 있다. 전사 표준용어를 사용하고 표준화되지 않은 용어도 연결하여 관리하여 커뮤니케이션 문제 해결
업무영영간 데이터 정합성 유지에 어려움이 있다. 영역 간 데이터 연관 관계를 한 눈에 파악할 수 있어 데이터 정합성 유지가 쉬워진다
각 업무영역 간 틀린 코드체게가 존재하여 통계작업을 하거나 보고서를 출력할 때 일관성이 없는 경우가 발생할 수 있다. 한 영역의 데이터 구조 변경 시 영향 분석을 통해 전 영역에 동시에 반영한다.
새로 업무를 담당하는 사람에게 보여줄 최근 데이터 구조에 관한 자료가 없다. 메타데이터 관리 시스템을 통해 전사 데이터 구조를 파악할 수 있다.
프로젝트가 생길 때 마다 데이터 명세를 정리하는 작업을 반복한다. 메타데이터 관리 시스템을 통해 항상 조회할 수 있으므로 별도로 데이터 항목 조사를 할 필요가 없다.
참조하고 있는 문서가 최종자료라는 것을 아무도 보장하지 않는다. 각자 자기가 소유한 문서만 바라본다. 모든 사람들이 항상 최근 자료를 참조하고 있다. 테이블 목록, 테이블 정의서, 코드 명세서와 같은 문서가 사라진다.
