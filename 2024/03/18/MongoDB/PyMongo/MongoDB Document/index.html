MongoDB

Documents

MongoDB stores data records as BSON documents. BSON is a binary representation of JSON documents, though it contains more data types than JSON. For the BSON spec, see
bsonspec.org
. See also BSON Types.


MongoDB는 데이터 레코드를 BSON 문서로 저장합니다. BSON은 JSON 문서의 이진 표현이지만 JSON보다 더 많은 데이터 유형을 포함합니다. BSON 사양은 bsonspec.org에서 확인할 수 있습니다. BSON 유형도 참조하세요.

Compatibility

MongoDB stores records as documents for deployments hosted in the following environments:

MongoDB는 다음 환경에 호스트된 배포에 대해 레코드를 문서로 저장합니다:

  • MongoDB Atlas: The fully managed service for MongoDB deployments in the cloud

  • MongoDB Enterprise: The subscription-based, self-managed version of MongoDB

  • MongoDB Community: The source-available, free-to-use, and self-managed version of MongoDB


  • MongoDB Atlas: 클라우드에서 MongoDB 배포를 완전히 관리하는 서비스

  • MongoDB Enterprise: MongoDB의 구독 기반 자체 관리 버전

  • MongoDB Community: 소스가 제공되며 무료로 사용할 수 있는 자체 관리형 MongoDB 버전

Document Structure

MongoDB documents are composed of field-and-value pairs and have the following structure:


MongoDB 문서는 필드-값 쌍으로 구성되어 있으며 다음과 같은 구조를 갖습니다:

{
   field1: value1,
   field2: value2,
   field3: value3,
   ...
   fieldN: valueN
}

The value of a field can be any of the BSON data types, including other documents, arrays, and arrays of documents. For example, the following document contains values of varying types:


필드의 값은 다른 문서, 배열 및 문서 배열을 포함한 BSON 데이터 유형 중 하나가 될 수 있습니다. 예를 들어, 다음 문서는 다양한 유형의 값이 포함되어 있습니다:

var mydoc = {
               _id: ObjectId("5099803df3f4948bd2f98391"),
               name: { first: "Alan", last: "Turing" },
               birth: new Date('Jun 23, 1912'),
               death: new Date('Jun 07, 1954'),
               contribs: [ "Turing machine", "Turing test", "Turingery" ],
               views : NumberLong(1250000)
            }

The above fields have the following data types:

  • _id holds an ObjectId.
  • name holds an embedded document that contains the fields first and last.
  • birth and death hold values of the Date type.
  • contribs holds an array of strings.
  • views holds a value of the NumberLong type.

위의 필드는 다음과 같은 데이터 유형을 가지고 있습니다:

  • _id: ObjectId를 보유합니다.
  • name: 이름은 first와 last 필드를 포함한 임베디드 문서를 보유합니다.
  • birth와 death: Date 유형의 값입니다.
  • contribs: 문자열 배열을 보유합니다.
  • views: NumberLong 유형의 값을 보유합니다.

Field Names

Field names are strings.

Documents have the following restrictions on field names:

  • The field name _id is reserved for use as a primary key; its value must be unique in the collection, is immutable, and may be of any type other than an array. If the _id contains subfields, the subfield names cannot begin with a ($) symbol.

  • Field names cannot contain the null character.

  • The server permits storage of field names that contain dots (.) and dollar signs ($).

  • MongodB 5.0 adds improved support for the use of ($) and (.) in field names. There are some restrictions. See Field Name Considerations for more details.


필드 이름은 문자열입니다.

문서는 필드 이름에 대해 다음과 같은 제한을 가지고 있습니다:

  • 필드 이름 _id는 주 키로 사용하기 위해 예약되어 있으며, 해당 값은 컬렉션 내에서 고유해야 하며 변경할 수 없으며, 배열이 아닌 모든 유형의 값이 될 수 있습니다. _id에 하위 필드가 포함된 경우 하위 필드 이름은 ($) 기호로 시작할 수 없습니다.

  • 필드 이름은 null 문자를 포함할 수 없습니다.

  • 서버는 점 (.) 및 달러 기호 ($)를 포함하는 필드 이름을 저장할 수 있습니다.

  • MongoDB 5.0에서는 필드 이름에 대한 ($) 및 (.) 사용 지원이 개선되었습니다. 일부 제한 사항이 있습니다. 자세한 내용은 Field Name Considerations를 참조하세요.

The MongoDB Query Language does not support documents with duplicate field names. While some BSON builders may support creating a BSON document with duplicate field names, inserting these documents into MongoDB is not supported even if the insert succeeds, or appears to succeed. For example, inserting a BSON document with duplicate field names through a MongoDB driver may result in the driver silently dropping the duplicate values prior to insertion, or may result in an invalid document being inserted that contains duplicate fields. Querying against any such documents would lead to arbitrary and inconsistent results.


MongoDB 쿼리 언어는 중복 필드 이름을 가진 문서를 지원하지 않습니다. 일부 BSON 빌더는 중복 필드 이름을 가진 BSON 문서를 생성하는 것을 지원할 수 있지만, 이러한 문서를 MongoDB에 삽입하는 것은 지원되지 않습니다. 삽입이 성공하더라도 또는 성공한 것처럼 보이더라도 중복 필드 이름을 가진 BSON 문서를 삽입하는 것은 지원되지 않습니다. 예를 들어, MongoDB 드라이버를 통해 중복 필드 이름을 가진 BSON 문서를 삽입하면 드라이버가 삽입 전에 중복 값을 무시하거나 중복 필드를 포함하는 잘못된 문서가 삽입될 수 있습니다. 이러한 문서를 대상으로 쿼리를 실행하면 임의적이고 일관성이 없는 결과가 발생할 수 있습니다.

Field Value Limit

MongoDB 2.6 through MongoDB versions with featureCompatibilityVersion (fCV) set to “4.0” or earlier

For indexed collections, the values for the indexed fields have a Maximum Index Key Length. See Maximum Index Key Length for details.

MongoDB 2.6부터 featureCompatibilityVersion (fCV)이 “4.0”이거나 그 이전 버전의 MongoDB 버전들

인덱스된 컬렉션의 경우, 인덱스 필드의 값은 최대 인덱스 키 길이를 갖습니다. 자세한 내용은 최대 인덱스 키 길이를 참조하십시오.

Dot Notation

MongoDB uses the dot notation to access the elements of an array and to access the fields of an embedded document.

MongoDB는 배열의 요소 및 내장 문서의 필드에 액세스하기 위해 점 표기법(dot notation)을 사용합니다.

Arrays

To specify or access an element of an array by the zero-based index position, concatenate the array name with the dot (.) and zero-based index position, and enclose in quotes:

배열의 요소를 제로 기반 인덱스 위치로 지정하거나 액세스하려면 배열 이름과 점(.) 그리고 제로 기반 인덱스 위치를 연결하고 따옴표로 묶어주세요.

"<array>.<index>"

For example, given the following field in a document:

예를 들어, 다음과 같은 문서의 필드가 있다고 가정합니다.

{
   ...
   contribs: [ "Turing machine", "Turing test", "Turingery" ],
   ...
}

To specify the third element in the contribs array, use the dot notation “contribs.2“.

For examples querying arrays, see:

contribs 배열의 세 번째 요소를 지정하려면 “contribs.2”와 같이 점 표기법을 사용합니다.
