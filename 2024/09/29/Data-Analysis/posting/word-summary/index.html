분석 지표 정리

데이터 분석

1. 마케팅 용어

Impression

광고가 사용자에게 보여지는 것 또는 검색 사용자에게 광고가 노출된 횟수

ROAS (Return On Ad Spend)

광고비 대비 발생한 매출 비율

= (매출 / 광고비) * 100

CTR (Click Through Rate)

광고 노출 대비 클릭 비율. 해당 광고가 특정 기간동안 클릭된 횟수를 노출된 횟수로 나눈값을 백분율로 표시한 값.

= (클릭 수 / 노출 수(Impression)) * 100

CVR (ConVersion Rate)

(1) 클릭 대비 설치 전환율

= (전환된 설치 / 클릭 수) * 100

(2) 설치 대비 이벤트 전환율

= (전환된 행동 / 설치) * 100

CPI (Cost Per Install)

설치 한 건당 발생한 광고비용

= 광고비 / 설치 수(Install)

CPC (Cost Per Click)

온라인 광고 가격을 책정하는 대표적인 방법. 사용자가 검색 결과로 보여진 광고를 클릭하여 광고주의 사이트로 이동했을 때에 광고비가 소진되는 과금 체계

CPM (Cost Per Mile)

온라인 광고 가격을 책정하는 방법 중 하나로, 1000회 광고 노출을 기준으로 가격을 책정하는 과금 체계

= 광고비 / 노출 수 * 1000

페이지 RPM

총 수입을 페이지 조회수 1000회로 나눈 값. 1000회 노출당 페이지 수익(RPM)은 예상 수입을 페이지 조회수로 나눈 다음 1000을 곱한 값. (예상수입/페이지 조회수) * 1000

CPC (Cost Per Click, PPC=Pay Per Click)

평균클릭비용. 전체 광고비를 클릭 수로 나눈 값으로 클릭 1회당 단가. (광고비/클릭) 광고 효과를 측정하는 지표 중 하나

2. 유저 관련 용어

UV (Uniuqe Visitor)

순 방문자. 유저의 고유 카운트

AU (Active User)

활성 유저. 산업에 따라 접속한 유저 혹은 구매 유저 등 다른 기준을 통해 활성 유저를 정의한다. 일반적으로 DAU와 동일시한다.

RU (Registerd User)

전체 가입자 수

NRU (New Registered User)

신규등록유저. 게임 설치 뿐만 아니라 계정 등록을 한 유저. 기간 내 순 신규 가입자 수

  • user_id : int : 각 사용자의 ID (기본 키)
  • username : varchar(50) : 사용자의 이름
  • email : varchar(100) : 사용자의 이메일 주소 (고유)
  • registration_date : datetime : 사용자가 등록한 날짜
  • last_login_date : datetime : 사용자가 마지막으로 로그인한 날짜
SELECT DATE_FORMAT(registration_date, '%Y-%m') AS registration_month
     , COUNT(DISTINCT user_id) AS nru
FROM users
WHERE registration_date BETWEEN '2024-01-01' AND '2024-01-31'
GROUP BY DATE_FORMAT(registration_date, '%Y-%m')
ORDER BY registration_month
;

NPU (New Paid User)

신규 구매(과금) 유저

MUV (Multigame Unique Visitor)

특정 기간 클라이언트 혹은 애플리케이션을 내려받아 로그인 및 플레이한 방문자

3. 유저 접속 관련 용어

CCU (Concurrent User)

특정 시간의 동시 접속자 수

  • session_id : int : 세션ID
  • user_id : int : 사용자ID
  • session_start : datetime : 세션시작시간
  • session_end : datetime : 세션종료시간(선택)
SELECT COUNT(DISTINCT user_id) AS ccu
FROM user_sessions
WHERE session_start <= '2024-09-29 15:00:00' -- 측정할 시점
  AND (session_end IS NULL OR session_end >= '2024-09-29 15:00:00');

MCU (Max Concurrent User, 또는 PCU=Peak Concurrent User)

기준 단위 동시 접속자 최대값 (예: 5분)

SELECT MAX(ccu) AS PCU
FROM (
    SELECT COUNT(DISTINCT user_id) AS ccu
    FROM user_sessions
    WHERE start_session <= '2024-09-29 23:59:59' -- 하루의 끝
    AND (session_end IS NULL OR session_end >= '2024-09-29 00:00:00'); -- 하루의 시작
    GROUP BY DATEFORMAT(start_session, '%Y-%m-%d %H:00:00') -- 시간 단위 그룹화
    -- GROUP BY DATEFORMAT(start_session, '%Y-%m-%d %H:%i:00') -- 분 단위 그룹화
)

ACU (Average Concurrent User)

기준 단위 동시 접속자 평균값 (예: 5분)

SELECT AVG(ccu) AS PCU
FROM (
    SELECT COUNT(DISTINCT user_id) AS ccu
    FROM user_sessions
    WHERE start_session <= '2024-09-29 23:59:59' -- 하루의 끝
    AND (session_end IS NULL OR session_end >= '2024-09-29 00:00:00'); -- 하루의 시작
    GROUP BY DATEFORMAT(start_session, '%Y-%m-%d %H:00:00') -- 시간 단위 그룹화
    -- GROUP BY DATEFORMAT(start_session, '%Y-%m-%d %H:%i:00') -- 분 단위 그룹화
)

DAU (Daily Active User)

하루 내 게임 순 방문자 수

SELECT event_date
     , COUNT(DISTINCT user_id) AS DAU
FROM fact_user_session_day
WHERE event_date BETWEEN DATE_SUB(NOW(), INTERVAL 1 MONTH) AND NOW()
GROUP BY event_date
ORDER BY event_date DESC
;

WAU (Weekly Active User)

한 주 내 게임 순 방문자 수

SELECT fo.event_date
     , COUNT(DISTINCT ft.customer_id) AS WAU
FROM fact_user_session_day AS fo
    LEFT JOIN fact_user_session_day AS ft
           ON ft.event_date BETWEEN DATE_SUB(fo.event_date, INTERVAL 6 DAY) AND fo.event_date
GROUP BY fo.event_date

MAU (Monlth Active User)

한달 내 게임 순 방문자 수

Stickiness

DAU / WAU
SELECT fo.event_date
     , COUNT(DISTINCT fo.customer_id) AS DAU
     , COUNT(DISTINCT ft.customer_id) AS WAU
     , COUNT(DISTINCT fo.customer_id) / COUNT(DISTINCT ft.customer_id) AS stickiness
FROM fact_user_session_day AS fo
    LEFT JOIN fact_user_session_day AS ft
           ON ft.event_date BETWEEN DATE_SUB(fo.event_date, INTERVAL 6 DAY) AND fo.event_date
GROUP BY fo.event_date
DAU / MAU
SELECT fo.event_date
     , COUNT(DISTINCT fo.customer_id) AS DAU
     , COUNT(DISTINCT ft.customer_id) AS MAU
     , COUNT(DISTINCT fo.customer_id) / COUNT(DISTINCT ft.customer_id) AS stickiness
FROM fact_user_session_day AS fo
    LEFT JOIN fact_user_session_day AS ft
           ON ft.event_date BETWEEN DATE_SUB(fo.event_date, INTERVAL 29 DAY) AND fo.event_date
GROUP BY fo.event_date

ARPDAU (Average Revenue Per Daily Active User)

SELECT event_date
     , SUM(revenue) AS daily_revenue
     , ROUND(SUM(revenue) / COUNT(DISTINCT user_id), 2) AS ARPDAU
FROM fact_user_session_day
WHERE event_date BETWEEN DATE_SUB(NOW(), INTERVAL 1 MONTH) AND NOW()
GROUP BY event_date
ORDER BY event_date DESC
;

4. 과금 관련 지표

PUR (Paid User Rate, 또는 BUR=Buying User Rate)

접속 유저 대비 결제 유저 비율

NPUR (New Paid User Rate)

접속 유저 대비 신규 결제 유저 비율

Revenue

실제 발생한 매출

ARPU (Average Revenue Per User)

유저당 매출

= 매출(Sale) / User

ARPPU (Average Revenue Per Paid User)

결제유저(PU)당 매출

= 매출(Sale) / PU(Paying User)

ARPAU (Average Revenue Per Active User)

활성유저(AU)당 결제액

= 매출(Sales) / AU(Active User)

RPU (Revenue Per User)

사용자 당 수익

CAC (Customer Acquisition COst)

고객 획득 비용

= 신규 고객 확보 비용 / 확보된 신규 고객 수

5. 모객 관련 용어

RET (Retension)

잔존율, 유지율, 유저 잔존 여부

LTV (LifeTime Value)

유저 1명당 생애 가치

= 누적 매출 (Sales) / RU
= 일평균 지불금액(ARPDAU) x 지속방문일 (LT, Life Time)
= 사용자당 연간 평균 이익 * 평균 고객 생애주기
= (((1인당 1년 평균매출) - (1인당 1년 들어가는 평균 서비스 비용)) / (1-고객유지비율)) - 고객획득비용(AC)
= ARPU X (1 / Churn rate)

** 이탈률 = 1 - Retention Rate(리텐션율)

이탈률 = ((측정 기간 시작 시점의 고객 수 - 측정 기간 종료 시점의 고객 수) / 측정 기간 시작 시점의 고객 수) * 100

LTR

고객 생애 매출

= 매출 / 고객수

5. 기타

IAP (In App Purchase)

신용카드를 통한 인앱 결제

IAA (In App Ads)

인 앱 광고로 발생되는 매출

DL(Download)

기간 내 다운로드 횟수

TS (Time Spent)

게임에 방문한 이용자의 평균 체류 시간

= ((시그마)로그아웃 시간 - 로그인 시간) / MU

문맥광고

페이지 내 컨텐츠를 분석하여 관련 키워드를 추출하고 광고를 노출하는 방식

Landing page

인터넷 사용자가 광고를 클릭하고 광고주의 사이트에 방문했을 때 최초로 보게 되는 웹 페이지

PV (Page View)

홈페이지에 들어온 접속자가 둘러 본 페이지

ASO (App Store Optimization)

앱 스토어 최적화. 앱 아이콘, 스크린샷, 영상, 디스크립션 등 앱을 설명하는 요소를 바꾸면서 전환율을 높이는 것. 앱스토어 최적화를 통해 사용자가 앱 설치를 많이 하면, CPI가 낮아진다.

FTUE (First Time User Experience)

최초 사용자 경험

웹 = cookie 기반, 앱 = device_id 기반 (IDFA: ios, GAID: 안드로이드)
