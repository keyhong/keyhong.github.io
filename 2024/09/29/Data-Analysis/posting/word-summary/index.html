AARRR 지표 정리

데이터 분석

1. 마케팅 용어

1.1 Imps

  • 중요도 : ⭐⭐⭐
  • Impression

광고가 사용자에게 노출된 횟수

1.2 ROAS

  • 중요도 : ⭐⭐⭐
  • Return On Ad Spend

광고비 대비 발생한 매출 비율. 마케팅 캠페인에 쓰인 1달러 당 수익률

  • 산출식 : (매출 / 광고비) * 100 (%)

1.3 CTR

  • 중요도 : ⭐⭐⭐
  • Click Through Rate

노출 대비 클릭 비율. 해당 광고가 특정 기간동안 클릭된 횟수를 노출된 횟수로 나눈값을 백분율로 표시한 값. 유저의 중복이 있음

  • 산출식 : (클릭 수 / 노출 수(Imps)) * 100 (%)

1.4 CVR

  • 중요도 : ⭐⭐
  • ConVersion Rate

A(클릭, 설치, 노출) 대비 B(설치, 이벤트, 클릭) 전환 비율. 전환의 기준은 정의에 따라 다르다. CTR과 다르게 unique하게 사용할 수도 있음

(1) 클릭 대비 설치 전환율

  • 산출식 : (전환된 설치 수(Install) / 클릭 수) * 100 (%)

(2) 설치 대비 이벤트 전환율

  • 산출식 : (전환된 행동 수 / 설치 수(Install)) * 100 (%)

(3) 노출 대비 클릭 전환율

  • 산출식 : (클릭한 unique 유저 수 / 노출(Imps)된 unique 유저 수) * 100 (%)

1.5 CPC

  • 중요도 : ⭐⭐⭐
  • Cost Per Click

온라인 광고 가격을 책정하는 대표적인 방법. 평균클릭비용. 사용자가 검색 결과로 보여진 광고를 클릭하여 광고주의 사이트로 이동했을 때에 광고비가 소진되는 과금 체계

  • 산출식 : 광고비 / 클릭 수

  • CTR이 높으면, 즉 광고가 많이 클릭될수록 CPC 방식에서는 광고주가 클릭당 비용을 지불하기 때문에, 클릭 수가 증가할수록 총 광고비도 함께 증가한다.

  • 기업의 아이덴티티를 제고하는 브랜드 광고라면, CPC 방식이 상대적으로 비용을 덜 지출하게 된다.

1.6 CPM

  • 중요도 : ⭐⭐
  • Cost Per Mile

온라인 광고 가격을 책정하는 방법 중 하나로, 1000회 광고 노출을 기준으로 가격을 책정하는 과금 체계

  • 산출식 : 광고비 / 노출 수(Imps) * 1,000

  • CTR이 높으면, 클릭 수와는 관계없이 CPM 방식에서는 광고주가 광고 노출 수에 기반하여 비용이 발생하기 때문에, 상대적으로 적은 비용으로 더 많은 클릭을 유도하여 총 광고비 부담을 줄일 수 있다.

  • 실제 상품 구매의 전환을 목표로 하는 브랜드 광고라면, CPM 방식이 상대적으로 비용을 덜 지출하게 된다.

1.7 eCPM

  • 중요도 : ⭐⭐⭐
  • effective Cost Per Mile

유효한 1,000회 노출에 대한 비용. 광고 수익을 측정하는 지표로, 광고가 천 번 노출될 때 발생하는 유효한 평균 수익

1.8 CAC

  • 중요도 : ⭐⭐⭐
  • Customer Acquisition Cost

고객 획득 비용

  • 산출식 : 신규 고객 확보 비용 / 확보된 신규 고객 수

1.9 CPA

  • Cost Per Action

특정 소비자 행동(방문, 다운로드, 가입, 리드 생성) 대비 광고 비용. 광고주에게 광고 비용을 효율적으로 관리할 수 있는 방법을 제공하며, 광고 성과에 대한 직접적인 반응을 측정. 광고가 실제로 소비자 행동으로 이어질 때만 비용이 발생하므로, 광고주가 ROI(투자 수익률)를 높이는 데 도움이 됨

  • 산출식 : 광고비 / 전환수

1.10 CPI

  • Cost Per Install

설치 한 건당 발생한 광고비용

  • 산출식 : 광고비 / 설치 수(Install)

1.11 RPM

  • Page Revenue per Mile

총 수입을 페이지 조회수 1,000회로 나눈 값

  • 산출식 : 예상 수입 / 페이지 조회수 * 1,000

1.12 UA

  • User Acquisition

유저로부터 특정 행동을 이끌어내는 마케팅 기법

2. 유저 관련 용어

2.1 UV

  • Uniuqe Visitor

순 방문자 수. 유저의 고유 카운트 (DISTINCT COUNT)

2.2 AU

  • Active User

특정 기간 동안 서비스나 제품을 사용한 사용자. 활성 유저. 산업군에 따라 접속 유저 혹은 구매 유저 등 다른 기준으로 활성 유저를 정의함

2.3 RU

  • Registerd User

전체 가입자 수

2.4 NRU

  • 중요도 : ⭐⭐⭐
  • New Registered User. 또는 NU로 사용

게임 설치 뿐만 아니라, 계정 등록을 한 신규등록유저. 기간 내 순 신규 가입자 수 (DISTINCT COUNT)

  • user_id : int : 각 사용자의 ID (@pk)
  • username : varchar : 사용자의 이름
  • email : varchar : 사용자의 이메일 주소 (고유)
  • registration_date : datetime : 사용자가 등록한 날짜
  • last_login_date : datetime : 사용자가 마지막으로 로그인한 날짜
일별 NRU 집계 쿼리
SELECT registration_date AS registration_day
     , COUNT(DISTINCT user_id) AS daily_nru
FROM users
WHERE registration_date BETWEEN '2024-01-01' AND '2024-01-31'
GROUP BY registration_date
ORDER BY registration_date ASC
;
월별 NRU 집계 쿼리
SELECT DATE_FORMAT(registration_date, '%Y-%m') AS registration_month
     , COUNT(DISTINCT user_id) AS monthly_nru
FROM users
WHERE registration_date BETWEEN '2024-01-01' AND '2024-01-31'
GROUP BY DATE_FORMAT(registration_date, '%Y-%m')
ORDER BY registration_month ASC
;

2.5 PU

  • Paying User

결제(구매, 과금) 유저

2.6 NPU

  • New Paid User

신규 결제(구매, 과금) 유저

3. 유저 유입 관련 용어

3.1 DAU

  • 중요도 : ⭐⭐⭐
  • Daily Active User

하루 내 게임 순 방문자 수

SELECT event_date
     , COUNT(DISTINCT user_id) AS DAU
FROM fact_user_session_day
WHERE event_date BETWEEN DATE_SUB(NOW(), INTERVAL 1 MONTH) AND NOW()
GROUP BY event_date
ORDER BY event_date DESC
;

3.2 WAU

  • Weekly Active User

한 주 내 게임 순 방문자 수

SELECT fo.event_date
     , COUNT(DISTINCT ft.customer_id) AS WAU
FROM fact_user_session_day AS fo
    LEFT JOIN fact_user_session_day AS ft
           ON ft.event_date BETWEEN DATE_SUB(fo.event_date, INTERVAL 6 DAY) AND fo.event_date
GROUP BY fo.event_date

3.3 MAU

  • 중요도 : ⭐⭐⭐
  • Monlth Active User

한달 내 게임 순 방문자 수

3.4 Stickiness

  • 중요도 : ⭐⭐⭐

사용자가 서비스에 얼마나 자주 돌아오는지를 나타내는 지표. 사용자 충성도와 서비스 유지율을 평가하는 데 사용

  • 산출식 1 (주 기준): DAU / WAU * 100 (%)
Stickiness: DAU / WAU
SELECT fo.event_date
     , COUNT(DISTINCT fo.customer_id) AS DAU
     , COUNT(DISTINCT ft.customer_id) AS WAU
     , COUNT(DISTINCT fo.customer_id) / COUNT(DISTINCT ft.customer_id) AS stickiness
FROM fact_user_session_day AS fo
    LEFT JOIN fact_user_session_day AS ft
           ON ft.event_date BETWEEN DATE_SUB(fo.event_date, INTERVAL 6 DAY) AND fo.event_date
GROUP BY fo.event_date
  • 산출식 2 (월 기준): DAU / MAU * 100 (%)
Stickiness: DAU / MAU
SELECT fo.event_date
     , COUNT(DISTINCT fo.customer_id) AS DAU
     , COUNT(DISTINCT ft.customer_id) AS MAU
     , COUNT(DISTINCT fo.customer_id) / COUNT(DISTINCT ft.customer_id) AS stickiness
FROM fact_user_session_day AS fo
    LEFT JOIN fact_user_session_day AS ft
           ON ft.event_date BETWEEN DATE_SUB(fo.event_date, INTERVAL 29 DAY) AND fo.event_date
GROUP BY fo.event_date

3.5 CCU

  • Concurrent User

특정 시간의 동시 접속자 수

  • session_id : int : 세션ID
  • user_id : int : 사용자ID
  • session_start : datetime : 세션시작시간
  • session_end : datetime : 세션종료시간(선택)
SELECT COUNT(DISTINCT user_id) AS ccu
FROM user_sessions
WHERE session_start <= '2024-09-29 15:00:00' -- 측정할 시점
  AND (session_end IS NULL OR session_end >= '2024-09-29 15:00:00');

3.6 MCU

  • Max Concurrent User

기준 단위 동시 접속자 최대값 (예: 5분). PCU (Peak Concurrent User)라고도 함

SELECT MAX(ccu) AS PCU
FROM (
    SELECT COUNT(DISTINCT user_id) AS ccu
    FROM user_sessions
    WHERE start_session <= '2024-09-29 23:59:59' -- 하루의 끝
    AND (session_end IS NULL OR session_end >= '2024-09-29 00:00:00'); -- 하루의 시작
    GROUP BY DATEFORMAT(start_session, '%Y-%m-%d %H:00:00') -- 시간 단위 그룹화
    -- GROUP BY DATEFORMAT(start_session, '%Y-%m-%d %H:%i:00') -- 분 단위 그룹화
)

3.7 ACU

  • Average Concurrent User

기준 단위 동시 접속자 평균값 (예: 5분)

SELECT AVG(ccu) AS PCU
FROM (
    SELECT COUNT(DISTINCT user_id) AS ccu
    FROM user_sessions
    WHERE start_session <= '2024-09-29 23:59:59' -- 하루의 끝
    AND (session_end IS NULL OR session_end >= '2024-09-29 00:00:00'); -- 하루의 시작
    GROUP BY DATEFORMAT(start_session, '%Y-%m-%d %H:00:00') -- 시간 단위 그룹화
    -- GROUP BY DATEFORMAT(start_session, '%Y-%m-%d %H:%i:00') -- 분 단위 그룹화
)

4. 과금 관련 지표

4.1 Revenue

  • 중요도 : ⭐⭐⭐

특정 기간 동안 기업이나 서비스가 제품 판매, 서비스 제공, 혹은 기타 수익 활동을 통해 벌어들인 총 금액. 인 앱 결제(IAP) 매출

4.2 PUR

  • 중요도 : ⭐⭐⭐
  • Paid User Rate

특정 기간동안 전체 사용자 중에서 실제로 결제를 한 사용자의 비율을 나타내는 지표. 서비스나 게임에서 수익화를 평가하는 데 매우 중요한 지표로 사용. 접속 유저 대비 결제 유저 비율. BUR (Buying User Rate)이라고도 함

  • 산출식 : PU(결제유저 수) / UV(순방문자 수) * 100

4.3 NPUR

  • New Paid User Rate

접속 유저 대비 신규 결제 유저 비율

4.4 ARPU

  • 중요도 : ⭐⭐⭐
  • Average Revenue Per User

사용자당 평균 수익

  • 산출식 : Revenue / User

4.5 ARPPU

  • 중요도 : ⭐⭐⭐
  • Average Revenue Per Paid User

유료 사용자(PU)당 평균 수익. 전체 사용자 중 결제를 한 유료 사용자 한 명당 평균적으로 발생한 수익을 나타내는 지표

  • 산출식 : Revenue / PU(Paying User)

4.6 ARPAU

  • Average Revenue Per Active User

활성유저(AU)당 결제액

  • 산출식 : Revenue / AU(Active User)

4.7 ARPDAU

  • Average Revenue Per Daily Active User

일일 활성 사용자(DAU)당 평균 수익. 주로 모바일 게임이나 앱의 수익성을 평가하는 데 사용됨

SELECT event_date
     , SUM(revenue) AS daily_revenue
     , ROUND(SUM(revenue) / COUNT(DISTINCT user_id), 2) AS ARPDAU
FROM fact_user_session_day
WHERE event_date BETWEEN DATE_SUB(NOW(), INTERVAL 1 MONTH) AND NOW()
GROUP BY event_date
ORDER BY event_date DESC
;

5. 모객 관련 용어

5.1 Retension Rate

  • 중요도 : ⭐⭐⭐

잔존율, 유지율, 유저 잔존 비율. 오늘 처음 진입한 유저(NRU) 중 몇 명이 다음 날 다시 접속했는지를 측정한 비율

Monthly Classic Retention Query (MySQL)
WITH records_preprocessed AS (
  SELECT r.customer_id
       , r.order_id
       , r.order_date
       , c.first_order_date
       , DATE_FORMAT(r.order_date, '%Y-%m-01') AS order_month
       , DATE_FORMAT(c.first_order_date, '%Y-%m-01') AS first_order_month
  FROM records AS r
    JOIN customer_stats AS c ON r.customer_id = c.customer_id
)
SELECT first_order_month
    , COUNT(DISTINCT customer_id) AS month0
    , COUNT(DISTINCT CASE WHEN (DATE_ADD(first_order_month, INTERVAL 1 MONTH) = order_month) THEN customer_id END) AS month1
    , COUNT(DISTINCT CASE WHEN (DATE_ADD(first_order_month, INTERVAL 2 MONTH) = order_month) THEN customer_id END) AS month2
    , COUNT(DISTINCT CASE WHEN (DATE_ADD(first_order_month, INTERVAL 3 MONTH) = order_month) THEN customer_id END) AS month3
    , COUNT(DISTINCT CASE WHEN (DATE_ADD(first_order_month, INTERVAL 4 MONTH) = order_month) THEN customer_id END) AS month4
    , COUNT(DISTINCT CASE WHEN (DATE_ADD(first_order_month, INTERVAL 5 MONTH) = order_month) THEN customer_id END) AS month5
    , COUNT(DISTINCT CASE WHEN (DATE_ADD(first_order_month, INTERVAL 6 MONTH) = order_month) THEN customer_id END) AS month6
    , COUNT(DISTINCT CASE WHEN (DATE_ADD(first_order_month, INTERVAL 7 MONTH) = order_month) THEN customer_id END) AS month7
    , COUNT(DISTINCT CASE WHEN (DATE_ADD(first_order_month, INTERVAL 8 MONTH) = order_month) THEN customer_id END) AS month8
    , COUNT(DISTINCT CASE WHEN (DATE_ADD(first_order_month, INTERVAL 9 MONTH) = order_month) THEN customer_id END) AS month9
    , COUNT(DISTINCT CASE WHEN (DATE_ADD(first_order_month, INTERVAL 10 MONTH) = order_month) THEN customer_id END) AS month10
    , COUNT(DISTINCT CASE WHEN (DATE_ADD(first_order_month, INTERVAL 11 MONTH) = order_month) THEN customer_id END) AS month11
FROM records_preprocessed
GROUP BY first_order_month
ORDER BY first_order_month ASC

5.2 Bounce Rate

특정 기간 동안의 고객의 이탈 비율

  • 산출식
    = 1 - Retention Rate(리텐션율)
    = ((측정 기간 시작 시점의 고객 수 - 측정 기간 종료 시점의 고객 수) / 측정 기간 시작 시점의 고객 수) * 100

5.3 LTV

  • 중요도 : ⭐⭐⭐
  • LifeTime Value

고객 1명당 생애 동안 발생시키는 가치 (매출)

  • 산출식
    = 누적 매출(Sales) / RU(등록유저)
    = 일평균 지불금액(ARPDAU) x 지속방문일 (LT, Life Time)
    = 사용자당 연간 평균 이익 * 평균 고객 생애주기
    = (((1인당 1년 평균매출) - (1인당 1년 들어가는 평균 서비스 비용)) / (1-고객유지비율)) - 고객획득비용(AC)
    = ARPU X (1 / Churn rate)

** 실제 고객이 생애주기 동안 발생시키는 매출은 미래에 대한 예측이 필요하며, 정확히 예측이 불가능함. 따라서 이를 평가하기 위한 다양한 방법론이 있음

5.4 LTR

고객 생애 매출

  • 산출식 : 매출 / 고객수

6. 기타

GMV

  • 중요도 : ⭐⭐
  • Gross Merchandise Value

총 상품 거래액을 의미하며, 온라인 마켓플레이스나 이커머스 플랫폼에서 일정 기간 동안 고객이 구매한 상품의 총 거래 금액. 세금, 수수료, 할인 등을 제외하기 전의 총 매출

OKR

  • 중요도 : ⭐⭐⭐
  • Objective and Key Results

목표(Objective)와 이를 달성하기 위한 핵심 결과(Key Results)를 설정하는 프레임워크로, 기업이나 팀의 성과를 측정하고 관리하기 위한 방법론. 기업의 모든 부서와 팀이 공통의 목표를 달성하기 위해 협력하는 구조를 제공

NSM

  • 중요도 : ⭐⭐⭐
  • North Star Metric

기업의 장기적인 성장과 성공을 위해 가장 중요한 핵심 지표. ‘북극성’처럼 기업의 전반적인 방향성을 제시하는 지표

OMTM

  • 중요도 : ⭐⭐⭐
  • One Metric That Matters

특정 시점에서 비즈니스 성장을 위해 가장 중요한 하나의 핵심 지표. 특정 팀이나 프로젝트에 초점을 맞춘 지표

IAP

  • In App Purchase

신용카드를 통한 인앱 결제

IAA

  • In App Ads

인 앱 광고로 발생되는 매출

DL

  • Download

기간 내 다운로드 횟수

TS

  • Time Spent

게임에 방문한 이용자의 평균 체류 시간

  • 산출식 : ∑(로그아웃 시간 - 로그인 시간) / MU

문맥광고

페이지 내 컨텐츠를 분석하여 관련 키워드를 추출하고 광고를 노출하는 방식

Landing page

인터넷 사용자가 광고를 클릭하고 광고주의 사이트에 방문했을 때 최초로 보게 되는 웹 페이지

PV

  • 중요도 : ⭐⭐⭐
  • Page View

홈페이지에 들어온 접속자가 둘러 본 페이지

ASO

  • 중요도 : ⭐⭐
  • App Store Optimization

앱 스토어 최적화. 앱 아이콘, 스크린샷, 영상, 디스크립션 등 앱을 설명하는 요소를 바꾸면서 전환율을 높이는 것. 앱스토어 최적화를 통해 사용자가 앱 설치를 많이 하면, CPI가 낮아진다.

FTUE

  • 중요도 : ⭐⭐⭐
  • First Time User Experience

최초 사용자 경험

User 분류 방법

Web >> cookie 기반, App >> device_id 기반 (IDFA: ios, GAID: 안드로이드)

Monetization

인 앱 결제 또는 인 앱 광고, 구독 등을 통한 유료화

Engagement

  • 중요도 : ⭐⭐

유입 유저의 접속 빈도와 기간에 대한 지표

Funnel Analysis

  • 중요도 : ⭐⭐⭐

유입부터 결제까지 주요 단계를 수치화한 분석

F2P

  • Free To Play

게임에 접속해서 컨텐츠를 즐기는 것은 무료지만, 캐릭터를 꾸미거나 더 깊은 재미를 즐기려면 돈을 부과하는 수익 모델
