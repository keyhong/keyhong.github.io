great_expectattions 용어 정리

Great_expectations

Data Context

Great Expectations 배포의 주요 진입점으로, 모든 지원 구성 요소에 대한 구성 및 메서드를 제공.

데이터 컨텍스트는 Great Expectations (GX) 배포의 주요 진입점이며, 모든 지원 GX 구성 요소에 대한 구성 및 메서드를 제공합니다.

GX API의 주요 진입점으로서, 데이터 컨텍스트는 비타입 입력 또는 일반 기본값을 기반으로 공통 객체에 접근하는 편리한 방법을 제공합니다.

데이터 컨텍스트는 또한 상위 구성 요소를 구성할 수 있게 하며, 데이터 컨텍스트 구성을 백업하기 위해 다양한 저장 방법론을 사용할 수 있습니다.

데이터 컨텍스트를 인스턴스화하고 구성을 저장한 후에는 항상 동일하게 동작합니다.

Relationships to other objects

데이터 컨텍스트는 저장소, 플러그인 및 데이터 문서를 구성하는 메서드를 제공합니다.

또한 데이터 소스, 기대사항 및 체크포인트를 생성, 구성 및 액세스하는 데 필요한 메서드를 제공합니다.

추가로, Data Context는 메트릭, 검증 결과 및 데이터 문서의 내용을 관리하는 데 도움을 줍니다.

Use Cases

GX 환경을 구성할 때 Data Context를 인스턴스화하게 됩니다. Data Context 인스턴스화 참조.

또한 Data Context를 사용하여 저장소, 플러그인 및 Data Docs에 대한 선택적 구성을 관리할 수 있습니다. 저장소를 구성하려면 GX 환경 구성 참조. Data Docs를 호스팅하고 공유하려면 Data Docs 호스팅 및 공유 참조.

데이터에 연결할 때 Data Context를 사용하여 데이터 소스를 생성하고 구성합니다. 데이터 소스를 생성하고 구성하는 방법에 대한 자세한 내용은 데이터 소스 연결 참조.

Expectations을 생성할 때 Data Context를 사용하여 Expectations Suites과 Expectations을 생성하고, 이를 Expectations Store에 저장합니다. Data Context는 또한 메트릭과 검증 결과를 관리합니다. Data Context는 Data Docs의 내용을 관리합니다 (예: 검증 결과와 Expectations을 표시). Expectations 생성에 대한 자세한 내용은 Expectations 생성 참조.

데이터를 검증할 때 Data Context는 체크포인트를 생성, 구성, 저장 및 액세스하기 위한 진입점을 제공합니다. Data Context를 사용하여 체크포인트를 생성하는 방법에 대한 자세한 내용은 Validate Data 참조.

Access to APIs

Data Context는 GX API에 대한 주요 진입점을 제공합니다.

Data Context는 공통 객체에 접근하는 편리한 방법을 제공합니다.

GX의 내부 워크플로는 강력한 타입을 가지지만, Data Context의 편의 메서드는 예외로 비타입 입력 또는 일반 기본값을 기반으로 한 접근을 허용합니다.

Configuration management

데이터 컨텍스트는 GX 배포의 핵심 구성 요소에 대한 기본 생성, 읽기, 업데이트 및 삭제(CRUD) 작업을 포함합니다. 여기에는 데이터 소스, 기대사항 모음, 체크포인트가 포함됩니다. 또한 데이터 컨텍스트는 데이터 문서, 저장소, 플러그인 등에 접근하고 통합할 수 있게 합니다.

Component management and config storage

데이터 컨텍스트는 데이터 소스, 체크포인트, 기대사항 모음과 같은 구성 요소를 생성하고, 이러한 구성 요소에 대한 정보가 저장되는 위치를 관리하는 데 도움을 줍니다.

실제 배포를 위해서는 데이터 소스 및 생산 환경에 따라 이러한 구성 요소를 정의해야 합니다. 이는 로컬 환경이 아닌 다른 곳에 이러한 구성 요소에 대한 정보를 저장하는 것을 포함할 수 있습니다. 특정 환경 및 데이터 소스에 대한 구현 예제를 보려면 통합 참조.

GX Cloud compatibility

데이터 컨텍스트는 GX 프로젝트 전체를 포함하기 때문에, GX 클라우드는 이를 참조하여 오픈 소스 GX에서 GX 클라우드로의 원활한 업그레이드를 허용할 수 있습니다.

Instantiating a Data Context

데이터 컨텍스트를 생성한 후에는 Python에서 DataContext를 인스턴스화하여 향후 작업을 시작할 가능성이 큽니다.

예를 들어:

import great_expectations as gx

context = gx.get_context()

대안으로 특정 디렉토리를 지정하려면 context_root_dir 파라미터를 사용할 수 있습니다.

GX 클라우드를 사용하는 경우, get_context를 호출하기 전에 클라우드 환경 변수를 설정합니다.

Untyped inputs

GX의 코드 표준은 강력한 타입 입력을 지향합니다. 그러나 데이터 컨텍스트의 편의 함수는 이 표준에 대한 주목할 만한 예외입니다. 예를 들어, 타입 입력으로 배치를 얻으려면 다음 코드를 실행합니다:

from great_expectations.core.batch import BatchRequest, RuntimeBatchRequest

batch = context.get_batch_list(batch_request=batch_request)[0]

비타입 입력을 선호하는 경우 다음 예제와 유사한 코드를 실행합니다:

data_asset_name = "<your_data_asset_name>"

context.get_batch_list(
    datasource_name="taxi_datasource",
    data_connector_name="default_inferred_data_connector_name",
    data_asset_name=data_asset_name,
)

예제 코드에서 get_batch() 메서드는 의도한 타입을 추론하여 올바른 내부 메서드로 전달하는 역할을 합니다.

비타입과 타입 입력 간의 이 구분은 GX 코드베이스 내에서 중요한 아키텍처 결정 사항을 반영합니다. 내부 워크플로는 강력한 타입을 가지지만, DataContext의 일부 편의 메서드에 대해서는 예외가 허용됩니다.

강력한 타입 검사는 더 깨끗한 코드 작성과 강력한 보장 및 오류 상태에 대한 더 나은 이해를 가능하게 합니다. 또한 GX가 정적 타입 검사기, 사이클로메트릭 복잡성 분석 등의 도구를 활용할 수 있게 합니다.

타입 입력 요구는 GX 시작을 어렵게 만들 수 있습니다. 예를 들어, 이전 코드 예제에 나온 첫 번째 메서드는 BatchRequest가 무엇인지 모르는 경우 위협적일 수 있습니다. 또한 Batch Request가 great_expectations.core.batch에서 가져온다는 것을 알아야 합니다. 비타입 입력을 허용하면 GX를 훨씬 더 빠르게 시작할 수 있습니다. 그러나 비타입 입력은 혼란을 초래할 위험이 항상 존재합니다. 위험을 줄이거나 없애기 위해 GX는 다음 지침을 사용합니다:

  • 타입 추론은 보수적입니다. 타입 추론에 추측이 필요하다면, 메서드는 오류를 반환합니다.

  • 추측 없이 추론할 수 있는 대체 입력을 결정하는 데 도움을 주기 위해 유익한 오류가 제공됩니다.

Action

Validation Result를 받아서 처리하는 run 메서드를 가진 Python 클래스.

Batch

Data Asset에서 선택된 레코드들.

Batch Request

Batch를 생성하기 위해 Data Source에 제공되는 요청.

Checkpoint

Great Expectations의 실제 배포에서 데이터를 검증하는 주요 수단.

Checkpoint Store

Great Expectations의 실제 배포에서 데이터를 검증하는 수단에 대한 정보를 저장하고 검색하는 커넥터.

Custom Expectation

Great Expectations 라이브러리 외부에서 개발된 Expectation 클래스의 확장.

Data Asset

Great Expectations 라이브러리 외부에서 개발된 Expectation 클래스의 확장.

Data Assistant

데이터를 설명하기 위해 질문을 던지고, 관찰된 것을 설명하며, 답변을 기반으로 Metrics를 제시하고 Expectations를 제안하는 유틸리티.

Data Docs

Expectations, Validation Results 등을 상세히 설명하는 Great Expectations 메타데이터로 생성된 사람이 읽을 수 있는 문서.

Data Docs Store

Expectations, Validation Results 등을 상세히 설명하는 Great Expectations 메타데이터로 생성된 사람이 읽을 수 있는 문서에 대한 정보를 저장하고 검색하는 커넥터.

Data Source

다양한 소스 시스템에서 데이터를 액세스하고 상호작용하기 위한 표준 API를 제공.

Evaluation Parameter

간단한 표현식을 평가하거나 이전에 생성된 메트릭스를 참조하여 Validation 중에 사용되는 동적 값.

Evaluation Parameter Store

간단한 표현식을 평가하거나 이전에 생성된 메트릭스를 참조하여 Validation 중에 사용되는 파라미터에 대한 정보를 저장하고 검색하는 커넥터.

Execution Engine

메트릭스를 계산하기 위해 데이터를 처리할 수 있는 시스템.

Expectation

데이터에 대한 검증 가능한 주장.

Expectation Store

데이터에 대한 검증 가능한 주장 모음에 대한 정보를 저장하고 검색하는 커넥터.

Expectation Suite

데이터에 대한 검증 가능한 주장 모음.

Metric

열의 평균과 같은 데이터의 계산된 속성.

MetricProviders

Expectations를 지원하기 위해 Metrics를 생성하고 등록하며, Expectation 소프트웨어 개발 키트(SDK)의 중요한 부분.

Metric Store

열의 평균과 같은 데이터의 계산된 속성에 대한 정보를 저장하고 검색하는 커넥터.

Plugin

Great Expectations의 구성 요소 및/또는 기능을 확장.

Renderer

Expectations, Validation Results 등을 Data Docs 또는 이메일 알림이나 슬랙 메시지와 같은 다른 출력물로 변환하는 메서드.

Store

Great Expectations의 메타데이터에 대한 정보를 저장하고 검색하는 커넥터.

Supporting Resource

Great Expectations 코드베이스 외부에서 사용되는 리소스.

Validation

Expectation Suite를 Batch에 적용하는 행위.

Validation Result

데이터가 Expectation 또는 Expectation Suite에 대해 검증될 때 생성되는 결과.

Validation Result Store

데이터가 Expectation Suite에 대해 검증될 때 생성된 객체에 대한 정보를 저장하고 검색하는 커넥터.

Validator

데이터를 대상으로 Expectation Suite를 실행하는 데 사용.
