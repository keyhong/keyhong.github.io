Create a Custom Query Expectation

QueryExpectations는 SQL 및 Spark용으로 활성화된 Expectation 유형으로, ColumnAggregate, ColumnMap 및 Table과 같은 기본 Expectation 클래스와 비교하여 복잡한 워크플로우를 가능하게 합니다.

QueryExpectations를 사용하면 사용자가 직접 작성한 쿼리의 결과에 대해 기대치를 설정하고 데이터베이스에 중간 쿼리를 수행할 수 있습니다. 이 접근 방식은 데이터베이스로의 추가 왕복을 초래할 수 있지만 Custom Expectations의 고급 기능을 활용할 수 있습니다.

QueryExpectations는 쿼리의 결과를 평가하여 해당 쿼리로 반환된 데이터에 대한 의미론적 질문에 답합니다. 예를 들어, expect_queried_table_row_count_to_equal은 쿼리로 테이블에서 반환된 행 수가 얼마인지를 답합니다.

Prerequisites

  • Quickstart 완료.
  • 구성되고 기능하는 개발 환경.

Choose a name for your Expectation

먼저, 자신만의 Expectation 이름을 결정하십시오. 관례에 따라 QueryExpectations는 항상 expect_queried_로 시작합니다.

모든 QueryExpectations는 Active Batch의 매개변수화를 지원하며, 일부 QueryExpectations는 Column의 매개변수화도 지원합니다. 이 튜토리얼에서는 두 가지 접근 방식을 모두 다룹니다.

Batch Parameterization

Expectation에는 동일한 이름의 두 버전이 있습니다: CamelCaseName과 snake_case_name. 예를 들어, 이 튜토리얼에서는 사용합니다:

  • ExpectQueriedTableRowCountToBe
  • expect_queried_table_row_count_to_be

Batch & Column Parameterization

  • ExpectQueriedColumnValueFrequencyToMeetThreshold
  • expect_queried_column_value_frequency_to_meet_threshold

Copy and rename the template file

관례에 따라 각 Expectation은 Expectation의 이름을 snake_case로 변환한 이름의 파이썬 파일에 보관됩니다.

Custom QueryExpectation의 템플릿 파일은 여기에서 찾을 수 있습니다. 파일을 다운로드하여 적절한 디렉터리에 놓고 적절한 이름으로 변경하십시오.

$ cp query_expectation_template.py /SOME_DIRECTORY/expect_queried_table_row_count_to_equal.py

Storing Expectation files

개발 중에는 특정 위치에 Expectation 파일을 저장할 필요가 없습니다. Expectation 파일은 독립적이며 GX가 설치된 상태에서는 어디서든 실행할 수 있습니다. 그러나 새로운 Expectation을 다른 GX 구성 요소와 함께 사용하려면 파일을 다음 위치 중 하나에 저장해야 합니다:

  • 개인적으로 사용할 Custom Expectation을 빌드하는 경우, GX 배포의 great_expectations/plugins/expectations 폴더에 파일을 넣고 사용할 때마다 해당 디렉터리에서 Custom Expectation을 가져올 수 있습니다. 해당 DataContext를 인스턴스화하면 자동적으로 디렉터리의 모든 플러그인을 사용할 수 있습니다.

  • 오픈 소스 프로젝트에 기여하기 위해 Custom Expectation을 빌드하는 경우, Great Expectations 라이브러리 자체의 레포지토리에 파일을 넣어야 합니다. 대부분의 경우, 이는 contrib/: great_expectations/contrib/SOME_PACKAGE/SOME_PACKAGE/expectations/ 내의 패키지 내에 있을 것입니다. 이러한 Expectations를 사용하려면 해당 패키지를 설치해야 합니다.

Custom Expectations에 대한 자세한 내용은 Use a Custom Expectation을 참조하십시오.

Generate a diagnostic checklist for your Expectation

템플릿 파일을 복사하고 이름을 변경한 후 다음과 같이 실행할 수 있습니다.

$ python expect_queried_table_row_count_to_be.py

템플릿 파일은 Expectation의 print_diagnostic_checklist() 메서드를 실행하도록 설정되어 있습니다. 이는 새 Expectation에 대한 진단 스크립트를 실행하고 전체 프로덕션 준비 단계를 체크리스트로 반환합니다.

ExpectQueryToMatchSomeCriteria의 완성도 체크리스트:
  ✔ 유효한 library_metadata 객체가 있음
    단일 줄 짧은 설명을 포함한 docstring이 있음
    최소 하나의 긍정적 및 부정적 예제 케이스가 있으며 모든 테스트 케이스가 통과함
    최소 하나의 실행 엔진에서 핵심 로직이 실행되고 테스트를 통과함
    모든 린팅 검사를 통과함
    기본 입력 유효성 검사 및 타입 검사가 있음
    명령적 및 진단적 Statement Renderers를 모두 포함함
    모든 적용 가능한 실행 엔진 및 SQL 방언에 대해 핵심 로직이 테스트를 통과함
    코드 소유자가 결정한 강력한 테스트 스위트를 보유함
    코드 표준 및 스타일 가이드에 대한 코드 소유자의 수동 검토를 통과함
    체크가 없는 첫 번째 단계가 구현해야 할 다음 단계입니다. 이 가이드는 작동하는 Custom Expectation에 필요한 최소 다섯 단계를 안내하며, 이는 Experimental 수준에서 오픈 소스에 기여하는 데 필요한 전부입니다.

Change the Expectation class name and add a docstring

이제 Custom Expectation의 기능적 기초를 마련하기 시작합니다.

우선 Expectation의 이름과 docstring을 업데이트합니다.

Expectation 클래스 이름 변경

class ExpectQueryToMatchSomeCriteria(QueryExpectation):

당신의 실제 Expectation 클래스 이름으로 변경하십시오. 대문자 카멜 케이스로 작성합니다.

class ExpectQueriedTableRowCountToBe(QueryExpectation):

또한, 다음과 같이 새로운 단일 줄 docstring을 작성할 수 있습니다.

"""TODO: Add a docstring here"""

대신에 다음과 같이 작성하십시오.

"""Expect the expect the number of rows returned from a queried table to equal a specified value."""

단일 줄 docstring이 “Expect “로 시작하고 마침표로 끝나는지 확인하십시오. 또한 파일 하단의 클래스 이름을 다음과 같이 변경해야 합니다.

ExpectQueryToMatchSomeCriteria().print_diagnostic_checklist()

을 다음과 같이 변경하십시오.

ExpectQueriedTableRowCountToBe().print_diagnostic_checklist()

나중에, 보다 자세한 docstring을 작성할 수 있습니다. Expectation Docstring Formatting을 참조하세요.
