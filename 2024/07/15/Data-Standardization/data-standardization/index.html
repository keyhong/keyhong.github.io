데이터 표준화

코드, 용어, 데이터 도메인, 메타데이터 등의 표준을 수립하여 공공데이터베이스에 일관되게 적용하는 일련의 활동

데이터 표준화의 이해

데이터 표준화의 목적

데이터 표준화는 누구나 같은 의미로 이해하고 같은 방법으로 사용할 수 있는 기준을 정의함으로서 정확한 정보의 유통 및 활용을 목적으로 함

누구나 같은 의미로 이해 같은 방법으로 사용
- 전산
- 현업
- 고객		 - 의미를 명확하게 알 수 있는 명칭
- 유일한 의미를 가지는 용어		 - 데이터 표현방법
- 제약 조건

데이터 명칭 표준화 데이터 형식, 규칙 표준화
- 포함되는 문자 : 한글, 영문, 숫자, 특수문자 …
- 기본단어
- 기본단어의 배열 순서
- 주제별 명명 기준 : 조직, 고객, 매체, 연락처, 수, 금액 …
- 동음이의어, 이음동의어
- 풀어쓴 말, 줄인 말		 - 데이터 타입
- 데이터 길이
- 기본 값
- 제약조건
- 값의 범위
- 단위

데이터 표준화 구성요소

구성요소 주요내용
정책 데이터 표준화 지침(가이드) - 데이터 표준 관리 원칙, 절차, 조직, R&R
- 데이터 사전 생성 명명 규칙, 제약조건 등
- 용어 오너쉽(ownership): 생성 및 사용의 책임 소관 부서
데이터 사전 - 단어사전
- 용어사전
- 도메인사전
- 코드 사전
시스템 메타데이터 관리 시스템 - 데이터 표준 사전 정보 관리, 데이터 모델 형상 정보 관리, 데이터베이스 형상 정보 관리
- 데이터 영향도 분석

문제점 및 개선 효과

현재 데이터 운영 문제 데이터 표준화 개선 효과
각 업무영역 간 틀린 표준용어를 사용하며 커뮤니케이션에 어려움이 있다. 전사 표준용어를 사용하고 표준화되지 않은 용어도 연결하여 관리하여 커뮤니케이션 문제 해결
업무영영간 데이터 정합성 유지에 어려움이 있다. 영역 간 데이터 연관 관계를 한 눈에 파악할 수 있어 데이터 정합성 유지가 쉬워진다
각 업무영역 간 틀린 코드체게가 존재하여 통계작업을 하거나 보고서를 출력할 때 일관성이 없는 경우가 발생할 수 있다. 한 영역의 데이터 구조 변경 시 영향 분석을 통해 전 영역에 동시에 반영한다.
새로 업무를 담당하는 사람에게 보여줄 최근 데이터 구조에 관한 자료가 없다. 메타데이터 관리 시스템을 통해 전사 데이터 구조를 파악할 수 있다.
프로젝트가 생길 때 마다 데이터 명세를 정리하는 작업을 반복한다. 메타데이터 관리 시스템을 통해 항상 조회할 수 있으므로 별도로 데이터 항목 조사를 할 필요가 없다.
참조하고 있는 문서가 최종자료라는 것을 아무도 보장하지 않는다. 각자 자기가 소유한 문서만 바라본다. 모든 사람들이 항상 최근 자료를 참조하고 있다. 테이블 목록, 테이블 정의서, 코드 명세서와 같은 문서가 사라진다.

표준화 수행 방법

표준화 지침 마련

전사 차원의 데이터 표준화에 대한 정책을 수립하여 회사의 방향을 직원 간 공유하는 지침. 데이터 표준 원칙, 범위, 생성 규칙, 절차, 관리 양식 등 표준 관리 기준에 대한 상세한 가이드를 관련 부서와 협의하여 정의

주요 지침 구성

주요 지침 구성 내용
데이터 표준화 목적 - 프로젝트 개발, 유지보수 대상 오브젝트들에 대하여 명확한 명칭부여 기준을 정함으로써 프로그램의 가독성을 향상시키고 일관성을 유지하여 개발자의 생산성 향상과 인터페이스 개선 및 시스템의 품질 향상 등의 목적을 정의
데이터 표준화 대상 - 관리 대상 시스템(DB), 범정부 표준 적용 여부 등
- 표준 사전 대상 정의(단어, 용어, 도메인, 코드 등)
표준화 명명 규칙 - 단어, 용어, 도메인 생성 명명 규칙 기준
- 코드 생성 명명 규칙
- 각 사전 간의 관계 정의
데이터 관리 조직 및 주요 표준화 프로세스 정의 - 데이터 관리를 위한 조직과 조직 간(DA, 모델러, IT운영, 현업, DBA)의 R&R 정의
- 주요 표준화 데이터 생성 및 적용에 대한 운영 프로세스 수립
- 메타데이터 관리 시스템을 통한 활용 및 관리 프로세스 수립

구성 사례

데이터 관련 조직, 데이터 표준화 개요, 표준 단어, 표준 용어, 표준 도메인, 데이터 오브젝트 명명규칙, 표준화 운영 조직 및 프로세스

데이터 표준사전 구축

표준 데이터 생성 방안은 기존의 코드 표준과 현행 시스템 데이터 현황을 분석하여 표준 데이터 요소를 정의하고, 정의된 데이터 항목을 승인을 거쳐 데이터 표준으로 정함

표준 데이터 체크 리스트

  • 데이터 요소의 표준 지침에 적합한가?

  • 전사적인 코드 표준에 적합한가?

  • 동음이의어, 이음동의어?

  • 기존의 표준 코드는 반영되었는가?

기관에서 사용하고 있는 용어에 대해 모든 구성원들이 유일한 의미로 공유하기 위해 표준 데이터 사전을 생성

사전 내용
단어 사전 - 용어를 생성 조립하는 단위로 한글, 영문 정식명, 영문 약어, 단어 설명
- 이음동의어, 동음이의어, 금칙어 정의
용어 사전 - 업무 용어: 업무적으로 사용하는 용어에 대한 설명
- 표준 용어: 논리/물리 모델에 적용하는 속성/컬럼 ID
도메인 사전 - 표준 용어에 적용되는 데이터 속성(type, length) 정의 (DBMS별 논리/물리 속성 정의)
- 데이터 속성 관리를 위한 도메인 유형 정의 (~금액, ~번호, ~코드, ~수, ~일자 …)
코드 사전 - 코드컬럼ID는 표준 용어와 같이 정의 됨
- 코드컬럼ID에 코드 허용 값 범위 통합 관리 (계층 구조, 조회하면 콤보박스 Display 기준)

표준 사전 상세 내용

데이터 표준화를 위해 명확한 분류 기준을 정의하여 각 요소간 일관성을 갖춤

데이터 사전 내용
단어 테이블과 컬럼의 한글명 및 영문명을 구성하는 최소 원소를 가리키며 용어는 반드시 정의된 단어의 조합으로 구성됨
※ 하나의 단어는 동시에 용어이거나 도메인 일 수 있다.
용어생성 구성분류 기본단어 용어를 구성하는 단어(주제어, 수식어)
분류단어 용어를 생성시 최우측에 정의 할 수 있는 도메인성 단어.
도메인성 단어: 실제 데이터 값에 대한 형식(type)이나 의미를 유추 할 수 있도록 구성된 단어
단어사용 유형분류 표준단어 현재 표준단어로 사용이 허용된 단어
비표준단어 동음단어 표준단어와 유사한 의미로 정의되어 사용중인 단어(이음동의어)
금칙단어 현재 시점에 사용이 허락되지 않은 단어
용어 단어의 조합(의미가 전달 가능한 단독 단어 가능)으로 이루어져 엔터티 속성 및 테이블 컬럼의 명칭으로 사용되는 용어
도메인 속성의 일반적인 특징을 나타내는 데이터의 성격을 분류한 것으로 속성이 취할 수 있는 값의 집합
예) 코드, 금액, 일자, 수, 내용, 명 등
도메인 유형
(분류)		 도메인 관리를 위해 분류 체계를 구분함
예) 도메인대분류(명/내용/주소, 수/금액,율, 날짜, 번호, 코드)
예) 도메인소분류(명, 금액, 날짜 일련번호, 여부)
도메인명
(인포타입)		 데이터 타입과 길이를 대표하기 위해 정의한 명칭데이터 논리모델링 또는 표준용어 정의시 DBMS에 종 속적이지 않고 논리적 의미로 공통 정의된 타입
예) 금액N13, 내용V2000, 수N5
물리타입 정의된 인포타입을 물리화하기 위해 DBMS에 별로 물리TYPE 또는 LENGTH를 정의함
예) [인포타입:논리] VARCHAR() -> [물리] VARCHAR2() [Oracle]
예) [인포타입:논리] NUMBER() -> [물리] NUMBER() [Oracle]
예) [인포타입:논리] NUMBER() -> [물리] NUMERIC() [Sybase IQ]

데이터 표준사전 구축 - 표준화 상세 절차

데이터 표준화 (지침)

1.1 표준데이터 관계도

표준단어

용어를 구성하는 원소로 의미적으로 더 이상 쪼개지지 않는 최소 단위의 형태소 (단어)

  • 유사어(금칙어) + 기본단어 분류단어

도메인

데이터의 형식 및 길이를 데이터의 특성에 따라 정형화한 명칭

도메인 그룹 (예: 금액, 일자, 내용 등)
표준도메인 (예: 금액 Number[10], 금액 Number[5])

표준용어

표준단어의 조합으로 구성되고, 정형화된 데이터의 타입과 길이를 갖는 통일된 하나의 속성(컬럼) 명칭

  • 수식어[선택] + 주제어[권장] + 수식어[계약] + 분류어[필수] + 번호[선택]

** 주제어 : 업무명이나, 용어의 성격 또는 의미적 주제/객체가 되는 단어
** 분류어 : 데이터의 특성을 한정

1.2 표준데이터 명명규칙

표준단어 명명 규칙

명명규칙 설명
Only 명사형 단어 사용 동사/부사/접속사/복수형/소유격 불가
축약단어 사용 불가 주민번호(X), 주민등록번호(O)
특수문자, 띄어쓰기 불가
Only 숫자 사용 불가 1(X), 200(X), 1월(O)
접두사/접미사 단독 사용 불가 기^납입(X), 기납입(O)
고유명사는 단일어 사용 금융^결제원(X), 금융결제원(O)
영문단어 사용 원칙 - 한글화 적용 : FUND(X), 펀드(O)
- 범용적인 경우 예외 (약어, 경제/금융용어 등) : IP(O), MMK(O)

도메인 사용 규칙

  • 도메인 그룹 -> 표준용어의 분류어로 사용 (특별한 경우가 아니면, 일련번호 -> 순번 사용)
상위 도메인그룹 도메인 그룹명
금액 그룹 가격, 가액, 금액, 급여금, 보상금, 보험금, 보험료, 수수료, 연액, 원금, 전액, 적립금, 준비금, 지가, 차액, 환급금
날짜 그룹 년도, 년월, 생년월일, 시각, 시분, 월, 일, 일시, 일자
명칭 그룹 명, ID
번호 그룹 번호, 비밀번호, 순번
비율 그룹 금리, 률, 비율, 세율, 율, 이율, 환율
수량 그룹 개월수, 건수, 기간, 길이, 년수, 량, 매수, 성적, 수, 수량, 순서, 순위, 시간수, 연령, 월초성적, 일수, 점수, 주수, 지수, 차수, 평점, 학년, 회차, 횟수
연락처 그룹 우편번호, 전화번호, 주소
코드 그룹 코드, 여부, 유무

  • 표준도메인 -> 전사 표준도메인 우선 적용 (데이터 타입/길이)

    • 사례 : 금액 Number[15], 일자 Varchar[8], 사원번호 Varchar[10] 등

    • 적용할 데이터 타입/길이가 없는 경우, 수용 가능한 값 (Value)의 타입을 정하고 최대 길이 사용

표준용어 명명 규칙

  • 표준용어 구성

단어 + 단어 + … + 분류어 - 표준도메인

** 표준용어는 반드시 1개의 표준도메인과 Mapping (같은 타입/길이 표현)

  • 금액, 수 관련 용어 어순

    • 기간수식어 + 기간(시간) + 대상 + 장소 + 전(후) + 총 + 누적 + 도메인수식어 + 원화(외화) + 도메인

      • 기간수식어 : 최근, 과거, 이전, 최초, 최종 등

      • 기간(시간) : 당월, 3개월, 1일, 전년말 등

      • 대상 : 은행집계, 20세미만 등

      • 장소 : 해외, 국내

    예) 당월^20세미만^총^가입자^수

  • 전사 표준용어 우선 적용

  • 표준용어는 반드시 분류어(도메인그룹)으로 종료

  • 의미를 훼손하지 않는 범위 내에서 최대한 간결하고 명확한 표현 지향 -> 5개 이내의 단어 조합 권장

  • 표준용어 영문명은 표준단어의 영문약어에 언더바(‘_’)를 붙여서 정의

  • 표준용어 영문명 길이는 ‘_’ 포함 전체 30자 이내로 구성 (시스템 제약)

    예) 고객^마감^관리^코드 -> CUST_CLS_MGNT_CD (16자)

1.3 표준코드 생성 규칙

표준 코드 구성

성별코드

코드값 코드값명
1 남자
2 여자
  • 코드명 : 코드를 대표하는 명칭
  • 코드값 : 코드의 값을 구분하는 값 (문자, 숫자)
  • 코드값명 : 각 코드값에 대한 실제 의미

코드 명 = 수식어[필수] + 코드유형[선택] + 코드[필수]

** 코드유형 : 구분, 분류, 유형, 형태, 등급, 사유, 방법, 관계, 기준, 그룹, 단계 등

표준코드 유형

  • 일반 코드 : 공통코드 테이블에서 통합되어 관리되는 코드 -> ‘성별코드’ 등

  • 목록 코드 : 별도의 테이블에서 개별적으로 정의되어 관리되는 코드 -> ‘상품코드’ 등

    • 코드 위치 정보(테이블명, 코드값컬럼명, 코드값명컬럼명) 관리

표준코드 생성규칙

  • 코드값

    • 중복사용불가 -> [1] 남자, [1] 여자 (X)

    • 데이터 타입은 문자형을 사용 -> Char, Varchar

    • 데이터 길이는 코드값 증가를 고려하여 결정

    • 숫자 타입 코드값이 데이터 길이보다 작은 경우, 앞에서부터 문자 '0'을 채워 코드값 정의

    에시) 남자 01, 여자 02

  • 코드값명

    • 중복사용불가 -> [1] 남자, [2] 남자 (X)

    • 의미가 불분명한 코드값명 사용 금지 -> ‘NULL’ 등 (기타 등 사용 권장)

    • 스페이스바 등 특수문자 사용 지양

  • 외부공인기관에서 정의한 코드는 생성규칙 대상에서 제외

표준코드 오류 사례

예) 민원유형코드 / Varchar[3]

민원유형___ -> 분류어 누락

코드값 코드값명 문제
01 남자 표준도메인 길이 불일치
02 장애 코드값 중복, 표준도메인 길이 불일치
02 입원 코드값 중복, 표준도메인 길이 불일치
03 특이 코드값명 중복, 표준도메인 길이 불일치
4 특이 코드값명 중복, 코드값 길이 불일치, 표준도메인 길이 불일치

1.4 데이터 표준화 일정

AS-IS

  1. 표준화 교육 (2~3일)

  2. AS-IS 표준데이터(코드템플릿) 배포 (1일)

TO-BE 데이터 표준화

  1. 단어/도메인/용어 표준화 완료(코드템플릿 수집) (1달)

  • 단어/도메인/용어 표준화

    • 기 표준단어/도메인/표준용어 검색 및 적용 대상 확인 (개발자)
    • 신규 표준단어/도메인/표준용어 신청 (개발자)
    • 표준 대상 검증 및 표준화 (표준담당자)

  • 코드값 조사 및 작성 (개발팀)

    • 차세대 표준코드 100% 적용여부
    • 차세대 표준코드 부분 적용여부 (추가/변경분 코드값 작성)
    • 신규 표준코드 등록여부 (코드값 작성, 오너쉽 포함)
  1. 코드 표준화 완료 (10일)

고객 검토

  1. 자료정제 및 표준데이터 적재 (2주)

변경 관리

  1. 메타시스템 변경 관리
