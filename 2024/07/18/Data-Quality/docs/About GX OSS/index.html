About Great Expectations OSS

Great Expectations는 데이터를 검증하고 문서화하는 데 있어 최고의 도구입니다. 시작할 준비가 되었다면, Quickstart를 참고하세요.

소프트웨어 개발자들은 자동화된 테스트가 복잡한 코드베이스를 관리하는 데 필수적이라는 것을 오래전부터 알고 있었습니다. Great Expectations는 동일한 원칙, 자신감, 그리고 가속을 데이터 과학 및 데이터 엔지니어링 팀에게 제공합니다.

Why use Great Expectations?

Great Expectations를 사용하면 로드하고 변환하는 데이터에 대해 기대하는 바를 명시할 수 있으며, 데이터 문제를 빠르게 잡아낼 수 있습니다 – Expectations는 기본적으로 데이터에 대한 유닛 테스트입니다. 뿐만 아니라, Great Expectations는 이러한 Expectations를 기반으로 데이터 문서화 및 데이터 품질 보고서도 생성합니다. 데이터 과학 및 데이터 엔지니어링 팀은 Great Expectations를 다음과 같은 용도로 사용합니다:

  • 다른 팀이나 벤더로부터 받은 데이터를 테스트하고 그 유효성을 보장합니다.

  • 데이터 파이프라인의 단계로서 변환된 데이터를 검증하여 변환의 정확성을 보장합니다.

  • 데이터 품질 문제가 데이터 제품으로 유입되는 것을 방지합니다.

  • 전문가로부터 지식을 효율적으로 수집하고, 암묵적인 지식을 명시적으로 만듭니다.

  • 풍부하고 공유 가능한 데이터 문서를 개발합니다.

데이터 팀이 Great Expectations를 어떻게 사용하는지 더 알고 싶다면, Great Expectations의 사례 연구를 참고하세요.

Key Features

Expectations

Expectations는 데이터에 대한 단언입니다. Great Expectations에서는 이러한 단언이 간단하고 사람이 읽기 쉬운 Python 메서드 형태로 선언적 언어로 표현됩니다. 예를 들어, “passenger_count” 열이 1에서 6 사이의 값을 가지기를 원한다는 것을 단언하려면 다음과 같이 작성할 수 있습니다:

expect_column_values_to_be_between(
    column="passenger_count",
    min_value=1,
    max_value=6
)

그런 다음 Great Expectations는 이 명령문을 사용하여 주어진 테이블의 passenger_count 열이 실제로 1에서 6 사이에 있는지 검증하고, 성공 또는 실패 결과를 반환합니다. 이 라이브러리는 현재 수십 가지의 매우 표현력 있는 내장 Expectations를 제공하며, 사용자 정의 Expectations를 작성할 수도 있습니다.

Data validation

Expectations를 생성한 후, Great Expectations는 데이터를 일괄로 불러와서 해당 Expectations 모음으로 검증할 수 있습니다. Great Expectations는 Expectation Suite 내의 각 Expectation이 통과했는지 실패했는지를 알려주고, 테스트에 실패한 예기치 않은 값을 반환하여 데이터 문제 디버깅을 크게 가속화할 수 있습니다!

Data Docs

Great Expectations는 Expectations를 깨끗하고 사람이 읽기 쉬운 형식인 Data Docs로 렌더링합니다. 이 HTML 문서에는 검증이 실행될 때마다 Expectation Suites와 데이터 검증 결과가 모두 포함됩니다 – 이를 지속적으로 업데이트되는 데이터 품질 보고서로 생각하세요. 다음 이미지는 샘플 Data Doc을 보여줍니다:

Support for various Data Sources and Store backends

Great Expectations는 현재 Pandas 데이터프레임, Spark 데이터프레임, SQLAlchemy를 통한 SQL 데이터베이스와 같은 다양한 데이터 소스에 대해 Expectations를 네이티브로 실행하는 것을 지원합니다.

이는 데이터를 검증하기 위해 데이터베이스에 데이터를 저장할 필요가 없다는 것을 의미합니다: CSV 파일이나 데이터프레임에 로드할 수 있는 모든 데이터에 대해 Great Expectations를 실행할 수 있습니다.

Great Expectations는 매우 구성 가능성이 높습니다.

메타데이터 스토어를 구성하여 Expectations 및 검증 결과와 같은 모든 관련 메타데이터를 파일 시스템, 데이터베이스 백엔드, S3 및 Google Cloud Storage와 같은 클라우드 스토리지에 저장할 수 있습니다.

What does Great Expectations NOT do?

Great Expectations는 파이프라인 실행 프레임워크가 아닙니다.

Great Expectations는 [Airflow], [dbt], [Prefect], [Dagster], [Kedro와] 같은 DAG 실행 도구와 원활하게 통합됩니다. Great Expectations는 파이프라인을 대신 실행하지 않습니다. 대신, 검증을 파이프라인의 한 단계로 실행할 수 있습니다.

Great Expectations는 데이터 버전 관리 도구가 아닙니다.

Great Expectations는 데이터를 자체적으로 저장하지 않습니다. 대신, 데이터에 대한 메타데이터(예: Expectations, 검증 결과 등)를 다룹니다. 데이터 자체를 버전 관리하려면 [DVC], [Quilt], [lakeFS]와 같은 도구를 확인하세요.

Great Expectations는 현재 Python 환경에서 가장 잘 작동합니다.

Great Expectations는 Python 기반입니다. Python 프로그래밍 환경을 사용하지 않고도 커맨드 라인에서 호출할 수 있지만, 다른 생태계에서 작업 중이라면 다른 도구가 더 나은 선택일 수 있습니다. 순수한 R 환경에서 실행 중이라면 대안으로 [assertR]을 고려할 수 있습니다. TensorFlow 생태계 내에서는 [TFDV]가 Great Expectations와 유사한 기능을 수행합니다.

Community Resources

Great Expectations는 Great Expectations 커뮤니티를 지원하고 성장시키는 데 전념하고 있습니다. 훌륭한 도구를 만드는 것만으로는 충분하지 않습니다. Great Expectations는 훌륭한 커뮤니티도 구축하고자 합니다.

오픈 소스가 항상 친근하고 환영받는다는 평판을 가지고 있는 것은 아니며, 이는 우리를 슬프게 합니다. 모두가 오픈 소스에 속하며, Great Expectations는 여러분이 환영받는다고 느낄 수 있도록 노력하고 있습니다.
