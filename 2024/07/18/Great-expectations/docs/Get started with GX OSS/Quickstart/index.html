Quickstart

이 가이드는 GX OSS(Open Source Software)를 설치하고, 샘플 데이터에 연결하며, 첫 Expectations를 작성하고, 데이터를 검증하며, 검증 결과를 검토하는 예시이다.

GX OSS를 처음 사용하거나 자신 또는 조직에 적합한지 확신이 서지 않을 때 아래 순서로 테스트를 해보는 것이 것이 좋다. Databricks나 SQL을 사용하여 데이터를 저장하는 경우, GX와 Databricks 시작하기 또는 GX와 SQL 시작하기를 참조한다.

⚠️ WINDOWS SUPPORT
GX OSS의 오픈 소스 Python 버전에 대한 Windows 지원은 현재 제공되지 않는다. Windows 환경에서 GX OSS를 사용하는 경우 오류나 성능 문제를 겪을 수 있다.

1. Data validation workflow

다음은 GX OSS 데이터 검증 워크플로우를 보여준다. 기본 워크플로우 단계로, 이 과정을 통해 가설 설정부터 검증을 자동화하는 모든 과정이 포함된다고 생각하면 된다.

1.1 Prerequisites

  • Python 3.8 ~ 3.11 버전을 요구한다. Python 다운로드는 Python downloads에서 할 수 있다.
  • pip (Package Installer for Python). 파이썬의 패키지 매니저로, 파이썬을 설치하면 기본적으로 함께 설치된다.
  • 인터넷 브라우저. 패키지 매니저를 사용하려면 네트워크를 사용할 수 있어야 한다.

1.2 Install GX OSS

  1. Python 가상 환경 내의 빈 기본 디렉토리에서 다음 명령을 실행한다. 설치가 완료되는 데 몇 분이 걸릴 수 있다.
great_expectations 다운로드
$ pip install great_expectations

  1. Python에서 다음 코드를 실행하여 great_expectations 모듈을 가져올 수 있다.
great_expectations 라이브러리 임포트
import great_expectations as gx

1.3 Create a Data Context

다음 명령을 실행하여 Data Context 객체를 생성할 수 있다. Data Context는 great_expectations를 사용하기 위한 엔트리 포인트 객체이다.

Data Context 객체 생성
context = gx.get_context()

1.3.1 Connect to data

다음 명령을 실행하면great_expectations GitHub 리포지토리에 이미 저장된 테스트용 .csv 데이터에 연결하며, Validator 객체를 생성한다.

지정된 URL 경로의 파일에 접근하여 Validator 객체 생성
validator = context.sources.pandas_default.read_csv(
    "https://raw.githubusercontent.com/great-expectations/gx_tutorials/main/data/yellow_tripdata_sample_2019-01.csv"
)

이 코드 예제는 Pandas의 기본 Data Context, Data Source를 사용하여 지정된 URL 경로의 파일에서 .csv 데이터에 액세스한다.

1.4 Create Expectations

다음 명령을 실행하여 두 개의 Expectations를 생성한다.

2개의 Expectations(품질 검사에 대한 기대값)을 생성
validator.expect_column_values_to_not_be_null("pickup_datetime")
validator.expect_column_values_to_be_between(
    "passenger_count", min_value=1, max_value=6
)
validator.save_expectation_suite(discard_failed_expectations=False)

  • 첫 번째 Expectation은 도메인 지식을 사용한다. >> 검증 1: pickup_datetime은 null이 아니어야 한다

  • 두 번째 Expectation은 passenger_count 열과 함께 명시적으로 kwargs를 사용한다. >> 검증 2: passenger_count의 1~6 사이의 값이어야 한다.


이후 Expectation Suite에 저장한다.

2개의 Expectations이 정의된 Validator를 Expectation Suite에 저장
validator.save_expectation_suite(discard_failed_expectations=False)

Expectation Suite는 기본적으로 현재 데이터 상태를 설명하며, save_expectation_suite(...) 호출 시 실패한 Expectations는 제외된다. 그러나 discard_failed_expectations를 False로 설정하면 실패한 Expectations도 포함되어 이상적인 데이터 상태를 설명할 수 있다.

1.5 Validate data

다음 명령을 실행하여 Checkpoint를 정의하고 데이터를 검토하여 정의된 Expectations와 일치하는지 확인한다.

Great Expectations에서 Checkpoint를 생성하거나 업데이트
checkpoint = context.add_or_update_checkpoint(
    name="my_quickstart_checkpoint",
    validator=validator,
)

다음 명령을 실행하여 Data Docs를 반환한다.

Data Docs 반환
checkpoint_result = checkpoint.run()

Great Expectations를 통해 데이터 검증을 수행하고, 검증 결과를 HTML 형식으로 Data Docs에 저장하여 나중에 쉽게 확인하고 공유할 수 있다.

검증 결과를 보는 코드
context.view_validation_result(checkpoint_result)
