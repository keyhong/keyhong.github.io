Great Expectations

About Great Expectations OSS

Great Expectations는 데이터를 검증하고 문서화하는 주요 도구

처음 시작한다면, Quickstart를 참고한다.

소프트웨어 개발자들은 오랫동안 자동화된 테스트가 복잡한 코드베이스를 관리하는 데 필수적이라는 사실을 알고 있었습니다. Great Expectations는 Data Science 및 Data Engineering 팀에 동일한 discipline, confidence, acceleration를 제공한다.

1. Great Expectations를 사용하는 이유

Great Expectations를 사용하면 로드(load)하고 변환(transform)하는 데이터에서 기대(expectations)하는 것을 주장하고 데이터 문제를 빠르게 파악할 수 있다. 기대는 기본적으로 데이터에 대한 단위 테스트이다.

이뿐만 아니라 Great Expectations는 이러한 기대에서 데이터 문서화 및 데이터 품질 보고서도 작성한다. Data Science 및 Data Engineering 팀은 Great Expectations를 다음과 같은 용도로 사용할 수 있다:

  • 다른 팀이나 공급업체(Vendor)에서 수집한 데이터를 테스트하고 그 유효성을 확인한다.
  • 변환(transform)의 정확성을 보장하기 위해 데이터 파이프라인의 단계로 변환된 데이터를 검증한다.
  • 데이터 품질 문제가 데이터 제품으로 유출되는 것을 방지한다.
  • 주제 전문가의 지식 수집을 간소화하고 암묵적 지식을 명시한다.
  • 데이터에 대한 풍부한 공유 문서를 개발한다.

데이터 팀이 Great Expectations를 어떻게 사용하고 있는지 자세히 알아보려면 Great Expectations의 사례 연구를 참조할 수 있다.

2. Key Features

2.1 Expectations

Expectations는 데이터에 대한 주장이다. Great Expectations에서는 이러한 주장은 간단한 사람이 읽을 수 있는 파이썬 방법의 형태로 선언적인 언어로 표현된다.

예를 들어, “passenger_count” 열을 1에서 6 사이로 설정해야 한다고 주장하려면 다음과 같이 말할 수 있습니다:

예를 들어, “passenger_count” 열을 1에서 6 사이로 설정해야 한다고 주장하려면 다음과 같이 말할 수 있다:

"passenger_count" 열이 1~6 사이의 값을 가지기를 원하는 Expectations
expect_column_values_to_be_between(
    column="passenger_count",
    min_value=1,
    max_value=6
)

그런 다음 이 문장을 사용하여 주어진 표의 passenger_count 열이 실제로 1에서 6 사이인지 검증하고 성공 또는 실패 결과를 반환한다. 라이브러리는 현재 수십 개의 매우 표현력이 높은 기본 제공 Expectations를 제공하며 사용자 지정 기대치를 작성할 수 있다.

2.2 Data validation

Expectations를 생성한 후, Great Expectations는 모든 배치 또는 여러 배치의 데이터를 로드하여 해당 Expectations 모음으로 검증할 수 있다.

Great Expectations는 Expectation Suite 내의 각 Expectation이 합격 또는 불합격 여부를 알려주며, 테스트에 실패한 예상치 못한 값을 반환하므로 데이터 문제 디버깅이 크게 빨라질 수 있다.

2.3 Data Docs

Great Expectations는 Expectations를 깨끗하고 사람이 읽기 쉬운 형식인 Data Docs로 렌더링한다.

이 HTML 문서에는 검증이 실행될 때마다 Expectation Suites와 데이터 데이터 검증 결과가 모두 포함되어 있다. 이를 지속적으로 업데이트되는 데이터 품질 보고서로 생각하면 쉽다.

2.4 다양한 Data Sources 및 Store 백엔드 지원

Great Expectations는 현재 Pandas 데이터프레임, Spark 데이터프레임, SQLAlchemy를 통한 SQL 데이터베이스 등 다양한 데이터 소스에 대해 Expectations를 네이티브로 실행하는 것을 지원한다.

이는 데이터를 검증하기 위해 데이터베이스에 데이터를 저장할 필요가 없다는 것을 의미한다. CSV 파일이나 데이터프레임에 로드할 수 있는 모든 데이터에 대해 Great Expectations를 실행 가능하다.

Great Expectations는 매우 configurable하다. 메타데이터 스토어를 구성하여 Expectations 및 검증 결과와 같은 모든 관련 메타데이터를 파일 시스템, 데이터베이스 백엔드, S3 및 Google Cloud Storage와 같은 클라우드 스토리지에 저장 가능하다.

3. Great Expectations가 하지 않는 것

(1) Great Expectations는 파이프라인 실행 프레임워크가 아니다.

Great Expectations는 Airflow, dbt, Prefect, Dagster, Kedro와 같은 DAG 실행 도구와 원활하게 통합됨. Great Expectations는 파이프라인을 대신 실행하지 않음. 대신, validation을 파이프라인의 한 단계로 실행 가능


(2) Great Expectations는 데이터 버전 관리 도구가 아니다.

Great Expectations는 데이터를 자체적으로 저장하지 않음. 대신, 데이터에 대한 메타데이터(예: Expectations, 검증 결과 등)를 다룸. 데이터 자체를 버전 관리하려면 DVC, Quilt, lakeFS와 같은 도구를 확인해야 한다.


(3) Great Expectations는 현재 Python 환경에서 가장 잘 작동한다.

Great Expectations는 Python 기반. Python 프로그래밍 환경을 사용하지 않고도 커맨드 라인에서 호출할 수 있지만, 다른 생태계에서 작업 중이라면 다른 도구가 더 나은 선택일 수 있음. 순수한 R 환경에서 실행 중이라면 대안으로 assertR, TensorFlow 생태계 내에서는 TFDV가 Great Expectations와 유사한 기능을 수행한다.

4. Community Resources

Great Expectations는 커뮤니티를 지원하고 성장시키는 데 전념하고 있다. 훌륭한 도구를 만드는 것만으로는 충분하지 않다는 생각을 가지고, 훌륭한 커뮤니티도 구축한다.
