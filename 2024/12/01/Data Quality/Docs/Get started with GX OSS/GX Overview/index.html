Great Expectations overview

여기 제공된 정보는 Great Expectations (GX)의 신규 사용자와 그 구성 요소 및 주요 워크플로우를 이해하고자 하는 사람들을 위한 것입니다.

이 GX 개요는 GX 프로세스 및 상호작용을 제어하는 코드에 대한 깊이 있는 이해를 요구하지 않습니다.

더 고급 주제로 넘어가기 전에 시작하거나 GX 기능을 더 잘 이해하고자 할 때 이상적인 출발점입니다.

What is GX

GX는 데이터의 허용 가능한 상태를 설명하고 데이터가 그 기준을 충족하는지 검증하는 프레임워크를 제공하는 Python 라이브러리입니다.

GX core components

GX를 사용할 때 기본 객체와 프로세스를 액세스, 저장 및 관리하기 위해 다음 네 가지 핵심 구성 요소를 사용합니다:

  • Data Context: GX 프로젝트의 설정 및 메타데이터를 관리하며 GX Python API에 대한 진입점을 제공합니다.

  • Data Sources: 데이터 소스에 연결하고, 가져온 데이터를 미래 사용을 위해 조직합니다.

  • Expectations: 데이터가 따라야 할 기준을 식별합니다.

  • Checkpoints: 특정 데이터 집합에 대해 Expectations 집합을 검증합니다.

Data Context

데이터 컨텍스트는 GX 프로젝트의 설정 및 메타데이터를 관리합니다. Python에서 데이터 컨텍스트 객체는 GX API에 대한 진입점 역할을 하며, 직접 관리해야 하는 객체를 제한하기 위해 다양한 클래스를 관리합니다. 데이터 컨텍스트는 GX에서 사용되는 모든 메타데이터, GX 객체의 구성 및 데이터 검증 결과를 포함합니다.

다음은 사용할 수 있는 데이터 컨텍스트 유형입니다:

  • 일시적 데이터 컨텍스트: 메모리에 존재하며 현재 Python 세션을 넘어서 지속되지 않습니다.

  • 파일 데이터 컨텍스트: 폴더 및 구성 파일로 존재합니다. 그 내용은 Python 세션 간에 지속됩니다.

  • 클라우드 데이터 컨텍스트: Python 세션 간의 지속성을 지원하며, 추가로 Great Expectations Cloud에 대한 진입점 역할을 합니다.

자세한 내용은 데이터 컨텍스트 구성하기를 참조하세요.

The GX API

Python에서 데이터 컨텍스트 객체는 GX를 구성하고 상호작용하는 메서드를 제공합니다. 이 메서드와 이를 통해 액세스하는 객체 및 추가 메서드는 GX 공개 API를 구성합니다.

자세한 내용은 GX API 참조를 참조하세요.

Stores

스토어는 GX에서 사용하는 메타데이터를 포함합니다. 여기에는 GX 객체의 구성, GX가 데이터를 검증할 때 기록되는 정보 및 데이터 소스나 원격 환경에 액세스하는 데 사용되는 자격 증명이 포함됩니다. GX는 각 메타데이터 유형마다 하나의 스토어를 사용하며, 데이터 컨텍스트는 그 스토어가 어디에 있어야 하고 어떻게 액세스해야 하는지 GX에 알려주는 설정을 포함합니다.

자세한 내용은 GX 환경 구성하기를 참조하세요.

Data Docs

데이터 문서는 GX가 생성한 사람이 읽을 수 있는 문서입니다. 데이터 문서는 데이터가 준수해야 하는 기준과 그 기준에 따라 데이터를 검증한 결과를 설명합니다. 데이터 컨텍스트는 이 정보를 저장하고 검색하는 것을 관리합니다.

데이터 문서가 호스팅되는 위치를 구성할 수 있습니다. 스토어와 달리 여러 데이터 문서 사이트에 대한 구성을 정의할 수 있습니다. 각 데이터 문서 사이트가 제공하는 정보를 지정하여 다양한 사용 사례에 맞게 다양한 데이터 문서를 포맷하고 제공할 수 있습니다.

자세한 내용은 데이터 문서 호스팅 및 공유를 참조하세요.

Data Sources

데이터 소스는 폴더의 CSV 파일, AWS에 호스팅된 PostgreSQL 데이터베이스 또는 데이터 형식과 환경의 조합과 같은 데이터 자산에 GX를 연결합니다. 데이터 자산의 형식이나 위치에 관계없이 데이터 소스는 GX에 통합된 API를 제공합니다.

자세한 내용은 데이터 연결을 참조하세요.

Data Assets and Batches

데이터 자산은 데이터 소스 내의 레코드 컬렉션입니다. 데이터 소스는 데이터 자산에 어떻게 연결하는지 GX에 알려주고 데이터 자산은 데이터를 어떻게 구성하는지 GX에 알려줍니다. 데이터 자산의 레코드는 데이터 소스의 테이블이나 파일의 내용과 직접적으로 일치할 수 있지만 반드시 그래야 하는 것은 아닙니다. 예를 들어, SQL 데이터 소스에서 여러 테이블의 레코드를 결합하여 해당 테이블을 조인하는 단일 쿼리 데이터 자산으로 만들 수 있습니다. 파일 데이터 소스의 경우, 정규 표현식을 사용하여 특정 하위 폴더의 모든 .csv 파일의 내용을 데이터 자산으로 정의할 수 있습니다.

데이터 자산은 추가로 배치로 분할될 수 있습니다. 배치는 데이터 자산 내의 고유한 레코드 하위 집합입니다. 예를 들어, 주어진 테이블에서 작년의 모든 레코드로 구성된 SQL 데이터 소스에 데이터 자산이 있다고 가정해 봅시다. 그런 다음 해당 레코드를 해당 연도의 개별 월의 레코드에 해당하는 데이터 배치로 분할할 수 있습니다.

자세한 내용은 데이터 자산 관리하기를 참조하세요.

Batch Requests

배치 요청은 데이터 자산 내의 하나 이상의 배치를 지정합니다. 배치 요청은 GX에서 사용할 데이터를 검색하는 주요 방법입니다. 배치 요청은 여러 배치를 검색할 수 있기 때문에 단일 데이터 자산의 데이터를 활용하는 방법에 큰 유연성을 제공합니다.

예를 들어, GX는 여러 배치의 데이터를 위한 통계 분석 실행 프로세스를 자동화할 수 있습니다. 이것은 데이터 자산 내의 여러 배치에 해당하는 배치 요청을 제공할 수 있기 때문에 가능합니다. 또는 동일한 데이터 자산에서 단일 배치를 지정하여 단일 하위 집합에만 관심이 있을 때 모든 데이터에 대해 분석을 다시 실행할 필요가 없도록 할 수 있습니다.

월별로 배치로 분할된 데이터 자산의 예를 들어, 단일 배치 요청에서 모든 배치를 제공하여 기존 데이터의 각 월에 대한 통계 모델을 구축할 수 있습니다. 그런 다음 업데이트 후 단일 배치를 나타내는 배치 요청을 제공하여 최신 월의 데이터에 대해서만 분석을 실행하려고 할 수 있습니다.

자세한 내용은 데이터 자산에서 데이터 요청하기를 참조하세요.

Expectations

Expectation은 데이터에 대한 검증 가능한 주장입니다. 전통적인 Python 유닛 테스트의 어서션과 유사하게, Expectations는 예상되는 동작을 설명하기 위한 유연하고 선언적인 언어를 제공합니다. 특정 입력에 대해 코드의 예상 동작을 설명하는 전통적인 유닛 테스트와 달리, Expectations는 입력 데이터 자체에 적용됩니다. 예를 들어, 열에 null 값이 없도록 하는 Expectation을 정의할 수 있습니다. GX가 해당 Expectation을 데이터에서 실행하면 null 값이 발견되었는지 여부를 나타내는 보고서를 생성합니다.

Expectations는 전문가의 도메인 지식에서 직접 생성하거나 데이터 세트를 검사하는 동안 인터랙티브하게 생성하거나 GX가 제공하는 자동화 도구를 통해 생성할 수 있습니다.

사용 가능한 Expectations 목록은 Expectation 갤러리를 참조하세요.

자세한 내용은 Expectations 생성하기를 참조하세요.

Expectation Suites

Expectation Suites는 데이터를 설명하는 Expectations 모음입니다. GX가 데이터를 검증할 때, Expectation Suite는 포함된 모든 Expectations를 해당 데이터에 대해 실행하여 프로세스를 간소화합니다. 거의 모든 경우, Expectation을 생성할 때 Expectation Suite 객체 내부에서 생성하게 됩니다.

다양한 사용 사례를 다루기 위해 동일한 데이터에 대해 여러 Expectation Suites를 정의할 수 있습니다. 예를 들어, 원시 데이터에 대한 하나의 Expectation Suite와 동일한 데이터의 후처리에 대한 더 엄격한 Expectation Suite를 가질 수 있습니다. Expectation Suite는 특정 데이터 소스와 분리되어 있기 때문에 동일한 Expectation Suite를 여러 개의 서로 다른 데이터 소스에 적용할 수 있습니다. 예를 들어, 이전 데이터 세트를 기반으로 생성된 Expectation Suite를 재사용하여 새로운 데이터 세트의 품질을 검증할 수 있습니다.

자세한 내용은 Expectations 및 Expectation Suites 생성 및 관리하기를 참조하세요.

Data Assistants

데이터 어시스턴트는 데이터에 대한 질문을 통해 정보를 수집하고 관찰된 내용을 설명하며, 결정된 답변을 기반으로 메트릭 및 제안된 Expectations를 제시하여 Expectations 생성 프로세스를 자동화하는 도구입니다. 이것은 제공된 데이터에 대한 Expectations 생성 프로세스를 가속화할 수 있습니다.

자세한 내용은 데이터 어시스턴트를 참조하세요.

Custom Expectations

GX에는 50개 이상의 일반적인 Expectations가 포함된 내장 라이브러리가 포함되어 있습니다. 그러나 데이터 검증을 위해 더 전문화된 것이 필요한 경우 사용자 정의 Expectations를 생성할 수도 있습니다. 다른 GX 커뮤니티 구성원이 기여한 사용자 정의 Expectations를 플러그인으로 설치하여 사용할 수도 있습니다.

자세한 내용은 다음을 참조하세요:

  • 사용자 정의 Expectations 생성

  • 사용자 정의 Expectations 사용

Checkpoints

체크포인트는 GX의 운영 배포에서 데이터를 검증하는 주요 수단입니다. 체크포인트는 데이터 배치(또는 배치들)와 Expectation Suite(또는 여러 개)를 번들로 묶고, 그 Expectations를 해당 데이터에 대해 실행하는 추상화를 제공합니다.

자세한 내용은 데이터 검증하기를 참조하세요.
