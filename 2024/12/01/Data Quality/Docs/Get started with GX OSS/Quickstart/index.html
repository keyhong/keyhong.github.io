Quickstart

이 빠른 시작 가이드를 사용하여 GX OSS를 설치하고, 샘플 데이터에 연결하며, 첫 Expectations를 작성하고, 데이터를 검증하며, 검증 결과를 검토하십시오. GX OSS를 처음 사용하거나 자신 또는 조직에 적합한지 확신이 서지 않을 때 이곳에서 시작하는 것이 좋습니다. Databricks나 SQL을 사용하여 데이터를 저장하는 경우, GX와 Databricks 시작하기 또는 GX와 SQL 시작하기를 참조하세요.

⚠️ WINDOWS SUPPORT
GX OSS의 오픈 소스 Python 버전에 대한 Windows 지원은 현재 제공되지 않습니다. Windows 환경에서 GX OSS를 사용하는 경우 오류나 성능 문제를 겪을 수 있습니다.

Data validation workflow

다음 다이어그램은 이 빠른 시작 가이드를 통해 구현할 GX OSS 데이터 검증 워크플로우를 보여줍니다. 워크플로우 단계를 클릭하여 관련 내용을 확인하세요.

Prerequisites

  • Python 3.8에서 3.11 버전 설치. Python 다운로드 및 설치는 Python downloads를 참조하세요.
  • pip
  • 인터넷 브라우저

Install GX OSS

  1. Python 가상 환경 내의 빈 기본 디렉토리에서 다음 명령을 실행하세요:
$ pip install great_expectations

설치가 완료되는 데 몇 분이 걸릴 수 있습니다.

  1. 다음 Python 코드를 실행하여 great_expectations 모듈을 가져옵니다:
import great_expectations as gx

Create a Data Context

  • 다음 명령을 실행하여 Data Context 객체를 생성합니다:
context = gx.get_context()

Connect to data

  • 다음 명령을 실행하여 great_expectations GitHub 리포지토리에 저장된 기존 .csv 데이터에 연결하고 Validator 객체를 생성합니다:
validator = context.sources.pandas_default.read_csv(
    "https://raw.githubusercontent.com/great-expectations/gx_tutorials/main/data/yellow_tripdata_sample_2019-01.csv"
)

이 코드 예제는 Pandas의 기본 Data Context Data Source를 사용하여 지정된 URL 경로의 파일에서 .csv 데이터에 액세스합니다.

Create Expectations

  • 다음 명령을 실행하여 두 개의 Expectations를 생성하고 Expectation Suite에 저장합니다:
validator.expect_column_values_to_not_be_null("pickup_datetime")
validator.expect_column_values_to_be_between(
    "passenger_count", min_value=1, max_value=6
)
validator.save_expectation_suite(discard_failed_expectations=False)

첫 번째 Expectation은 도메인 지식을 사용합니다 (pickup_datetime은 null이 아니어야 합니다).

두 번째 Expectation은 passenger_count 열과 함께 명시적 kwargs를 사용합니다.

Expectation Suite를 생성할 때 기본 워크플로우는 관련 데이터의 상태를 정확하게 설명하는 Expectations로 채우는 것입니다. 따라서 Expectation Suite가 저장될 때 기본적으로 실패한 Expectations는 유지되지 않습니다. 그러나 save_expectation_suite(...)의 discard_failed_expectations 매개변수를 사용하여 현재 상태가 아닌 데이터의 이상적인 상태를 설명하는 Expectations를 생성한 경우 이 동작을 무시할 수 있습니다.

Validate data

  1. 다음 명령을 실행하여 Checkpoint를 정의하고 데이터를 검토하여 정의된 Expectations와 일치하는지 확인합니다:
checkpoint = context.add_or_update_checkpoint(
    name="my_quickstart_checkpoint",
    validator=validator,
)
  1. 다음 명령을 실행하여 Data Docs를 반환합니다:
checkpoint_result = checkpoint.run()
  1. 다음 명령을 실행하여 생성된 Data Docs에서 검증 결과의 HTML 표현을 봅니다:
context.view_validation_result(checkpoint_result)
