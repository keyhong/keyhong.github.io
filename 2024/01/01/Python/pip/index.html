[Python] 파이썬 pip 설치 경로 이해하기

Publish Date:   2024-01-01
Python

Pip이란?

“Pip Installs Packages”의 약자로 Python 패키지를 설치하고 관리하는 패키지 관리자입니다.

Pip으로 특정경로에 패키지 설치

$ pip install -target=[경로] [라이브러리명]

# example
$ pip install -target=/usr/.local/lib/python3.8/site-packages sqlalchemy

app 내부에서 import 가능한 패키지 보기

# 리스트 형식의 패키지와 버전을 이쁘게 보기 위해 사용
from pprint import pprint

# 파이썬의 내장 모듈. working_set의 reuturn type은 &lt;class 'pkg_resources.WorkingSet' &gt;. 이를 리스트로 변환 필요하다.
from pkg_resources import working_set
pprint(list(working_set))

# 출력 
# numpy 1.21.6 (/home/user/b2en/.anaconda3/lib/python3.7/site-packages)

# 해석 : [패키지] [버전] ([패키지 경로])

# pip install [패키지를 하면] 기본적인 설치경로는 anaconda 디렉토리의 lib/python<version>/site-packages 이다.

꿀팁!

가끔 pip uninstall이라는 패키지 삭제 명령어가 안 먹힐 때가 있다. 파이썬 패키지가 어디서 불러오는 지 과정을 알면 직접 패키지를 수동으로 지워주워 주어 삭제할 수 있다.

site-package 위치 찾기

다음 커맨드를 치면 현재 사용중인 파이썬(가상환경을 쓰고 있다면 가상환경으로 반드시 접속해야 합니다.)이 패키지를 임포트하는 경로들이 나오게 됩니다. 그 중 site-package라는 단어가 있는 경로로 접속해주세요.

$ python -m site

그럼 무수한 폴더들이 나오는 데, 각 폴더의 이름은 하나의 패키지를 의미합니다. 그 폴더를 지우시면 패키지가 삭제됩니다.

가끔 파이썬 패키지 중에는 바이너리(bin) 파일을 가진 패키지들이 있습니다. 예를 들면 airflow, dbt와 같이 CLI로 실행이 가능한 라이브러리 입니다. 이런 패키지들을 수동으로 제거할 때는 바이너리 파일이 별도의 폴더에 저장되기 때문에 같이 제거해주셔야 합니다. 바이너리 파일의 위치는 아래 명령어로 찾을 수 있습니다.

$ which <실행 가능한 패키지>

# example
$ which airflow
