require-dbt-version

Publish Date:   2024-01-01
DBT

Configuring quoting in projects

dbt_project.yml
require-dbt-version: version-range | [version-range]

Definition

require-dbt-version을 사용하여 프로젝트가 특정 dbt 버전 범위에서만 작동하도록 제한할 수 있습니다.

이 설정을 사용하면, 지원되지 않는 dbt 버전으로 프로젝트를 실행하려는 사용자에게 유용한 오류 메시지가 dbt에서 전송됩니다. 이는 패키지 유지자(dbt-utils와 같은)가 사용자의 dbt 버전이 패키지와 호환되는지 확인하는 데 유용할 수 있습니다. 이 설정을 사용하면 팀 전체가 로컬 개발에서 동일한 dbt 버전을 사용하여 동기화를 유지할 수 있어서 변경된 동작으로 인한 호환성 문제를 방지할 수 있습니다.

만약 이 구성이 지정되지 않으면, 버전 검사가 수행되지 않습니다.

YAML quoting

이 구성은 YAML 파서에 의해 문자열로 보간되어야 합니다. 따라서 구성의 값을 따옴표로 묶어야 하며, 공백을 피해야 합니다. 예를 들어:

dbt_project.yml
# ✅ These will work
require-dbt-version: ">=1.0.0" # Double quotes are OK
require-dbt-version: '>=1.0.0' # So are single quotes

# ❌ These will not work
require-dbt-version: >=1.0.0 # No quotes? No good
require-dbt-version: ">= 1.0.0" # Don't put whitespace after the equality signs

Examples

Specify a minimum dbt version

다음 예에서는 최소 경계를 지정하기 위해 >= 연산자를 사용하여, 이 프로젝트가 1.0.0 이상의 모든 dbt 버전과 함께 실행되도록 설정합니다:

dbt_project.yml
require-dbt-version: ">=1.0.0"

Pin to a range

다음 예에서는 상한 및 하한 경계를 설정하기 위해 쉼표로 구분된 목록을 사용하여, 이 프로젝트가 dbt 1.x.x 버전과 함께 실행되도록 설정합니다:

dbt_project.yml
require-dbt-version: [">=1.0.0", "<2.0.0"]

또는 

dbt_project.yml
require-dbt-version: ">=1.0.0,<2.0.0"

Require a specific dbt version

다음 예에서는 프로젝트가 dbt v1.5 버전에서만 실행되도록 설정하는 방법입니다:

dbt_project.yml
require-dbt-version: 1.5

Invalid dbt versions

프로젝트 또는 포함된 패키지에서 지정된 require-dbt-version과 호출된 dbt 버전이 일치하지 않으면, dbt는 즉시 다음과 같은 오류로 실패합니다:

$ dbt compile
Running with dbt=0.21.0
Encountered an error while reading the project:
Runtime Error
  This version of dbt is not supported with the 'my_project' package.
    Installed version of dbt: =0.21.0
    Required version of dbt for 'my_project': ['>=1.0.0', '<2.0.0']
  Check the requirements for the 'my_project' package, or run dbt again with --no-version-check

Disabling version checks

호환되지 않는 dbt 버전으로 인한 실패를 억제하려면 dbt run 실행 시 --no-version-check 플래그를 제공하십시오.

$ dbt run --no-version-check
Running with dbt=0.21.0
Found 13 models, 2 tests, 1 archives, 0 analyses, 204 macros, 2 operations....

자세한 사용법은 global configs을 참조하세요.

Recommendation

  • 이 구성은 권장됩니다.
  • v1 이전에는 필요한 dbt 버전을 마이너 릴리스에 고정해야 합니다. v1 이후에는 메이저 릴리스에 고정해야 합니다 (위 example 참조).
