query-comment

DBT

query-comment

dbt_project.yml
query-comment: string

query-comment 구성은 다음과 같이 딕셔너리 입력도 허용합니다:

dbt_project.yml
models:
  my_dbt_project:
    +materialized: table

query-comment:
  comment: string
  append: true | false
  job-label: true | false  # BigQuery only

Definition

dbt가 데이터베이스에 대해 실행하는 각 쿼리에 주석으로 삽입할 문자열입니다. 이 주석은 모델 및 테스트와 같은 특정 dbt 리소스에 SQL 문을 할당하는 데 사용될 수 있습니다.

query-comment 구성은 문자열을 반환하는 매크로를 호출할 수도 있습니다.

기본적으로 dbt는 실행하는 리소스의 정보를 포함하는 JSON 주석을 쿼리 맨 위에 삽입합니다. 이 정보에는 dbt 버전, 프로필 및 타겟 이름, 그리고 노드 ID 등이 포함됩니다. 예를 들어:

/* {"app": "dbt", "dbt_version": "0.15.0rc2", "profile_name": "debug",
    "target_name": "dev", "node_id": "model.dbt2.my_model"} */

create view analytics.analytics.orders as (
    select ...
  );

Using the dictionary syntax

딕셔너리 구문은 두 개의 키를 포함합니다:

  • comment(선택 사항, 기본값은 위의 설명 참조): 쿼리에 주석으로 삽입될 문자열입니다.
  • append(선택 사항, 기본값=false): 주석을 쿼리의 맨 아래에 추가할지 여부를 지정합니다. 기본적으로 주석은 쿼리의 맨 위에 추가됩니다 (append: false).

이 구문은 Snowflake와 같은 데이터베이스에서 유용합니다. Snowflake는 보통 remove leading SQL comments 때문에 이 구성을 사용하여 주석을 원하는 위치에 삽입할 수 있습니다.

Examples

Prepend a static comment

이 설정은 dbt가 실행하는 모든 SQL 쿼리의 맨 위에 주석으로 /* executed by dbt */를 추가합니다. append: false로 설정되어 있으므로 주석은 쿼리의 맨 위에 삽입됩니다.

dbt_project.yml
models:
  my_dbt_project:
    +materialized: table

query-comment:
  comment: string
  append: true | false
  job-label: true | false  # BigQuery only

Example output:

/* executed by dbt */

select ...

Disable query comments

dbt_project.yml
query-comment:

Or:

dbt_project.yml
query-comment: null

Prepend a dynamic comment

다음 예제는 활성화된 dbt 타겟에서 구성된 user에 따라 달라지는 주석을 삽입하는 방법을 보여줍니다.

dbt_project.yml
query-comment: "run by {{ target.user }} in dbt"

Example output:

/* run by drew in dbt */

select ...

Append the default comment

다음 예제는 딕셔너리 구문을 사용하여 기본 주석을 추가(선행하지 않음)하는 방법을 보여줍니다.

comment: 필드가 생략되어 기본값이 추가될 수 있도록 합니다.

dbt_project.yml
query-comment:
  append: True

Example output:

select ...
/* {"app": "dbt", "dbt_version": "0.16.`0rc2`", "profile_name": "debug", "target_name": "dev", "node_id": "model.dbt2.my_model"} */
;

BigQuery: include query comment items as job labels

만약 query-comment.job-label이 true로 설정되면, dbt는 실행하는 쿼리에 대해 query comment 항목들(만약 딕셔너리로 지정된 경우) 또는 comment 문자열을 job label로 포함시킵니다. 이 레이블들은 BigQuery-specific config에서 지정된 레이블들에 추가하여 포함됩니다.

dbt_project.yml
query-comment:
  job-label: True

Append a custom comment

다음 예제는 활성 dbt 타겟에서 구성된 user에 따라 달라지는 주석을 추가하는 데 딕셔너리 구문을 사용합니다.

dbt_project.yml
query-comment:
  comment: "run by {{ target.user }} in dbt"
  append: True

Example output:

select ...
/* run by drew in dbt */
;

Intermediate: Use a macro to generate a comment

query-comment 구성은 dbt 프로젝트 내의 매크로를 참조할 수 있습니다. 단순히 macros 디렉터리에 원하는 이름의 매크로(query_comment이 좋은 시작점입니다!)를 생성하면 됩니다.

macros/query_comment.sql
{% macro query_comment() %}

  dbt {{ dbt_version }}: running {{ node.unique_id }} for target {{ target.name }}

{% endmacro %}

그런 다음 dbt_project.yml 파일에서 해당 매크로를 호출하면 됩니다. YAML 파서가 {를 딕셔너리의 시작으로 해석하지 않도록 매크로를 따옴표로 묶어야 합니다.

dbt_project.yml
query-comment: "{{ query_comment() }}"

Advanced: Use a macro to generate a comment

다음 예제는 JSON 쿼리 주석을 보여줍니다. 이를 통해 dbt 프로젝트의 성능 특성을 이해할 수 있습니다.

macros/query_comment.sql
{% macro query_comment(node) %}
    {%- set comment_dict = {} -%}
    {%- do comment_dict.update(
        app='dbt',
        dbt_version=dbt_version,
        profile_name=target.get('profile_name'),
        target_name=target.get('target_name'),
    ) -%}
    {%- if node is not none -%}
      {%- do comment_dict.update(
        file=node.original_file_path,
        node_id=node.unique_id,
        node_name=node.name,
        resource_type=node.resource_type,
        package_name=node.package_name,
        relation={
            "database": node.database,
            "schema": node.schema,
            "identifier": node.identifier
        }
      ) -%}
    {% else %}
      {%- do comment_dict.update(node_id='internal') -%}
    {%- endif -%}
    {% do return(tojson(comment_dict)) %}
{% endmacro %}

위와 같이 이 매크로를 호출하세요:

dbt_project.yml
query-comment: "{{ query_comment(node) }}"

Compilation context

쿼리 주석을 생성할 때 사용할 수 있는 다음 컨텍스트 변수들이 있습니다:

Context Variable Description
dbt_version 사용 중인 dbt의 버전입니다. 릴리즈 버전 관련 자세한 내용은 Versioning 문서를 참조하세요.
env_var env_var를 참조하세요.
modules modules를 참조하세요.
run_started_at dbt 호출이 시작된 시점
invocation_id dbt 호출의 고유 ID
fromjson fromjson를 참조하세요.
tojson tojson를 참조하세요.
log log를 참조하세요.
var var를 참조하세요.
target target를 참조하세요.
connection_name 연결의 내부 이름을 나타내는 문자열입니다. 이 문자열은 dbt에 의해 생성됩니다.
node 파싱된 노드 객체의 사전 표현입니다. node.unique_id, node.database, node.schema 등을 사용할 수 있습니다.
