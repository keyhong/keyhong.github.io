dbt_project.yml

모든 dbt project에는 dbt_project.yml 파일이 필요합니다. 이 파일은 dbt가 디렉토리를 dbt 프로젝트로 인식하는 방법입니다. 또한 dbt가 프로젝트를 운영하는 방법을 알려주는 중요한 정보를 포함하고 있습니다.

dbt는 몇 군데에서 YAML을 사용합니다. YAML이 처음이라면 배열, 사전 및 문자열이 어떻게 표현되는지 학습하는 것이 좋습니다.

기본적으로 dbt는 현재 작업 디렉토리와 해당 상위 디렉토리에서 dbt_project.yml을 찾지만, --project-dir 플래그나 DBT_PROJECT_DIR 환경 변수를 사용하여 다른 디렉토리를 설정할 수 있습니다.

dbt v1.5 이상에서는 dbt_project.yml 파일의 dbt-cloud config 아래에 project-id를 사용하여 dbt Cloud 프로젝트 ID를 지정할 수 있습니다. 프로젝트 ID를 확인하려면 dbt Cloud 프로젝트 URL을 확인하세요. 예: https://cloud.getdbt.com/11/projects/123456에서 프로젝트 ID는 123456입니다.

참고할 점은 dbt_project.yml 파일에서 config가 아닌 경우(예: macros), “property”을 설정할 수 없습니다. 이는 모든 종류의 리소스에 적용됩니다. 자세한 내용은 Configs and properties를 참조하세요.
