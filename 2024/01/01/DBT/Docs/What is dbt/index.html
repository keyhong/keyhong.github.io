Publish Date:   2024-01-01
DBT

What is dbt?

DBT는 더 많은 작업을 수행하면서 더 높은 품질의 결과를 얻는 데 도움이 되는 변환 워크플로우입니다. DBT를 사용하여 분석 코드를 모듈화하고 중앙 집중화할 수 있으며, 동시에 데이터 팀에 소프트웨어 엔지니어링 워크플로우에서 일반적으로 찾을 수 있는 가드레일을 제공할 수 있습니다. 프로덕션 환경에 안전하게 배포하기 전에 데이터 모델을 협업하고 버전 관리하며, 쿼리를 테스트하고 문서화하고 모니터링하고 시각화할 수 있습니다.

DBT는 분석 코드를 데이터 플랫폼에 컴파일하고 실행하여 여러분과 팀이 지표, 통찰력 및 비즈니스 정의에 대한 단일 진실의 원천에 대해 협업할 수 있도록 합니다. 이 단일 진실의 원천은 데이터에 대한 테스트를 정의할 수 있는 능력과 결합되어, 논리 변경 시 오류를 줄이고 문제가 발생할 때 경고를 제공합니다.

The dbt Viewpoint에서는 왜 분석가가 소프트웨어 엔지니어처럼 더 많은 작업을 할 수 있는지에 대해 자세히 알아볼 수 있습니다. 또한 joining the dbt Community하여 전 세계의 다른 데이터 실무자들이 dbt를 어떻게 활용하는지도 알아볼 수 있습니다.

dbt products

DBT를 사용하여 데이터를 신속하게 협업적으로 변환하고 버전 관리, 모듈화, 이식성, CI/CD 및 문서화와 같은 소프트웨어 엔지니어링 최상의 실천 방법을 따르는 분석 코드를 배포할 수 있습니다. 이는 SQL에 익숙한 데이터 팀의 누구나 안전하게 프로덕션 수준의 데이터 파이프라인에 기여할 수 있다는 것을 의미합니다.

dbt Cloud

DBT Cloud는 dbt를 배포하는 가장 빠르고 신뢰할 수 있고 확장 가능한 방법을 제공합니다. 데이터 팀이 웹 기반 사용자 인터페이스(UI)를 통해 단일 완전 관리형 서비스를 사용하여 데이터 모델을 개발, 테스트, 스케줄링 및 조사하여 데이터 변환을 최적화할 수 있습니다.

www.getdbt.com에서 요금제와 가격에 대해 알아볼 수 있습니다.

dbt Cloud features에 대해 자세히 알아보고 dbt Cloud quickstarts 중 하나를 시도해보세요.

dbt Core

dbt Core는 데이터 전문가가 데이터를 변환할 수 있게 해주는 오픈 소스 도구로, dbt를 수동으로 설정하고 로컬로 유지하는 사용자에게 적합합니다. 명령 줄을 통해 install dbt Core 할 수 있습니다. quickstart for dbt Core로 더 자세히 알아보세요.

dbt Cloud와 dbt Core에 대한 자세한 정보는 How dbt Cloud compares with dbt Core를 참조하십시오.

dbt optimizes your workflow

  • 보일러플레이트 DML 및 DDL 작성을 피하기 위해 트랜잭션 관리, 테이블 삭제 및 스키마 변경을 관리합니다. 필요한 데이터셋을 반환하는 SQL select문 또는 Python DataFrame으로 비즈니스 로직을 작성하고, dbt가 materialization를 처리합니다.

  • 각 분석마다 원시 데이터에서 시작하는 대신에 재사용 가능하거나 모듈식인 데이터 모델을 작성하여 이후의 작업에서 참조할 수 있습니다.

  • 쿼리 실행 시간을 현저히 단축하세요: 메타데이터를 활용하여 최적화하고자 하는 실행 시간이 긴 모델을 찾고, dbt가 쉽게 구성하고 사용할 수 있도록 만든 incremental models을 활용하세요.

  • macros, hooks 및 package management를 활용하여 코드를 DRY하게 작성하세요.

dbt provides more reliable analysis

  • SQL을 복사하여 붙여넣는 것은 논리가 변경될 때 오류를 발생시킬 수 있습니다. 대신, 재사용 가능한 데이터 모델을 구축하여 이후의 모델 및 분석에 활용하세요. 한 번 모델을 변경하면 해당 변경 사항이 모든 종속 항목으로 전파됩니다.

  • 특정 데이터 모델의 정식 버전을 발행하여 모든 복잡한 비즈니스 로직을 캡슐화합니다. 이 모델을 기반으로 한 모든 분석은 비즈니스 로직을 재구현할 필요 없이 동일한 비즈니스 로직을 통합할 것입니다.

  • branching, pull requests 및 code reviews와 같은 성숙한 소스 제어 프로세스를 활용하세요.

  • 기본 데이터에 대해 빠르고 쉽게 데이터 품질 테스트를 작성하세요. 많은 분석 오류는 데이터의 엣지 케이스에서 발생합니다. 테스트는 분석가가 이러한 엣지 케이스를 찾고 처리하는 데 도움이 됩니다.

The power of dbt

dbt 사용자로서 주요 관심사는 핵심 비즈니스 로직을 반영하는 모델(예, select 쿼리)을 작성하는 것입니다 - 테이블 및 뷰를 생성하거나 모델의 실행 순서를 정의하는 보일러플레이트 코드를 작성할 필요가 없습니다. 대신, dbt가 이러한 모델을 데이터 웨어하우스 객체로 변환하는 작업을 처리합니다.

Feature Description
질의를 관계로 형성하기 위한 보일러플레이트 코드 처리 생성한 각 모델에 대해 쉽게 materialization를 구성할 수 있습니다. materialization는 select 쿼리의 빌드 전략을 나타냅니다 - materialization 뒤의 코드는 select 쿼리를 새 관계를 생성하거나 기존 관계를 업데이트하는 문으로 감싼 견고한 보일러플레이트 SQL입니다. Materializations를 참조하세요.
코드 컴파일러 사용 SQL 파일에는 가벼운 템플릿 언어인 Jinja를 포함 할 수 있습니다. SQL에서 Jinja를 사용하면 질의에서 제어 구조를 사용할 수 있는 방법을 제공합니다. 예를 들어, if 문과 for 루프가 있습니다. 또한 macros를 통해 반복되는 SQL을 공유할 수 있도록 합니다. Macros에 대해 더 읽어보세요.
모델 실행 순서 결정 데이터를 변형할 때 종종 단계별 접근이 합리적입니다. dbt는 ref function를 통해 변형을 단계별로 구현할 수 있는 메커니즘을 제공합니다. 데이터 웨어하우스의 기존 테이블 및 뷰를 select 하는 대신 다른 모델에서 select 할 수 있습니다.
dbt 프로젝트 문서화 dbt Cloud에서는 dbt 프로젝트가 실행될 때 문서를 자동으로 생성할 수 있습니다. dbt는 dbt 모델의 문서를 작성하고 버전 관리하며 공유하는 메커니즘을 제공합니다. 각 모델 및 필드에 대한 설명(plain text 또는 markdown)을 작성할 수 있습니다. Documentation에 대해 자세히 알아보세요.
모델 테스트 테스트는 각 모델에서 생성된 결과에 대한 주장을 설정함으로써 각 모델의 SQL의 무결성을 향상시키는 방법을 제공합니다. 프로젝트를 버튼 클릭으로 또는 dbt Cloud IDE 명령 창을 사용하여 빌드, 테스트 및 실행할 수 있습니다. models Testing에 대한 테스트 작성에 대해 더 읽어보세요.
패키지 관리 dbt에는 패키지 관리자가 함께 제공되며, 이를 통해 분석가들은 공개 및 비공개 dbt 코드 저장소를 사용하고 게시할 수 있으며, 다른 사람들이 참조할 수 있습니다. Package Management에 대해 자세히 알아보세요.
시드 파일 로드 분석 작업에서는 종종 raw 값이 더 읽기 쉬운 값으로 매핑(예: 국가 코드를 국가 이름으로 변환) 되어야 하거나 정적 또는 드물게 변경되는 데이터로 보강되어야 합니다. 이러한 데이터 소스를 시드 파일이라고하며, project의 CSV 파일로 저장할 수 있으며 seed 명령을 사용하여 데이터 웨어하우스로 로드할 수 있습니다. Seeds에 대해 더 읽어보세요.
데이터 스냅샷 데이터 원본의 레코드는 시간이 지남에 따라 변경될 수 있는 경우가 많습니다. 이를 분석에서 과거 값을 재구성하려면 처리하기가 어려울 수 있습니다. dbt는 snapshots을 통해 특정 시점에 원시 데이터를 스냅샷으로 캡처하는 메커니즘을 제공합니다.
