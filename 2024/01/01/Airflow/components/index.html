What is Airflow?

작업을 올바른 방식으로, 올바른 순서로, 올바른 시간에 실행하는 오픈소스 오케스트레이터

  • Kafka, Spark, Hive 등 영러 플랫폼들과 상호작용할 수 있습니다.

  • 파이썬으로 구현되어 코드 작성이 다이나믹합니다. 언어의 특성상 자유도가 높지만, 여러 명에서 다중 작업시 코드의 통일성이 떨어질 수도 있는 단점이 있습니다.

  • 플러그인을 생성할 수 있어, operator에 기능을 추가하여 다양한 기능을 구현할 수 있도록 확장 가능합니다.

  • Flask로 구현된 Airflow Web UI와 API가 잘 구현 되어 있습니다.

What airflow is not?

Not a data streaming solution neither a data processing framework

  • Airflow는 Batch를 위한 스케줄링 프로그램입니다. Streaming 데이터 처리는 Spark, Storm, Flink와 같은 플랫폼에서 처리해야 합니다.

Core Components

Web server

Flask server with Gunicon serving the UI

Scheduler

Daemon in charge of scheduling workflows

  • 작업 스케줄링과 데이터 파이프라인 스케줄링을 담당합니다.

Metastore

Database where metadata are stored

  • 보통은 MySQL이나 Postgres를 사용합니다.

Executor

태스크(task)가 실행되는 매커니즘. 작업의 한 단위인 태스크 인스턴스를 실행하는 주체

  • 작업이 실행되는 방식을 정의하는 클래스, 작업이 실행되는 시스템입니다.

  • 클러스터 환경에서 어떤 시스템에서 작업이 실행될지를 정의합니다.

Executor의 종류

  • Sequential Executor

단일 장비에 한번에 하나의 태스크만 프로세스로 실행할 수 있는 기본 실행 형태

  • 기본 DB인 SQLite와 함께 사용가능 합니다.

  • SQLite가 멀티 유저를 위한 데이터베이스가 아니기 때문에, 동시 접근 불가능하다는 점에서 Task를 병렬로 돌릴 수 없어 실제로는 거의 쓰이지 않습니다.

  • Local Executor

단일 장비에 웹 서버와 스케줄러를 같이 기동하고 태스크를 프로세스 형태로 스폰(spawn)해 실행하는 형태

  • Task 를 병렬로 실행 가능한 Executor 입니다.

  • 옵션 self.parallelism 설정을 통해 최대 병렬처리 수를 설정 가능합니다.

  • 설정값이 0이 아닌 경우, 태스크를 설정값 만큼 병렬처리 가능합니다. (Limited Parallelism, 워커의 구현체는 LocalWorker 클래스)

  • 설정값이 0인 경우, 태스크를 제한없이 병렬처리가 가능합니다. (Unlimited Parallelism, 워커의 구현체는 QueuedLocalWorker 클래스)

  • 단일 장비 환경에서 작동하기 때문에 SPOF(Single point of failure, 단일실패지점) 문제가 있습니다.

  • Celery Executor

태스크를 병렬로 실행 가능하며, 메시지큐(MQ)를 추가적적으로 연동하여 실행하는 형태

  • 워커를 스케일아웃할 수 있는 방법 중 하나입니다.

  • Celery 백엔드로 MQ 를 연동하여 스케줄러를 실행해야 하는 서버로 메세지를 보냅니다. (Clery Executor → Message Broker → Worker 서버의 Clery Worker)

  • 다른 서버로 전달되는 태스크는 명령을 실행하는 문자열입니다. (airflow run ~)

  • 태스크는 Clery Wroker 에서 생성한 Sub Process 형태로 실행합니다.

  • Master Node에 대한 SPOF 문제는 여전히 있습니다. 따라서 Master Node에 프로세스 모니터링이 필요합니다.

  • Message Broker에 대한 관리가 필요합니다.

설명은 안했지만 그 밖의 Executor

  • Debug Executor, Dask Executor, **Kubernetes Executor, CeleryKubernetes Executor

Worker

Process/sub process executing your task

  • 작업이 효과적으로 실행되는 프로세스입니다.

DAG (Directed Acyclic Graph)

A graph with nodes and directed agee’s but no loops

  • aware datetime objects

Airflow의 실행시간은 UTC TIMEZONE에 기반을 두고 있습니다. 사용자는 이를 조절하여 start_date, end_date 설정하여 맞추어야 합니다.

  • Pendulum

airflow에 적용되는 UTC TIMEZONE 시간을 현재 한국에 맞게 조절하기 위한 파이썬 라이브러리입니다.

  • catchup

과거(start_date)로부터 현재시점까지 중간에 비워져 있는 Airflow DagRun 스케줄링을 전부 실행하는 옵션입니다.

DAG Run

object representing an instantiation of the DAG in time : 제 시간에 수행되는 Dag의 인스턴스화를 나타내는 객체

dag-run document

`

  • DAG가 실행될 떄, DAG Run 인스턴스가 생성되고, 해당 인스턴스 안의 모든 Task가 순서대로 실행됩니다.

  • DAG의 실행상태는 Task에 의존합니다.

  • 각 DAG Run은 개별적으로 실행되고, 동시에 많은 DAG를 실행할 수 있어 높은 병렬성을 가집니다.

DAG Run Status

  • DAG Run의 상태는 DAG가 끝나야 결정됩니다.

  • DAG 실행은 DAG 내의 Task와 의존성에 따라 다릅니다.

  • DAG Run 의 상태는 success, failed or skipped 와 같은 종결상태 중 하나로, 다른 상태로 전환 불가능한 경우에 할당됩니다. (“leaf nodes”, “leaves” : No Children)

    • success : “leaf nodes” 가 success 또는 skipped 인 경우

    • failed : “leaf nodes” 가 failed 또는 upstream_failed(상위 Task가 에러가 나서 멈춘 경우) 인 경우

    • (주의) TriggerRule 옵션에 따라 중간 상태가 failed 이 나더라도, 마지막에 필수적 수행(all_done) 및 성공시 DAG Run 상태는 success가 됩니다.

크론 사전 설정(Cron Presets)

  • DAG의 schedule_interval 실행 아규먼트로 넣을 수 있는 간단한 사전 설정 옵션값 입니다

  • schedule_interval 은 아규먼트로는 cron expression(“minute hour day month year”) , datetime.timedelta 객체, presets 중 하나를 쓸 수 있습니다.

  • 더 정교한 스케줄링을 원한다면 custom timetable 객체를 만들어 활용 가능합니다.

preset meaning cron
None 예약하지 않고 “externally triggered” DAG 에만 사용
@once 한 번만 예약
@continuous 이전 실행이 완료되는 즉시 실행
@hourly 한 시간에 한 번씩 시간 종료 시 실행 0 * * * *
@daily 하루에 한 번 자정(24:00)에 실행 0 0 * * *
@weekly 주 1회 일요일 자정(24:00)에 실행 0 0 * * 0
@monthly 매월 1일 자정(24:00)에 실행 0 0 1 * *
@quarterly 분기별 1일 자정(24:00)에 1회 실행 0 0 1 */3 *
@yearly 연 1회 1월 1일 자정(24:00)에 실행 0 0 1 1 *
  • DAG 는 백엔드 데이터베이스의 DAG Run 실행 항목과 함께 각 스케줄링에 대해 인스턴스화 됩니다.

Operator

An object encapsulating task, the job that you want to execute

  • 세 가지 유형이 있습니다. : Action Operators, Transfer Operators, Sensor Operators

  • Task Instance : DAG에서 실행되는 오퍼레이터가 실행되는 오퍼레이터입니다.

  • Workflow : 오퍼레이터 사이의 종속성이 있습니다.
