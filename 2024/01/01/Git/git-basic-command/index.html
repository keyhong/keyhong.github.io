[Git] Basic Command

Publish Date:   2024-01-01
Git

Git Config Command

Git config 확인하기

$ git config --list

global User 저장하기

$ git config --global user.name <user>

global Email 저장하기

$ git config --global email.name <email>

global config 삭제하기

$ git config --unset --global <config> <value>

# example
$ git config --unset --global user.name hong

Git Working Command

원격저장소에서 프로젝트 가져오기 (From GitHub)

$ git clone <원격저장소 URL> [원격저장소 파일을 넣으려고 만드는 폴더]

로컬저장소 생성하기 (로컬저장소 히스토리 내역 추가)

$ git init <로컬저장소>

Modified 상태의 파일을 Stage 하기

# "." : 현재 디렉토리 하위 내 모든 파일
$ git add <추가할 파일 또는 . ( "." : 현재 디렉토리 하위 내 모든 파일)>
    -u, --update : <pathspec>과 일치하는 항목이 이미 있는 위치에서 인덱스를 업데이트

git add 취소하기

$ git reset HEAD

변경사항 커밋하기

$ git commit -m <"메세지">

작업 디렉토리(Working Directoy)와 스테이징 영역(Staging Area) 확인

$ git status

원격저장소 push

$ git push
  --force : 강제로 push

원격저장소 추가

$ git remote add <저장소 별칭> <원격저장소 URL>

원격저장소 확인

$ git remote -v

원격저장소 삭제

$ git remote rm <저장소 별칭>

원격 저장소에서 최신 메타데이터 정보를 확인하라는 명령을 전달

$ git fetch [원격저장소 별칭]

로컬 저장소와 원격 저장소에의 차이 보기

$ git diff <원격저장소 별칭>/<로컬 저장소 별칭(master 브랜치)>

Git Branch Command

branch 만들기

$ git branch <브랜치명>

branch 리스트 확인

$ git branch -a

branch 삭제

$ git branch -d <브랜치명>

branch 강제 삭제

$ git branch -D <브랜치명>

Git Reset Command

git add 취소하기

$ git reset HEAD

가장 최신 커밋, 스테이징 취소하기

$ git reset HEAD^

이전 commit 이력 조회

$ git reflog

이전 commit 이력으로 되돌리기

$ git reset --hard HEAD@{N}
