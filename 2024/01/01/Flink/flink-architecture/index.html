[Flink] Architecture

Publish Date:   2024-01-01
Flink

Flink란?

무제한 및 제한 데이터 스트림에 대한 상태 저장 계산을 위한 프레임워크 및 분산 처리 엔진

Flink Architecture

모든 일반 클러스터 환경에서 실행(독립성), 메모리 내 속도와 모든 규모에서 계산을 수행하도록 설계(분산처리 최적화)

무제한 및 제한된 데이터 처리

모든 종류의 데이터는 이벤트 스트림으로 생성. 데이터는 무제한 스트림 또는 제한 스트림으로 처리

무제한 스트림(Unbounded streams)

시작이 있지만, 끝이 정의되어 있지 않음. 종료되지 않고 지속적 처리

  • 이벤트는 수집된 후 즉시 처리

  • 입력이 제한되지 않고, 특정 시점에 완료 되지 않기 때문

  • 정확한 시간 및 상태 제어를 통해 Flink의 런타임은 무제한 스트림에서 모든 종류의 응용 프로그램을 실행 가능

제한 스트림(Bounded streams)

시작과 끝이 있음. 연산을 수행하기 전에 데이터를 일정 시간 동안 수집하여 처리

  • 데이터 세트는 항상 정렬될 수 있기에 정렬된 수집이 필요하진 않음

  • 일괄 처리라고도 함

  • 제한 스트림은 고정 크기의 데이터 세트를 위해 특별히 설계된 알고리즘 및 데이터 구조에 의해 내부적으로 처리되므로 우수한 성능을 제공

어디서나 배포 가능

Apache Flink는 분산 시스템이며 애플리케이션을 실행하기 위해 컴퓨팅 리소스가 필요

  • Hadoop YARN, Apache Mesos, Kubernetes 와 같은 모든 일반적인 클러스터 리소스 관리자와 통합 가능

  • 독립 실행형 클러스터로 별도의 실행도 가능

  • 구성된 클러스터에 따라 필요한 리소스를 자동 식별하여 리소스 관리자에게 요청

  • 장애가 발생한 컨테이너를 대체하기 위해 새로운 리소스 할당 요청

  • 애플리케이션을 제출하거나 제어하기 위한 모든 통신은 REST call 로 이루어지며, 이 덕분에 많은 환경과 통합 가능

어떤 규모에서도 실행 가능(확장성)

어떤 규모에서도 stateful streaming 할 수 있게 고안

  • 클러스터에서 분산되고 동시에 실행되는 수천 개의 작업으로 병렬화 실행

  • 클러스노 노드를 확장함에 따라 CPU, 메인 메모리, 디스크 및 네트워크 IO의 리소스 확장성이 높음

  • 비동기식 incremental 체크포인트 알고리즘으로 exactly-once 처리를 보장하여 처리 대기 시간에 미치는 영향을 최소화

  • 하루에 수조 건의 이벤트를 처리, 여러 테라바이트의 상태를 유지, 수천 개의 코어에서 실행되는 애플리케이션을 보장

In-Memory Performance 활용

Stateful한 Flink Application은 local state access에 최적화

  • 작업 상태는 항상 메모리에 유지

  • state 크기가 메모리를 초과하는 경우 접근이 효율적인 디스크 상의 데이터 구조로 유지

  • task들은 local state에 접근하여(대부분 메모리에서), 처리 지연 시간이 느림

  • Flink는 지속적인 스토리지에 주기적, 비동기적인 체크포인트를 만들어 장애 상태의 발생에도 exactly-once 일관성을 보장
