tuple, set, dict 정복하기

tuple

왜 쓸까?

  • 프로그램을 작동하는 동안 변경되지 않는 데이터의 저장 (Immutable)
    Ex) 학번, 이름, 우편번호 등등
  • 함수의 반환 값등 사용자의 실수에 의한 에러를 사전에 방지

t = (1) # int로 인식
1

t = (1, ) # 값이 하나일 경우 () 없이 정의하면 int()로 정의됩니다. 하나의 값에는 Tuple은 반드시 ‘,’ 를 붙어주어야 tuple로 정의됩니다.
(1,)

set

집합

  • 값을 순서없이 저장, 중복 불허하는 자료형

  • set 객체 선언을 이용하여 객체 생성

  • 삽입/삭제 : O(1) (python 기준)

  • 중복을 허용하지 않는 컨테이너 자료형

s = set()
s.add(456)
s.add(12)
s.add(456)
s.add(7890)
s.add(7890)
s.add(456)

print('size:', len(s))

for i in s:
    print(i)
```    

```python
# set 함수를 사용해서 중복을 제거하는 1, 2, 3의 객체 집합 생성  
s = set([1, 2, 3, 1, 2, 3]) 
print(s)
# 출력
# {1, 2, 3}
# add() 함수로 한 개의 element를 추가할 수 있습니다.
# 아래의 코드에서는 element 1을 추가하려 했지만, 중복 불허로 추가 되지 않았습니다.
s = set([1, 2, 3])
s.add(1) 
print(s)
# 출력
# {1, 2, 3}

# 한번에 여러 element를 추가하려면 update()를 사용합니다.
s = set([1, 2, 3])
s.update([1, 4, 5, 6, 7]) 
print(s)
# 출력
# {1, 2, 3, 4, 5, 6 ,7}
# remove() 함수로 한 개의 element를 제거할 수 있습니다.
# set은 mutable하기 때문에 remove() 함수의 결과를 별도의 변수에 저장하지 않아도 됩니다.
s = set([1, 2, 3])
s.remove(1) 
print(s)
# 출력
# {2, 3}

# discard() 함수로도 똑같이 한 개의 element를 제거할 수 있습니다.
s = set([1, 2, 3])
s.discard(1)
print(s)
# 출력
# {2, 3}

set.remove() vs set.discard()

둘 다 똑같이 set의 element를 제거합니다. 다만, set에 없는 element 값을 제거하려 했을 때,
remove()는 KeyError를 반환하고, discard()는 에러를 반환하지 않고 아무 행위를 하지 않는 차이가 있습니다.

s = set([1, 2, 3])
s.remove(4) 
# 출력
# Traceback (most recent call last):
#   File "<stdin>", line 1, in <module>
# KeyError: 4

# discard() 함수로도 똑같이 한 개의 element를 제거할 수 있습니다.
s = set([1, 2, 3])
s.discard(4)
# 출력
# None

s.clear() # 모든 원소 삭제


#### 집합 연산

- 두 개 이상의 set 자료형은 수학에서 활용하는 다양한 집합 연산이 가능합니다.

```python
s1 = set([1, 2, 3, 4, 5])
s2 = set([3, 4, 5, 6, 7])

# s1과 s2의 합집합
s1.union(s2)
# 출력
# {1, 2, 3, 4, 5, 6, 7}

# s1과 s2의 합집합을 파이프(|) 연산 기호로도 표현 가능합니다.
s1 | s2
# 출력
# {1, 2, 3, 4, 5, 6, 7} 

# s1과 s2의 교집합
s1.intersection(s2) 
# 출력
# {3, 4, 5}

# s1과 s2의 합집합을 앰퍼샌드(&) 연산 기호로도 표현 가능합니다.
s1 & s2
# 출력
# {3, 4, 5}

# s1과 s2의 차집합(s1에서 s2의 요소를 제거)
s1.difference(s2) 
# 출력
# {1, 2}

# s1과 s2의 차집합을 마이너스(-) 연산 기호로도 표현 가능합니다.
s1 - s2
# 출력
# {1, 2}

dict(dictionary) == map

  • 삽입/삭제 : O(1) (python 기준)

  • key, value 자료구조

  • key는 중복 불가, value는 중복 가능

m = {}
m['Yoondy'] = 40
m['Sky'] = 100
m['Jerry'] = 50

print('size:', len(m))

for k in m:
    print(k, m[k])
```        


> 데이터를 저장 할 때는 구분 지을 수 있는 키(key)와 값(value)을 함께 저장하는 자료구조

- { key1: value1, key2: value2, key3: value3 ... } 형태

- 구분을 위한 데이터 고유값을 Indentifier 또는 key 라고 함

- Key 값을 활용하여, 데이터 값(Value)를 관리함

- Key 값은 Hashable한 값만 들어올 수 있다
  
- key 값을 매칭하여 key로 value를 검색
  
- 다른 언어에서는 Hash Table 이라는 용어를 사용 (HashTable과 Dict는 조금 다르긴 함)
  
```python
# Dict 생성, country_code = dict()도 가능
country_code = {} 

country_code = {"America": 1, "Korea": 82, "China": 86, "Japan": 81}
country_code
# 출력
# {'America': 1, 'China': 86, 'Korea': 82, 'Japan': 81}

country_code.items() # Dict 데이터 출력
# 출력
# dict_items([('America': 1), ('China': 86), ('Korea': 82), ('Japan': 81)])

country_code.keys() # Dict 키 값만 출력
# 출력
# dict_keys(['America', 'China', 'Korea', 'Japan'])

country_code["German"] = 49 # Dict 추가
country_code
# 출력
# {'America': 1, 'German': 49, 'China': 86, 'Korea': 82, 'Japan': 81}

country_code.values() # Dict Value만 출력
# 출력
# dict_values([1, 49, 86, 82, 81]) 

# items가 tuple로 나와 k, v에 언팩킹(unpacking)
for k, v in country_code.items(): 
	print('Key :', k)
	print('Value :', v)

'Korea' in country_code.keys() # Key 값에 'Korea'가 있는지 확인
# 출력
# True

82 in country_code.values() # Value 값에 82가 있는 지 확인
# 출력
# True
