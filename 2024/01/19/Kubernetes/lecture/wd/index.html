Region을 선택한다.

AWS Region을 선택한다. Region별 이용 가격에 차이가 있다. (아르헨티나, 유럽이 가장 비싸고 나머지는 비슷하다.) Region은 물리적 서버가 얼마나 가까이 있는 지를 결정한다. 프로덕션 환경에서는 서비스를 제공하는 국가와 있는 것이 좋다.

먼저 EC2 인스턴스 하나를 생성한다.

생성된 인스턴스에 진입 후

# root 계정 전환
sudo su - 

# hostctl로 hostname 변경
hostnamectl set-hostname master

# hostname 변경확인
hostname

도커 설치


# 도커 설치
# https://docs.docker.com/engine/install/ubuntu/

# 1) 패키지 매니저 업데이트
$ sudo apt-get update

# 2) https를 사용해서 레포지토리를 사용할 수 있도록 필요한 패키지를 설치한다. 
$ sudo apt-get install -y  apt-transport-https ca-certificates curl software-properties-common

# 3) Docker 공식 리포지토리에서 패키지를 다운로드 받았을 때 위변조 확인을 위한 GPG 키를 추가한다.
$ curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo apt-key add -

# 4) Docker.com 의 GPG 키가 등록됐는지 확인한다. 
$ apt-key fingerprint     

# 5) Docker 공식 저장소를 리포지토리로 등록한다.
$ add-apt-repository "deb [arch=amd64] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable"


# 6) 저장소 등록정보에 기록됐는지 확인한다. 
$ grep docker /etc/apt/sources.list
deb [arch=amd64] https://download.docker.com/linux/ubuntu bionic stable

# 7) 리포지토리 정보를 갱신
$ apt-get update

# 8) docker container engine 을 설치한다.
$ apt-get install -y docker-ce

# 9) docker version
$ docker 버전 확인

docker container engine 을 설치한다.

AWS에서는 스왑핑 기능이 없기 때문에 아래 과정은 고려하지 않아도 된다. 그러나 로컬에 직접 설치한다면 반드시 해줘야 하는 과정이다.

# 보완리눅스가 설치되어 있다면
$ setenforce 0

# 방화벽 끄기
$ ufw disable

# modprobe br_netfilter
$ cat <<EOF | sudo tee /etc/modules-load.d/k8s.conf
br_netfilter
EOF

$ cat <<EOF | sudo tee /etc/sysctl.d/k8s.conf
net.bridge.bridge-nf-call-ip6tables = 1
net.bridge.bridge-nf-call-iptables = 1
EOF

$ sudo sysctl --system

쿠버네티스(kubeadm/kubelet/kubectl) 설치 (1.21버전까지만 도커 지원)

# 1) 
$ apt update

# 2) 
$ apt-get install -y apt-transport-https ca-certificates curl

# 3) 
$ curl -s https://packages.cloud.google.com/apt/doc/apt-key.gpg | sudo apt-key add -

# 4) 
$ 
cat <<EOF | sudo tee /etc/apt/sources.list.d/kubernetes.list
deb https://apt.kubernetes.io/ kubernetes-xenial main
EOF

# 5)
$ apt update

# 6)
$ apt-get install -y kubelet=1.24.0-00 kubeadm=1.24.0-00 kubectl=1.24.0-00

# 7)
$ kubectl version
$ kubelet --version
$ kubeadm version

# 8)
$ apt-mark hold kubelet kubeadm kubectl

# cgroup
cat <<EOF | sudo tee /etc/docker/daemon.json
{
  "exec-opts": ["native.cgroupdriver=systemd"],
  "log-driver": "json-file",
  "log-opts": {
    "max-size": "100m"
  },
  "storage-driver": "overlay2"
}
EOF

$ sudo systemctl enable docker
$ sudo systemctl daemon-reload
$ sudo systemctl restart docker

# 7. containerd

$ vi /etc/containerd/config.toml
:q!
$ rm /etc/containerd/config.toml
$ systemctl restart containerd
$ systemctl enable containerd
$ apt update
