Intro to the DataStream API

이 교육의 중점은 DataStream API를 충분히 폭넓게 다루어 스트리밍 응용 프로그램을 작성하는 데 충분한 기초를 제공하는 것입니다.

What can be Streamed?

Flink의 DataStream API는 직렬화할 수 있는 모든 것을 스트리밍할 수 있습니다. Flink의 고유한 직렬화기가 사용됩니다:

  • 기본 유형, 즉 String, Long, Integer, Boolean, Array
  • 복합 유형: Tuples, POJOs 및 Scala case classes

그리고 Flink는 다른 유형에 대해서는 Kryo를 사용합니다. 또한 Flink에서 다른 직렬화기를 사용할 수 있습니다. 특히 Avro는 잘 지원됩니다.

Java tuples and POJOs

Flink의 고유한 직렬화기는 튜플과 POJOs에서 효율적으로 작동할 수 있습니다.

Tuples

Java의 경우, Flink는 자체 Tuple0부터 Tuple25 유형을 정의합니다.

Tuple2<String, Integer> person = Tuple2.of("Fred", 35);

// zero based index!  
String name = person.f0;
Integer age = person.f1;

POJOs

다음 조건을 충족하면 Flink는 데이터 유형을 POJO 유형으로 인식하고 “이름으로” 필드 참조를 허용합니다:

  • 클래스가 공개되어 있으며 독립적입니다 (비정적인 내부 클래스가 없음)
  • 클래스에는 공개된 매개변수가 없는 생성자가 있습니다
  • 클래스 내의 모든 비정적이고 비순간적인 필드 (그리고 모든 상위 클래스)는 public이거나 (그리고 final이 아닌) 공개된 getter 및 setter 메서드를 사용하여 Java 빈 네이밍 규칙을 따릅니다.

예를 들어:

public class Person { // 클래스가 공개되어 있으며 독립적
    public String name;  
    public Integer age;  
    public Person() {} // 공개된 매개변수가 없는 생성자
    public Person(String name, Integer age) {  
        . . .
    }
}  

Person person = new Person("Fred Flintstone", 35);

Flink의 serializer는 supports schema evolution for POJO types.

Scala tuples and case classes

이들은 예상한 대로 작동합니다.

모든 Flink Scala API는 폐기되었으며 향후 Flink 버전에서 제거될 예정입니다. 여전히 Scala로 응용 프로그램을 작성할 수 있지만 DataStream 및/또는 Table API의 Java 버전으로 전환하는 것이 좋습니다.

FLIP-265 Deprecate and remove Scala API support을 참조하십시오. Scala API 지원 폐기 및 제거

A Complete Example

이 예제는 입력으로 사람에 관한 레코드 스트림을 받아와 성인만 포함하도록 필터링합니다.

import org.apache.flink.streaming.api.environment.StreamExecutionEnvironment;
import org.apache.flink.streaming.api.datastream.DataStream;
import org.apache.flink.api.common.functions.FilterFunction;

public class Example {

    public static void main(String[] args) throws Exception {
        final StreamExecutionEnvironment env =
                StreamExecutionEnvironment.getExecutionEnvironment();

        DataStream<Person> flintstones = env.fromElements(
                new Person("Fred", 35),
                new Person("Wilma", 35),
                new Person("Pebbles", 2));

        DataStream<Person> adults = flintstones.filter(new FilterFunction<Person>() {
            @Override
            public boolean filter(Person person) throws Exception {
                return person.age >= 18;
            }
        });

        adults.print();

        env.execute();
    }

    public static class Person {
        public String name;
        public Integer age;
        public Person() {}

        public Person(String name, Integer age) {
            this.name = name;
            this.age = age;
        }

        public String toString() {
            return this.name.toString() + ": age " + this.age.toString();
        }
    }
}

Stream execution environment

모든 Flink 애플리케이션에는 실행 환경이 필요합니다. 이 예제에서는 env로 나타냅니다. 스트리밍 애플리케이션은 StreamExecutionEnvironment를 사용해야 합니다.

당신의 애플리케이션에서 수행된 DataStream API 호출은 StreamExecutionEnvironment에 첨부된 작업 그래프를 작성합니다. env.execute()가 호출되면 이 그래프는 패키지되어 JobManager로 전송되며, JobManager는 작업을 병렬화하고 이를 실행하기 위해 Task Manager에게 작업 조각을 분배합니다. 작업의 각 병렬 조각은 task slot에서 실행됩니다.

execute()를 호출하지 않으면 애플리케이션이 실행되지 않습니다.

이 분산 런타임은 애플리케이션이 직렬화 가능하도록 하는 것에 의존합니다. 또한 클러스터의 각 노드에서 모든 종속성이 사용 가능해야 합니다.

Basic stream sources

위의 예제는 env.fromElements(...)을 사용하여 DataStream<Person>을 구성합니다. 이는 프로토타입이나 테스트에서 간단한 스트림을 빠르게 만들기 위한 편리한 방법입니다. StreamExecutionEnvironment에는 fromCollection(Collection) 메서드도 있습니다. 따라서 다음과 같이 수행할 수도 있습니다:

List<Person> people = new ArrayList<Person>();

people.add(new Person("Fred", 35));
people.add(new Person("Wilma", 35));
people.add(new Person("Pebbles", 2));

DataStream<Person> flintstones = env.fromCollection(people);

프로토타입을 만들 때 데이터를 스트림으로 가져오는 또 다른 편리한 방법은 소켓을 사용하는 것입니다.

DataStream<String> lines = env.socketTextStream("localhost", 9999);

또는 파일

DataStream<String> lines = env.readTextFile("file:///path");

실제 응용 프로그램에서 가장 일반적으로 사용되는 데이터 소스는 지연 시간이 낮고 처리량이 높은 병렬 읽기를 지원하며 되감기와 재생을 결합한 것입니다. 이는 고성능과 내결함성을 위한 전제 조건입니다. 예를 들어 Apache Kafka, Kinesis 및 다양한 파일 시스템이 있습니다. REST API 및 데이터베이스도 스트림 보강에 자주 사용됩니다.

Basic stream sinks

위의 예제에서는 adults.print()를 사용하여 결과를 작업 관리자 로그에 인쇄합니다(IDE에서 실행 중인 경우 IDE 콘솔에 나타납니다). 이는 스트림의 각 요소에 toString()을 호출합니다.

출력은 다음과 유사한 형태를 가집니다.

1> Fred: age 35
2> Wilma: age 35

1> 과 2>는 출력을 생성한 하위 작업(즉, 스레드)을 나타냅니다.

운영 환경에서는 주로 FileSink, 다양한 데이터베이스 및 여러 발행-구독 시스템이 사용됩니다.

Debugging

운영 환경에서는 애플리케이션이 원격 클러스터 또는 컨테이너 세트에서 실행될 것입니다. 그리고 실패할 경우 원격으로 실패합니다. JobManager 및 TaskManager 로그는 이러한 실패를 디버깅하는 데 매우 유용할 수 있지만, IDE 내에서 로컬 디버깅을 수행하는 것이 훨씬 쉽습니다. 이것은 Flink가 지원하는 기능입니다. 여기서 중요한 점은 브레이크 포인트를 설정하고 로컬 변수를 조사하며 코드를 단계별로 진행할 수 있다는 것입니다. 또한 Flink의 코드로 진입하여 Flink의 내부를 더 자세히 이해하는 데 도움이 될 수 있습니다. Flink가 어떻게 작동하는지 궁금하다면 이것이 좋은 방법일 수 있습니다.

Hands-on

이제 간단한 DataStream 애플리케이션을 코딩하고 실행하는 방법에 대해 충분히 알고 있습니다. flink-training-repo를 클론하고 README의 지침을 따라 한 번째 연습인 Filtering a Stream (Ride Cleansing)을 수행하십시오.
