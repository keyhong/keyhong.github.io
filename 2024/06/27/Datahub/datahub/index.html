Datahub

Datahub

메타데이터 관리, 데이터 검색 및 데이터 거버넌스를 간소화하도록 설계된 최신 데이터 카탈로그

사용자는 (1) 데이터의 효율적으로 탐색 및 이해하고, (2) 데이터 계보(lineage) 추적하고, (3) 데이터 프로파일링(profiling)하고, (4) 데이터 계약 설정이 가능. 메타데이터 관리 플랫폼은 사용자들이 빠르게 변화하는 데이터의 복잡성을 관리하고 전사적으로 관리되는 데이터의 가치를 활용할 수 있도록 구축 되었다.

Quickstart

데이터허브에서 제공하는 Quickstart를 이용하면 로컬에서도 바로 테스트 환경을 만들 수 있다. 물론, DBMS에서 데이터를 주입을 테스트해보려면 도커 컨테이너를 추가로 올려야 한다.

Prerequisites

Python 3.8+ 설치되어야 한다. (Check using python3 –version).

Resource Allocation

  • 프로덕션에서는 kubernetes 환경 위에서 helm으로 구동하는 것을 권장

  • 테스트 환경을 위한 로컬에서도 구동할 수 있으며, Docker 엔진에 충분한 하드웨어 리소스를 할당이 필요
    (CPU 2개, RAM 8GB, 스왑 영역 2GB, 디스크 공간 10GB)

  • 아래와 같이 zookeeper, kafka(broker, schema-registry), elasticsearch, datahbub(gms, frontend-react, actions), mysql 프로세스가 띄어진다. (초기화 컨테이너는 제외)

Install the DataHub CLI

공식문서에서는 pip과 poetry 두 개의 설치 방법을 알려주지만, 최근 파이썬 오픈소스 프로젝트들이 빌드시스템을 poetry로 전향하고 있어 여기서는 poetry를 사용했다.

# acryl-datahub 패키지 추가
$ poetry add acryl-datahub

# poetry shell (가상환경 실행)
$ poetry shell

# datahub 설치 확인
$ datahub version
> acryl-datahub, version 0.13.3

# datahub 시작
# 맥 M1/M2과 같은 Apple Silicon에서는 `no matching manifest for linux/arm64/v8 in the manifest list entries.` 오류가 나타날 수 있다.
# 이 문제를 해결하려면 `datahub docker quickstart`에 옵션을 `--arch` 옵션을 실행하여 기본 아키텍처를 재정의 해야 한다.
$ datahub docker quickstart

# datahub 중지
$ datahub docker quickstart --stop

Frontend Web UI

데이터허브 사용자들이 메타 정보를 검색, 용어 사전 등록, 데이터 주입 스케줄링이 가능한 유저 웹 인터페이스

서비스 포트는 9002를 사용한다.

Data Ingestion

여러 데이터베이스 소스로부터 메타 정보를 모아 메타 관리 시스템인 데이터허브로 주입하는 과정

데이터를 주입하는 방법은 크게 두 가지가 있다.

  • 데이터허브 Frontend Web에서 데이터 소스에 대한 정보를 직접 입력하는 방법

    • 클릭 몇 번만으로 소스 연결 설정, 주입 옵션 설정이 간편하다.

    • 메타 데이터를 소싱할 스케줄링을 만들 수 있다. (예: 일별 **시 **분).

    • 배치 스케줄링이 끝난 후 트리거를 거는 연계가 어려운 단점이 있다.

  • Recipes를 API나 CLI로 주입하는 방법

    • CLI를 사용하는 것은 datahub-actions 프로세스가 구동된 서버에서만 가능하다.

    • 일반적으로 스케줄링(예: 에어플로우)와 연계해서 사용하기 때문에 데이터허브 API를 통해 직접 주입하는 방법을 많이 사용하고 있다. 외부에서 API를 호출하려면 토큰을 발급받아 인증해야 한다.

Recipes

메타데이터 수집을 위한 주요 구성 파일. 수집 스크립트에 데이터를 어디에서 가져올지(source)와 어디에 넣을지(sink)를 알려준다.

  • 구성

레시피 파일의 기본 형식은 source와 sink로 구성된다.

# The simplest recipe that pulls metadata from MSSQL and puts it into DataHub
# using the Rest API.
source:
  type: mssql
  config:
    username: sa
    password: ${MSSQL_PASSWORD}
    database: DemoData
# sink section omitted as we want to use the default datahub-rest sink
sink:
  type: "datahub-rest"
  config:
    server: "http://localhost:8080"

Source

메타데이터를 추출하는 데이터 시스템

일반적으로 recipe 맨 위에 정의된다.
#my_recipe.yml
source: 
  type: <source_name>
  config: 
    option_1: <value>
    ...

Sink

메타데이터의 대상

일반적으로 recipe에서 source 뒤에 정의된다.
source: ...

sink:
  type: <sink_type>
  config: ...

  • 싱크 유형으로는 두 가지가 있다.

    • Kafka (datahub-kafka) :

UI Ingestion

  • 소스의 연결 정보를 쉽게 입력할 수 있다.
  • Web UI를 쓰면, 데이터를 수집할 소스를 연결할 후 실행시 필요한 라이브러리를 자동적으로 설치해준다.

Datahub Architecture

Datahub Component

프로세스 디펜던시

띄어지는 프로세스가 많아 DAG로 디펜던시(dependency)의 순서를 표시해보았다.

elasticsearch              >> elasticsearch-setup  >>  datahub-upgrade  >>  datahub-gms  >>  [ datahub-actions, datahub-frontend-react ]
zookeeper      >> broker   >> kafka-setup          >>  datahub-upgrade  >>  datahub-gms  >>  [ datahub-actions, datahub-frontend-react ]
mysql                      >> mysql-setup          >>  datahub-upgrade  >>  datahub-gms  >>  [ datahub-actions, datahub-frontend-react ]

프로세스 설명

Docker Service Name Description
datahub-upgrade DataHub 플랫폼의 업그레이드와 마이그레이션을 관리
datahub-gms DataHub의 메타데이터를 저장하고 관리하는 백엔드 서비스
datahub-actions 특정 이벤트 발생 시 자동화된 작업을 수행
datahub-frontend-react DataHub의 웹 프론트엔드로, React 기반으로 구현

1. DataHub Actions

DataHub Actions는 DataHub에서 특정 이벤트가 발생할 때 자동화된 작업을 수행하는 기능

Main Role Description
이벤트 트리거 DataHub 내에서 데이터셋 생성, 수정, 삭제 등의 이벤트 발생 시 트리거한다.
작업 정의 및 실행 이벤트에 반응하여 자동으로 수행될 작업을 정의하고 실행한다. 예를 들어, 데이터셋이 수정될 때 Slack 알림을 보내거나 데이터 품질 검사를 실행할 수 있다.
통합성 외부 시스템과 쉽게 통합하여 다양한 자동화 작업을 수행할 수 있다. 예를 들어, Jira 티켓 생성, 이메일 알림, Slack 메시지 전송 등이 가능하다.
모니터링 및 로깅 실행된 작업을 모니터링하고 로깅하여 문제 발생 시 해결할 수 있도록 지원한다.

2. DataHub Upgrade

DataHub Upgrade는 DataHub 플랫폼의 업그레이드 및 마이그레이션을 관리하는 도구

Main Role Description
데이터베이스 스키마 마이그레이션 DataHub의 메타데이터 저장소(DB)의 스키마를 최신 버전으로 마이그레이션 한다.
버전 관리 DataHub의 다양한 컴포넌트의 버전을 관리하고, 업그레이드 시 호환성을 보장한다.
자동화된 업그레이드 프로세스 업그레이드 과정이 자동화되어, 사용자가 최소한의 개입으로 DataHub를 최신 버전으로 유지할 수 있다.

3. DataHub Frontend React

DataHub Frontend React는 DataHub의 웹 프론트엔드를 구현한 React 기반 애플리케이션

Main Role Description
유저 인터페이스 사용자에게 직관적인 UI를 제공하여 데이터셋 검색, 탐색, 필터링 등을 용이하게 한다.
데이터 시각화 데이터 계보, 메타데이터 정보 등을 시각화하여 사용자에게 제공한다.
사용자 정의 가능성 React 기반으로 쉽게 커스터마이징 및 확장이 가능하다.
API 통신 DataHub의 백엔드(GMS)와 통신하여 실시간으로 데이터를 가져오고 표시한다.

4. DataHub GMS (Graph Metadata Service)

DataHub GMS는 DataHub의 백엔드 서비스로, 메타데이터를 저장하고 관리하는 핵심 컴포넌트

Main Role Description
메타데이터 저장소 다양한 데이터 소스의 메타데이터를 저장하고 관리한다.
API 제공 메타데이터 검색, 업데이트, 삭제 등을 위한 API를 제공한다.
데이터 계보 관리 데이터셋 간의 관계와 의존성을 추적하여 데이터 계보를 관리한다.
확장성 확장 가능한 아키텍처를 통해 대규모 메타데이터를 효과적으로 관리할 수 있다.
