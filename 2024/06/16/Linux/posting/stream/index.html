Stream & Redirection

Linux

History

쉘 스크립트를 읽을 때 항상 리디렉션이 복병이었는 데, 최근 조금 더 기본기를 다지면서 공부한 내용을 적어봅니다.

작업환경 만들기(docker - ubuntu)

  • ubuntu는 짝수년도 4월에 LTS(Long Term Support) 버전을 내고 있어요. 지금 시점으로 ubuntu:22.04 버전을 사용하려고 합니다.

  • docker run 명령어로 tag(:22.04)를 안주면 latest로 보다 높은 버전을 줄 수도 있습니다.

$ docker run -it ubuntu:22.04

  • it(-interactive, -tty)는 터미널(단말)에서 인터랙티브하게 커맨드를 주고 받겠다는 의미입니다.

  • ubuntu image는 다른 컨테이너처럼 터미널로 실행할 때 bash를 적어주지 않아도 default로 실행됩니다.

스트림

두 프로세스 간의 실행 결과를 전달하고 받을 수 있도록 이어주는 매체

  • 프롬프트에서 명령어를 날리면 성공(stdout) 혹은 에러(stderr)에 대한 실행 결과를 전달 받습니다.

  • 우리가 보는 shell도 하나의 프로세스라면 shell과 애플리케이션 두 프로세스 사이에는 통신이 필요합니다.

  • 스트림은 실행 결과를 어디로 전달할 지 두 프로세스를 이어주는 역할을 합니다.

  • 이게 왜 중요하나면, 실제 애플리케이션을 운영하면 사용자의 행동에 대한 무수한 로그가 찍히는 데 에러 로그만 별도의 파일로 분리하면 트러블 슈팅을 할 때 유용합니다.

스트림 출력의 3종류

  • 표준 입력(standard input)

사용자의 입력을 보내는 스트림. 기본적인 표준 입력 장치는 키보드입니다.

  • 표준 출력(standard output)

정상적인 표준 출력을 받는 스트림. 기본적인 표준 입력 장치는 모니터입니다.

  • 표준 에러(standard output)

정상적인 표준 에러를 받는 스트림. 기본적인 표준 입력 장치는 모니터입니다.

스트림 활용하기

  • 요즘 추세가 Cloud Native라 Dockerfile을 작성하기 위해서는 shell 스크립팅이 필수적입니다.

  • 쉘에서 가장 복잡한 게 스트림을 다른 프로세스나 파일에 전달하는 방법인 리디렉션(>, >>, 1>, 2>, …)과 파이프라인(|)인 것 같습니다

리디렉션(Redirection)

어떤 프로세스의 스트림을 다른 프로세스에 전달하는 방법

  • 본래 표준 출력과 표준 에러는 모니터(프롬프트)를 통해 전달됩니다. 하지만 리디렉션을 통해 다른 프로세스나 파일로 넘길 수 있습니다.

  • 형식: 부등호 기호(<, >, <<, >>)

  • >: 덮어쓰기(overwrite) (**주의: 파일로 전달했을 때 파일 내용을 덮어쓰니 기존 내용을 지워집니다.)

덮어쓰기(overwrite)
# 출력 리디렉션(본래 내 프롬프트에 출력되어야 할 결과를 'hi'라는 파일로 전달)
$ echo "hi" > hi

# 'hi'라는 파일 내용 확인
$ cat hi
hi

  • 여기서 > 를 >>로 바꾸면 기존 파일의 마지막에 새로운 줄을 덧붙이기(append) 할 수 있습니다.

  • >>: 덧붙이기(append)

덧붙이기(append)
# (1) 파일 확인
$ cat hi
hi

# (2) echo의 결과를 리디렉션
$ echo "hi" >> hi

# (3) 파일 확인
$ cat hi
hi
hi
  • 우리는 hi라는 디렉터리가 아닌 파일을 만들었는 데 cd(change directory) 명령어로 hi에 접속하려고 한 결과를 hi2로 리디렉션하면 결과는 어떻게 나올 까요?
# (1) 에러가 발생할 결과를 리디렉션. 프로프트에는 출력
$ cd hi > hi2
bash: cd: hi: Not a directory

# (2) hi2파일은 생성됬지만, 어떤 결과도 리디렉션 되지 않음
$ cat hi2

  • 우리는 프로그래밍 할 때 어떤 함수의 파라미터에 사용자 입력을 받지 않았을 때 default를 설정합니다. 유사하게 리디렉션 >은 1>과 같습니다. 1이 생략되어 있습니다.

  • >1: 표준 출력 리디렉션

만약 프롬프트로 전달된 에러를 hi2로 보내고 싶다면 2>를 하셔야 합니다.

  • >2: 표준 에러 리디렉션
표준 에러 리디렉션 하기 (2>)
# 아래 커맨드 후 에러가 터미널에서는 보이지 않아요.
$ cd hi 2> hi2

# 대신 hi2라는 파일에 표준 출력이 전달됬어요.
$ cat hi2
bash: cd: hi: Not a directory

  • 표준 출력과 에러를 둘 다 보내고 싶을 수도 있습니다. 하지만 3>은 없습니다.

  • 이 때부터는 파일 디스크립터(file descriptor)로 리디렉션해야 해요.

파일 디스크립터

파일 디스크립터 &>는 파일 작업을 실행해주는 객체

  • 모든 파일에 대한 작업은 파일 디스크립터를 통해 수행이 되어집니다.

  • 파일 디스크립터를 통행 표준 출력과 표준 에러를 모두 전달하는 방법은 아래처럼 쓸 수 있어요.

# 리디렉션(파일 디스크립터)
$ cd hi &> hi2

# 아래도 됩니다. 위 리디렉션 보다 최신에 나왔어요.
$ cd hi >& hi2
