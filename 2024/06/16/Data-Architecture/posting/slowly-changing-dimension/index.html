Slowly Changing Dimension

데이터 아키텍처

SCD

  • Slowly Changing Dimensions

시간이 지남에 따라 변하는 Dimension. Master 테이블(원장성 테이블)에 분석을 위해 과거로 부터의 이력 관리가 필요한 경우 사용함

  • 현재 및 과거 버전을 관리함. 변화를 추적하도록 설계됨

SCD 유형

1. Type 0

원래 값을 유지

  • 변화가 거의 없음

  • 속성은 절대 변경되지 않음

  • Fact는 항상 원래 값(original value)으로 그룹화됨

2. Type 1

이전 값 덮어쓰기

  • 속성은 항상 가장 최근 업데이트 값을 반영하며, 과거의 이력은 관리하지 않음

3. Type 2

새로운 레코드를 추가

  • 새로운 기본 키(PK), flag 컬럼 또는 일자 컬럼(점 이력 또는 선분이력)이 추가해서 이력을 관리

4. Type 3

새로운 속성 열을 차원에 추가하여 과거 이력을 보존

  • 2가지 다른 현실로 쿼리할 수 있게 함

5. Type 4

이력 따로 관리하기 위해 별도의 히스토리 테이블을 사용

  • 메인 dimension은 현재 시간 기준으로 최신 상태의 데이터만 유지

6. Snapshot

테이블 파티션을 사용하고 dimension을 스냅샷으로 저장

  • 모든 데이터는 매일 또는 매주 스냅샷으로 저장됨
