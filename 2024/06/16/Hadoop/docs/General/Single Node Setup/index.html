Hadoop: Setting up a Single Node Cluster.

Purpose

이 문서는 Hadoop MapReduce 및 Hadoop 분산 파일 시스템 (HDFS)을 사용하여 간단한 작업을 신속하게 수행할 수 있도록 단일 노드 Hadoop 설치를 설정하고 구성하는 방법에 대해 설명합니다.

중요: 모든 프로덕션 Hadoop 클러스터는 HDFS 데이터에 대한 호출자 인증 및 접근 보안을 위해 Kerberos를 사용하며, 계산 서비스 (YARN 등)에 대한 접근을 제한합니다.

이 지침은 어떤 Kerberos 서비스와의 통합을 다루지 않습니다. 프로덕션 클러스터를 구축하는 모든 사용자는 배포의 중요한 부분으로 자신의 조직의 Kerberos 인프라에 연결해야 합니다.

Security 섹션에서 클러스터를 보호하는 방법에 대한 자세한 내용을 확인하십시오.

Prerequisites

Supported Platforms

  • GNU/Linux는 개발 및 프로덕션 플랫폼으로 지원됩니다. Hadoop은 2000개 노드를 갖는 GNU/Linux 클러스터에서도 시연되었습니다.

Required Software

Linux용 필수 소프트웨어는 다음과 같습니다:

  1. Java™가 설치되어 있어야 합니다. 권장되는 Java 버전은 HadoopJavaVersions에서 설명되어 있습니다.

  2. Hadoop 데몬을 관리하는 Hadoop 스크립트를 사용하려면 ssh가 설치되어 있어야 하며, 선택적으로 시작 및 중지 스크립트를 사용하려면 sshd가 실행되어야 합니다. 더 나은 ssh 자원 관리를 위해 pdsh도 설치하는 것이 권장됩니다.

Installing Software

만약 클러스터에 필요한 소프트웨어가 없다면 설치해야 합니다.

예를 들어 Ubuntu Linux에서는 다음과 같이 설치할 수 있습니다:

$ sudo apt-get install ssh
$ sudo apt-get install pdsh

Download

Hadoop 배포본을 얻으려면 Apache Download Mirrors 중 하나에서 최신 안정 버전을 다운로드하세요.

Prepare to Start the Hadoop Cluster

다운로드한 Hadoop 배포본을 해제하세요. 배포본에서 파일 etc/hadoop/hadoop-env.sh을 편집하여 다음과 같이 일부 매개변수를 정의하세요:

# set to the root of your Java installation
export JAVA_HOME=/usr/java/latest

Try the following command:

다음 명령어를 시도해보세요:

$ bin/hadoop

이렇게 하면 hadoop 스크립트의 사용 설명서가 표시됩니다.

이제 세 가지 지원되는 모드 중 하나로 Hadoop 클러스터를 시작할 준비가 되었습니다:

  • Local (Standalone) Mode
  • Pseudo-Distributed Mode
  • Fully-Distributed Mode

Standalone Operation

기본적으로 Hadoop은 비분산 모드로 구성되어 있어 단일 Java 프로세스로 실행됩니다. 이것은 디버깅에 유용합니다.

다음 예제는 풀린 conf 디렉토리를 입력으로 사용하고 지정된 정규 표현식에 대한 모든 일치 항목을 찾아서 표시합니다. 출력은 지정된 출력 디렉토리에 작성됩니다.

$ mkdir input
$ cp etc/hadoop/*.xml input
$ bin/hadoop jar share/hadoop/mapreduce/hadoop-mapreduce-examples-3.3.6.jar grep input output 'dfs[a-z.]+'
$ cat output/*

Pseudo-Distributed Operation

Hadoop은 각각의 Hadoop 데몬이 별도의 Java 프로세스에서 실행되는 가상 분산 모드에서도 단일 노드에서 실행할 수 있습니다.

Configuration

Use the following:

다음을 사용하세요:

etc/hadoop/core-site.xml
<configuration>
    <property>
        <name>fs.defaultFS</name>
        <value>hdfs://localhost:9000</value>
    </property>
</configuration>
etc/hadoop/hdfs-site.xml
<configuration>
    <property>
        <name>dfs.replication</name>
        <value>1</value>
    </property>
</configuration>

Setup passphraseless ssh

이제 암호구문 없이 localhost로 SSH 접속할 수 있는지 확인하세요:

$ ssh localhost

만약 암호구문 없이 localhost로 SSH 접속할 수 없다면, 다음 명령을 실행하세요:

$ ssh-keygen -t rsa -P '' -f ~/.ssh/id_rsa
$ cat ~/.ssh/id_rsa.pub >> ~/.ssh/authorized_keys
$ chmod 0600 ~/.ssh/authorized_keys

Execution

다음 지침은 로컬에서 MapReduce 작업을 실행하는 방법입니다. 만약 YARN에서 작업을 실행하려면 YARN on Single Node을 참조하십시오.

  1. 파일 시스템을 포맷하세요:
$ bin/hdfs namenode -format
  1. NameNode 데몬과 DataNode 데몬을 시작하세요:
$ sbin/start-dfs.sh

Hadoop 데몬 로그 출력은 $HADOOP_LOG_DIR 디렉터리에 작성됩니다 ($HADOOP_HOME/logs로 기본 설정됨).

  1. NameNode의 웹 인터페이스를 둘러보세요. 기본적으로 다음 위치에서 사용할 수 있습니다:

    • NameNode - http://localhost:9870/

  2. MapReduce 작업을 실행하는 데 필요한 HDFS 디렉터리를 생성하세요:

$ bin/hdfs dfs -mkdir -p /user/<username>
  1. 분산 파일 시스템으로 입력 파일을 복사하세요:
$ bin/hdfs dfs -mkdir input
$ bin/hdfs dfs -put etc/hadoop/*.xml input
  1. 제공된 예제 중 일부를 실행하세요:
$ bin/hadoop jar share/hadoop/mapreduce/hadoop-mapreduce-examples-3.3.6.jar grep input output 'dfs[a-z.]+'
  1. 출력 파일을 확인하세요: 분산 파일 시스템에서 출력 파일을 로컬 파일 시스템으로 복사하고 확인하세요:
$ bin/hdfs dfs -get output output
$ cat output/*

또는

분산 파일 시스템에서 출력 파일을 확인하세요:

$ bin/hdfs dfs -cat output/*
  1. 작업을 마쳤으면 다음 명령으로 데몬을 중지하세요:
$ sbin/stop-dfs.sh

YARN on a Single Node

일부 매개변수를 설정하고 ResourceManager 데몬 및 NodeManager 데몬을 추가로 실행하여 가상 분산 모드에서 YARN에서 MapReduce 작업을 실행할 수 있습니다.

다음 지침은 이미 the above instructions의 1. ~ 4. 단계가 실행되었다고 가정합니다.

  1. 다음과 같이 매개변수를 구성하세요:
etc/hadoop/mapred-site.xml
<configuration>
    <property>
        <name>mapreduce.framework.name</name>
        <value>yarn</value>
    </property>
    <property>
        <name>mapreduce.application.classpath</name>
        <value>$HADOOP_MAPRED_HOME/share/hadoop/mapreduce/*:$HADOOP_MAPRED_HOME/share/hadoop/mapreduce/lib/*</value>
    </property>
</configuration>
etc/hadoop/yarn-site.xml
<configuration>
    <property>
        <name>yarn.nodemanager.aux-services</name>
        <value>mapreduce_shuffle</value>
    </property>
    <property>
        <name>yarn.nodemanager.env-whitelist</name>
        <value>JAVA_HOME,HADOOP_COMMON_HOME,HADOOP_HDFS_HOME,HADOOP_CONF_DIR,CLASSPATH_PREPEND_DISTCACHE,HADOOP_YARN_HOME,HADOOP_HOME,PATH,LANG,TZ,HADOOP_MAPRED_HOME</value>
    </property>
</configuration>
  1. ResourceManager 데몬과 NodeManager 데몬을 시작하세요:
$ sbin/start-yarn.sh

  1. ResourceManager의 웹 인터페이스를 둘러보세요. 기본적으로 다음 위치에서 사용할 수 있습니다:

    • ResourceManager - http://localhost:8088/

  2. MapReduce 작업을 실행하세요.

  3. 작업을 마쳤으면 다음 명령으로 데몬을 중지하세요:

$ sbin/stop-yarn.sh

Fully-Distributed Operation

완전 분산, 비단순 클러스터를 설정하는 정보는 Cluster Setup을 참조하세요.
