PostgreSQL 쿼리 정리

PostgreSQL

1. 계정 (User)

1.1 User 조회

모든 User 조회 (1)
-- postgres 8.1 이하 버전
SELECT * FROM PG_CATALOG.PG_SHADOW;

-- 최신 버전
SELECT * FROM PG_CATALOG.PG_USER;
모든 User 조회 (2)
\du (Display User roles)
현재 접속 User 조회
SELECT CURRENT_USER;

1.2 User 생성 / 변경 / 삭제

User 생성
CREATE USER <생성할 계정> [WITH 권한] PASSWORD <계정 비밀번호>;
User 이름 변경
ALTER USER <기존계정> RENAME TO <새로운 계정>;
User 삭제
DROP ROLE <계정명>;

1.3 User 권한 변경 / 삭제

User 권한(Role) 변경
ALTER USER <계정> WITH CREATEROLE;
ALTER USER <계정> WITH CREATEDB;
ALTER USER <계정> WITH Replication;
ALTER USER <계정> WITH Bypass RLS;
User 권한(Role) 삭제
-- CREATEDB 권한 삭제
ALTER USER <생성할 계정> WITH NOCREATEDB;

-- SUPERUSER 권한 삭제
ALTER USER <생성할 계정> WITH NOSUPERUSER;

1.4 특정 User에 특정 권한 부여 / 회수

특정 User에 특정 Database/Schema의 모든 권한 부여
GRANT ALL PRIVILEGES ON DATABASE <데이터베이스명> to <계정>;
GRANT ALL PRIVILEGES ON DATABASE <스키마> to <계정>;
특정 User에 특정 DatabaseE/Schema의 모든 권한 회수
REVOKE ALL PRIVILEGES ON SCHEMA <데이터베이스명> FROM <계정>;
REVOKE ALL PRIVILEGES ON SCHEMA <스키마> FROM <계정>;
특정 User에 특정 Table의 SELECT 권한 부여
GRANT SELECT ON <테이블> to <계정>;
특정 User에 특정 Table의 SELECT 권한 회수
REVOKE SELECT ON <테이블> FROM <계정>;

1.5 접속 User 변경

현재 Database에서 접속 계정 변경
SET ROLE <계정>;

1.6 User의 Schema 탐색 경로

User의 Schema 탐색 경로 지정
-- 단일 search_path
ALTER USER <계정> SET search_path = <스키마>; 

-- 복수 search_path
ALTER USER <계정> SET search_path = <스키마1, 스키마2>;

2. 데이터베이스 (Database)

2.1 Database 조회

\list
\l

2.2 Database 생성 / 변경 / 삭제

Database 생성
CREATE DATABASE <데이터베이스명> OWNER <사용자 계정>;
Database 변경
\c <데이터베이스명> <접속할 계정명>;
Database 삭제
DROP DATABASE <데이터베이스명>;

2.3 Database의 Schema 탐색 경로

Database의 Schema 탐색 경로 조회
SHOW SEARCH_PATH;
Database의 Schema 탐색 경로 지정
ALTER DATABASE <데이터베이스명> SET SEARCH_PATH=<스키마>;

3. 스키마 (Schema)

3.1 Schema 조회

전체 Schema 조회 (1)
SELECT * FROM PG_CATALOG.PG_NAMESPACE;
전체 Schema 조회 (2)
\dn

3.2 Schema 생성 / 변경 / 삭제

Schema 생성
CREATE SCHEMA <스키마>;
Schema 이름 변경
ALTER SCHEMA <기존 스키마> RENAME TO <새로운 스키마>;
Schema 소유주 변경
ALTER TABLE <스키마> OWNER TO <새로운 소유주>;
Schema 삭제
DROP SCHEMA <스키마>;

4. 테이블 (Table)

4.1 Table 조회

Table 조회
SELECT * FROM PG_CATALOG.PG_TABLES WHERE schemaname = <'스키마'>;

4.2 Table 생성 / 변경 / 삭제

Table 생성
CREATE TABLE <스키마>.<테이블> (
  <컬럼명1> <데이터타입> [옵션],
  <컬럼명2> <데이터타입> [primary key],
  <컬럼명3> <데이터타입> [NOT NULL],
  <컬럼명4> <데이터타입> [NULL]
);
Table 오너 USER 변경
ALTER TABLE <스키마명>.<테이블> OWNER TO <계정명>;
Table + 데이터 영구 삭제 (Rollback 불가)
DROP TABLE <스키마>.<테이블>;
데이터만 삭제 (Rollback 불가. 테이블과 스키마는 유지)
TRUNCATE TABLE <스키마명>.<테이블>

4.3 Table Comment 생성

Table/Column Comment 생성
-- 테이블에 Comment 붙이기
COMMENT ON TABLE <스키마>.<테이블> IS <'Comment'>;

-- 컬럼명에 Comment 붙이기
COMMENT ON COLUMN <스키마>.<테이블> IS <'Comment'>;

4.4 Table 사이즈(용량) 확인

Table 사이즈(용량) 확인
-- 전체 테이블 사이즈 확인
SELECT spcname, pg_size_pretty(pg_tablespace_size(spcname)) from pg_tablespace;

-- 특정 테이블 사이즈 확인
SELECT pg_total_relation_size('<테이블>');

-- 'MB', 'GB' 등 human readable한 단위로 사이즈 확인하기
SELECT pg_size_pretty(pg_total_relation_size('<테이블>'));
