pd.append() deprecated

Python

1. pandas.DataFrame.append() deprecated

pandas.DataFrame.append() 함수를 사용하면 FutureWarning이 발생하며, pandas.concat() 함수를 대신 사용하라는 경고가 나타난다. 그 이유는 pandas.DataFrame.append() 함수가 메모리 사용과 처리 성능 측면에서 비효율적인 문제를 가지고 있기 때문이다.

  • pandas.DataFrame.append()는 내부적으로 결합할 데이터프레임을 깊은 복사(deep copy)하여 결합한다.

  • 데이터가 클수록 처리할 때 메모리 부족(Out Of Memory) 발생 가능성이 높아지며, 처리 속도가 느려진다.

2. pandas.concat()

pandas.concat() 함수는 여러 데이터프레임을 Iterable 객체 안에 넣어 행 또는 열로 병합하는 함수이다. 이 함수는 옵션 설정을 통해 데이터프레임을 복사하지 않고 병합할 수 있다. 함수의 copy 옵션을 변경(default는 True)함으로써 데이터프레임을 복사하지 않고 연결할 수 있으며, 이로 인해 메모리 절약과 성능 향상의 이점이 있다.

3. pandas.DataFrame.from_dict()

원래 사용하던 pandas.DataFrame.append() 함수를 사용하지 않으려고 하다 보니 한 가지 불편한 문제가 생겼다. Kafka에서 받은 하나의 row를 데이터프레임에 추가하려고 할 때, append()에 인자로 딕셔너리를 스키마에 맞게 바로 넣어주면 데이터를 쉽게 추가할 수 있었다. 하지만 concat() 함수는 결합 대상을 dict가 아닌 Series 또는 DataFrame 객체의 시퀀스를 받기 때문에, 중간에 dict를 데이터프레임으로 변환하는 과정이 필요하게 되었다.

물론, 경고가 나오는 append() 함수를 사용하더라도 딕셔너리 객체 하나를 삽입하는 것이기 때문에 복사하는 데 큰 메모리가 소모되지는 않는다. 하지만 향후 deprecated될 가능성이 있고, 코드의 일관성을 맞추기 위해 concat() 함수를 사용하기로 했다.

하나의 데이터프레임에 하나의 딕셔너리 객체를 추가하려면 어떤 함수를 사용해야 할까? 그때 사용하기 좋은 함수가 바로 pandas.DataFrame.from_dict()이다. 이름에 걸맞게 딕셔너리를 데이터프레임으로 변환해주는 함수로, DataFrame 클래스의 클래스 메서드이다. 필수 파라미터로 data에 key와 value로 구성된 딕셔너리 객체를 넣어주면 된다.

from_dict() 함수를 이용해 데이터프레임을 행으로 병합하는 코드는 아래와 같다.  

from_dict() 함수를 이용해 데이터프레임을 행으로 병합하는 코드
import pandas as pd

# 딕셔너리 데이터
data = {
    'Name': ['John', 'Emma', 'Peter'],
    'Age': [25, 28, 30],
    'City': ['New York', 'London', 'Paris']
}

# dict to pandas.DataFrame
df = pd.DataFrame.from_dict(data)

print(df)

######### 출력 #########
#    Name  Age      City
# 0  John   25  New York
# 1  Emma   28    London
# 2 Peter   30     Paris

# 데이터프레임 결합하기
result = pd.conact([df1, pd.DataFrame.from_dict(data)], axis=0, ignore_index=True, copy=False)

concat() 사용 시 copy 옵션을 False로 설정할 때 주의해야 할 점은 다음과 같다.  

  • 데이터프레임으로 결합할 딕셔너리의 값(value)은 array-like 형태로 넣어주어야 한다.

  • 결합 대상이 되는 데이터프레임은 더 이상 바뀌지 않는 불변(immutable) 객체로 사용해야 한다.
