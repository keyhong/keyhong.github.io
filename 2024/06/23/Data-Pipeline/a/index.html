Pipeline Flow

Kafka Common Property

  • Kafka 구성 요소 속성에는 KAFKA_ 접두사를 붙이십시오. 예를 들어, metric.reporters는 Kafka 속성이므로 KAFKA_METRIC_REPORTERS로 변환해야 합니다.
  • Confluent 구성 요소(cp-server) 속성에는 CONFLUENT_ 접두사를 붙이십시오. 예를 들어, confluent.metrics.reporter.bootstrap.servers는 Confluent Enterprise 기능 속성이므로 CONFLUENT_METRICS_REPORTER_BOOTSTRAP_SERVERS로 변환해야 합니다.
Property Description Settings
KAFKA_CONTROLLER_LISTENER_NAMES 컨트롤러에서 사용되는 리스너의 이름을 쉼표로 구분하여 나열합니다. process.roles=broker로 설정된 노드에서는 이 목록에서 첫 번째 리스너만 브로커에서 사용됩니다. ZooKeeper 모드의 브로커는 이 값을 설정하지 않아야 합니다. KRaft 컨트롤러가 독립 또는 결합 모드로 동작하는 경우, 노드는 이 속성에 나열된 모든 리스너에서 KRaft 컨트롤러로서 대기하게 되며, 각 리스너는 listeners 속성에 나타나야 합니다. CONTROLLER
KAFKA_LISTENER_SECURITY_PROTOCOL_MAP 리스너 이름별로 사용할 보안 프로토콜에 대한 키/값 쌍을 정의합니다. 이는 server properties 파일에서의 listener.security.protocol.map 구성 매개변수와 동등합니다. ‘CONTROLLER:PLAINTEXT,INTERNAL:PLAINTEXT,EXTERNAL:PLAINTEXT’
KAFKA_INTER_BROKER_LISTENER_NAME Kafka 브로커 간 통신에 사용할 리스너를 정의합니다. Kafka 브로커들은 일반적으로 내부 네트워크 (예: Docker 네트워크, AWS VPC 등)에서 서로 통신합니다. 호스트/IP는 브로커 머신에서 다른 머신에 접근 가능해야 합니다. 이는 server properties 파일에서의 inter.broker.listener.name 구성 매개변수와 동등합니다. INTERNAL
KAFKA_CONTROLLER_QUORUM_VOTERS 쿼럼 유권자의 쉼표로 구분된 목록입니다. 각 컨트롤러는 {id}@{host}:{port} 형식으로 ID, 호스트 및 포트 정보로 식별됩니다. ‘1@broker1:29093,2@broker2:29093’
KAFKA_PROCESS_ROLES 이 서버의 역할 (controller, broker 또는 broker,controller). ‘broker,controller’
KAFKA_GROUP_INITIAL_REBALANCE_DELAY_MS Kafka 컨슈머 그룹이 처음 rebalance를 시작하기 전에 대기해야 하는 시간(밀리 초) 0
KAFKA_OFFSETS_TOPIC_REPLICATION_FACTOR 토픽 내부의 오프셋(offset) 데이터를 저장하는 특수한 토픽의 복제 팩터(Replication Factor)를 설정하는 데 사용합니다. 2
KAFKA_TRANSACTION_STATE_LOG_REPLICATION_FACTOR 트랜잭션 상태 로그(transaction state log)를 복제하는 데 사용되는 설정 변수 2
CLUSTER_ID 각 KRaft 클러스터는 CLUSTER_ID 변수에 고유한 클러스터 ID가 할당되어야 합니다. kafka-storage 도구를 사용하여 고유 ID를 생성하는 방법은 Generate and format IDs를 참조하십시오. ‘luV2kvSyR8G89qbhm_QS2w==’
KAFKA_LOG_DIRS Kafka 브로커가 메시지 로그를 저장하는 디렉토리 경로를 지정 /tmp/kraft-combined-logs
KAFKA_METRIC_REPORTERS Kafka 브로커가 메트릭을 보내는 리포터(reporter) 클래스를 지정하는 역할 io.confluent.metrics.reporter.ConfluentMetricsReporter
CONFLUENT_METRICS_REPORTER_TOPIC_REPLICAS 1
CONFLUENT_METRICS_ENABLE Confluent Control Center와 같은 모니터링 도구를 통해 Kafka 클러스터의 메트릭 수집을 활성화 또는 비활성화하는 데 사용 ‘true’
CONFLUENT_SUPPORT_CUSTOMER_ID Confluent의 지원 서비스를 이용하는 고객을 식별하는 고유한 식별자 annoymous

Kafka control-center Property

Property Description Settings
KAFKA_NODE_ID 서버의 고유 식별자 1
KAFKA_LISTENERS Kafka가 바인드하고 수신 대기하는 리스너, 호스트/IP 및 포트의 쉼표로 구분된 목록입니다. 더 복잡한 네트워킹 환경에서는 머신의 네트워크 인터페이스에 연결된 IP 주소가 될 수 있습니다. 기본값은 0.0.0.0으로, 모든 인터페이스에서 수신 대기함을 의미합니다. 이는 server properties 파일에서의 listeners 구성 매개변수와 동등합니다 (ZooKeeper 모드: CONFLUENT_HOME/etc/kafka/server.properties 또는 KRaft 모드: CONFLUENT_HOME/etc/kafka/kraft/server.properties).

다중 노드(production)에서 ZooKeeper 모드로 실행 중인 경우, Dockerfile에서 KAFKA_ADVERTISED_LISTENERS 속성을 외부 호스트/IP 주소로 설정해야 합니다. 그렇지 않으면 클라이언트는 기본적으로 내부 호스트 주소에 연결을 시도합니다. 단일 노드 환경, “bare metal”(VM 또는 Docker 없음)에서는 모든 것이 hostname 또는 단순히 localhost일 수 있습니다. 그러나 여러 노드와 같은 더 복잡한 네트워킹 설정에서는 추가 구성이 필요합니다.		 ‘INTERNAL://broker1:29092,CONTROLLER://broker1:29093,EXTERNAL://0.0.0.0:9092’
KAFKA_ADVERTISED_LISTENERS ZooKeeper 모드에 필요합니다. 호스트 이름이 광고되고 클라이언트가 도달할 수 있는 방법을 설명합니다. 이 값은 클라이언트가 사용할 수 있도록 ZooKeeper에 게시됩니다.

TLS/SSL 또는 SASL 프로토콜을 사용하는 경우, 엔드포인트 값은 다음 형식으로 프로토콜을 지정해야 합니다:

• SSL: SSL:// 또는 SASL_SSL://
• SASL: SASL_PLAINTEXT:// 또는 SASL_SSL://
KAFKA_ADVERTISED_LISTENERS 변수를 localhost:29092로 설정하십시오. 이렇게 하면 Kafka가 Docker 호스트에서 위치를 광고함으로써 컨테이너 외부에서 접근할 수 있습니다.		 ‘INTERNAL://broker1:29092,EXTERNAL://localhost:9092’
CONFLUENT_METRICS_REPORTER_BOOTSTRAP_SERVERS broker1:29092

Kafka Individual Property

Property Description Settings
CONTROL_CENTER_SCHEMA_REGISTRY_URL Schema Registry의 URL을 지정하는 데 사용 ‘http://schema-registry:8081‘
CONTROL_CENTER_REPLICATION_FACTOR 컨트롤 센터 토픽의 복제 팩터입니다. 운영 환경에서는 이 값을 3으로 설정하는 것을 권장합니다. 1
CONTROL_CENTER_INTERNAL_TOPICS_PARTITIONS Control Center 내부에서 사용하는 토픽의 파티션 수를 설정하는 데 사용 1
CONTROL_CENTER_MONITORING_INTERCEPTOR_TOPIC_PARTITIONS Control Center 모니터링 인터셉터 토픽의 파티션 수를 지정하는 데 사용 1
CONFLUENT_METRICS_TOPIC_REPLICATION 1
CONFLUENT_METRICS_ENABLE Confluent Control Center와 같은 모니터링 도구를 통해 Kafka 클러스터의 메트릭 수집을 활성화 또는 비활성화하는 데 사용 ‘false’
PORT 9021

Property Environment variable replacement or pattern Description
bootstrap.servers CONTROL_CENTER_BOOTSTRAP_SERVERS Kafka 클러스터에 초기 연결을 설정하는 데 사용되는 호스트:포트 쌍입니다. 여러 개의 부트스트랩 서버를 사용할 수 있으며, host1:port1,host2:port2,host3:port3... 형식으로 지정할 수 있습니다.
