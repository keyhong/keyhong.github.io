[PRJ] 기술을 통해 자동화된 데이터 품질 검사 시스템 구축하기(in-progress)

Publish Date:   1996-01-19

데이터 업무를 하면서 느끼는 문제

데이터를 다루는 각 직업은 우선적인 가치관이 다릅니다. 예를 들어, DA는 안정적인 데이터 플랫폼 구축을 고민하고, 모델러는 비즈니스 분석과 테이블 간 관계 표현을 고민합니다. 이 외에도 데이터 사이언티스트, 데이터 엔지니어, 머신러닝 엔지니어 등 다양한 직업이 있습니다. 데이터 품질을 유지하는 관점에서 이들의 생각을 모두 존중하지만, 때때로 품질에 대한 관심이 후순위로 밀려나는 경우가 있어 고민이 많습니다. 저 역시 이런 문제를 겪었고, 이를 기술적으로 어떻게 해결할 수 있을지 고민해 보았습니다.

1. 정돈되지 않은 SQL과 SQL을 정돈하는 데 필요한 많은 시간

“규원님 BI 차트가 안나오는 데, 혹시 에어플로우 확인해줄 수 있어요?”

“음.. 프로시저는 정상적으로 호출됬는 데, 왜 안 나오지? 쿼리를 봐야겠어요!”

“(5분 후) 혹시.. 이 쿼리 짜신 분 있을까요??”

정돈되지 않은 쿼리는 데이터 다운타임을 길게 만드는 문제가 있습니다. DA에게 SQL 포맷을 맞춰달라고 부탁하기도 어렵고, 포맷 기준에 대한 토론이 자주 벌어지곤 합니다.

그럼에도 불구하고, 팀 내부 혹은 전사적으로 최소한의 쿼리 규칙을 만들어 지키는 것은 데이터 조직의 성숙도를 높이는 데 중요한 역할을 합니다. 하지만 그 방법을 구체화하는 것은 별개의 문제입니다.

모델러보다 훨씬 앞서 이 문제를 접한 사람들은 개발자들입니다. 개발자들은 린팅과 포매터 같은 정적 분석 도구를 통해 이 문제를 극복했고, 코드 품질을 높이는 데 큰 기여를 했습니다. SQL에서도 유사한 도구가 존재하지만, 많은 사람들이 이를 활용하지 못하고 있습니다. 이 글에서는 그 도구들을 소개하고자 합니다.

SQLFluff

SQLFluff는 방언(dialect)에 유연하고 구성 가능한 SQL 린터입니다.

방언이란 특정 DBMS에 맞춘 SQL의 변형을 의미합니다. MySQL, PostgreSQL, Oracle SQL 등 각각의 DBMS는 SQL에서 약간씩 다르게 동작하는 부분이 있습니다. 이런 다양한 DBMS의 쿼리를 특정 포맷에 맞게 변환해주는 기능입니다. 직접 보면 이해가 빠릅니다.

이를 어떻게 활용할 수 있을까요? 환경을 구성하는 것 자체는 쉽습니다. 다만, 어떤 포맷으로 다듬을 것인지에 대한 .sqlfluff 파일과 vscode와 연계하여 자동화된 환경 구성을 제공하는 것이 사용자 입장에서 편리합니다. 이러한 기본 환경을 GitHub를 통해 배포했습니다.

2. 품질을 검사하는 방법

품질을 유지하려는 사람들에게 가장 큰 고민은 아마도 실수를 줄이는 것일 것입니다. 어떤 경험 많은 사람이라도 인간이기에 실수를 범할 수 있습니다. 이를 줄이기 위해서는 자동화된 검증 시스템을 구축해야 하며, 이를 통해 빠른 문제 발견과 보완이 가능해져야 합니다.

그러나 문제는 어떤 도구를 사용하여 강력한 시너지를 만들 수 있는가입니다. 데이터를 다루는 과정에서 가장 크게 느끼는 것은 다양성입니다. 데이터의 형식, 쿼리 엔진, DBMS는 고정되어 있지 않고 계속 발전하고 있습니다. 품질은 표준을 준수하는 것이 중요합니다. 그러나 표준을 준수하는 행위는 변화에도 영향을 받지 않습니다.

전 회사에서 20년 차 경력의 DBA이신 팀장님의 말씀을 듣고 배웠습니다. "DBA마다 다르지만, 나는 ANSI 쿼리를 사용해. 다른 DBMS에서도 재사용할 수 있거든."

표준은 어디에서든 범용적으로 활용 가능하다고 생각합니다. 그리고 표준은 많은 사람들 사이에서 가장 많은 채택을 받은 것입니다. 이는 문제가 발생해도 계속해서 유지보수되는 것입니다. 그래서 범용적인 도구로 사람이 아닌 시스템에 의한 자동화된 품질 검사기를 만들어보고자 합니다.

DBT + Aiflow + Trino

Data Quality

“데이터 품질은 주어진 데이터셋이 사용자의 요구를 얼마나 잘 충족하는지에 따라 정의됩니다. 데이터 품질은 데이터 기반 결정이 가능한한 정확하게 이루어지도록 하는 중요한 기준입니다.”

Data Quality Dimensions

차원 정의 방법
🔥 정확성(Accuracy) 데이터는 “현실”을 정확하게 나타낸다. 데이터 교차 검증, 정기 데이터 검사
🔥 일관성(Consistency) 데이터 값은 다른 데이터 셋의 값들과 충돌하지 않는다. 데이터 표준화, 데이터 정리 자동화, 오류 보고
🔥 완전성(Completeness) 특정 용도에 필요한 모든 데이터가 존재하며 사용할 수 있다. 누락 데이터, NULL 값 분석, 레코드 카운팅
유효성(Validity) 데이터는 예상된 포맷, 타입 및 범위에 맞춘다. 데이터 유형 확인, 범위 확인, 패턴 일치
적시성(Timeliness) 데이터는 예상되고 필요할 때 이용 가능하다. 타임스탬프 분석, 실시간 모니터링
유일성(Uniqueness) 데이터는 데이터 셋에서 한 번만 나타난다. 중복 없음 확인

🔥정확성, 🔥일관성, 🔥완전성을 갖추면 데이터 무결성(Data Integrity)을 갖췄다고 한다.
